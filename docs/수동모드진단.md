## 1. 용어 정리 — “수동”이 코드에서 쓰이는 네 가지 층

| 구분 | 위치 | 의미 |
|------|------|------|
| **A. 탭 헤더 자동(Auto) / 수동(Manual)** | `StructureHeader`, `FunctionL1Header`, `FailureL1Header` 등 | 마스터 DB에서 추천 목록을 끌어오는 **자동** vs 셀/우클릭 중심 **수동** 편집 UX. Import 유무와 무관하게 탭별 `isAutoMode` 상태로 토글. |
| **B. 상단 입력모드 `inputMode`** | `page.tsx` → `TopMenuBar` 등 | 주석상 의도: `auto` = 기존 모달 중심, `manual` = 컨텍스트 메뉴 중심. **TopMenuBar에서는 2026-02-05 이후 수동/자동 토글 UI가 제거**되어, `inputMode`는 주로 사이드 패널 등에만 전달되는 형태. |
| **C. Import “수동 템플릿”** | `TemplateGeneratorPanel`, `excel-template.ts` | 엑셀 템플릿 모드. `MANUAL_MODE_EXCLUDE`로 일부 시트(고장·FC 시트 등) 제외. **워크시트 구조 탭의 수동 행 편집과는 별도 축.** |
| **D. API `manualMode` (DB)** | `POST /api/fmea/save-position-import` | `atomicDBRef` 기준 **L2가 아직 없을 때** `buildManualPositionData` 결과를내며 **`manualMode: true`**. L2/L3 구조만 안전하게 동기화(FM 없을 때 구조 클린 리셋 등). |

진단에서 말하는 **“수동 모드”**는 주로 **A(탭별 Auto/Manual) + 구조/기능/고장 탭의 우클릭 메뉴·모달 편집 + D(신규 FMEA 구조 DB 선저장)** 을 가리킨다.

## 2. DB · 저장 파이프라인

### 2.1 이원화 저장 (`useWorksheetSave.ts`) — **코드 기준 정확 분기**

`save`는 **`atomicDBRef.current`(ref)** 를 본다(`setAtomicDB` 직후에도 최신값).

| 조건 | 동작 |
|------|------|
| `db && Array.isArray(db.l2Structures) && db.l2Structures.length > 0` | `syncConfirmedFlags` 후 **`saveAtomicDBDirect`** (일반 Atomic 저장). |
| **그 외** (`db` 없음 **또는** `l2Structures` 비어 있음) | `state.l2`에서 이름/번호 있는 공정만 → **`buildManualPositionData`** → **`POST ... save-position-import`** `{ manualMode: true }`. |

**시사점 (문서 정정):**

- “`atomicDB` 객체만 없을 때”가 아니라, **Atomic에 적재된 L2가 한 건도 없으면** Path D를 탄다. Import 직후 로더가 L2를 채우기 전·빈 DB 객체 등과도 맞물릴 수 있다.
- Import 후 `l2Structures.length > 0`이면 Path D는 타지 않고 **Atomic 직접 저장**만 사용한다.
- 코드 주석의 “atomicDB 없음”은 **실제 조건보다 좁게 읽히므로** 구현 주석은 “L2 없음 → 수동 구조 선저장”으로 정리하는 것이 안전하다.

### 2.2 `manualStructureBuilder.ts`

- `WorksheetState.l2[]` → 최소 `PositionAtomicData` (L1/L2/L3 구조, FM/FC/FL/RA 빈 배열).
- 공정·작업요소 **클라이언트 id 보존** (`proc_new_…`, `L2-MAN-…` 등).
- 유효 공정이 0이면 `null` → 저장 스킵.

### 2.3 `save-position-import` — `manualMode && !force`

- **FM 건수가 0이면:** 프로젝트 스키마에서 L3 보조 엔티티 → L3Structure → L2 보조 → L2Structure 순 **구조만 삭제 후 재삽입**에 가까운 동기화(고아 FM/FC 방지 목적, 주석 참고).
- **FM이 이미 있으면:** 구조 전량 삭제는 하지 않고 **skipDuplicates upsert 폴백** 로그만 남김.

**위험도:** 수동으로 구조만 바꾸고 FM이 이미 있는 DB에서는 `manualMode`가 “완전 구조 리셋”을 하지 않을 수 있음 → 기대와 다를 수 있으니 운영 시 시나리오별 확인 필요.

### 2.4 `manualMode` + FM>0 — QA 체크리스트 (☆3)

| 단계 | 확인 |
|------|------|
| 1 | 신규 PFMEA에서 구조만 저장 → `failure_modes` 0건일 때 L2/L3 재삽입 로그 `[save-position-import] manualMode: L2/L3 구조 동기화 (FM 없음 — 안전)` |
| 2 | 동일 프로젝트에 FM(고장형태) 1건 이상 생성·저장 후, 워크시트에서 공정/작업요소만 대량 수정·저장 |
| 3 | 서버 로그에 `manualMode: FM=n건 존재 — skipDuplicates 폴백` 가 나오는지 확인 |
| 4 | PG(또는 `pipeline-verify` GET)에서 **고아 FM/FC/FL** 없음, 화면 구조와 DB L2/L3 명칭·순서가 기대와 일치하는지 **수동 대조** |
| 5 | 기대와 다르면 **Import `force` 클린 재적재** 또는 워크시트 Atomic 경로(`saveAtomicDBDirect`)로만 구조를 맞추는지 정책 결정 — 자동 카테시안 FL 생성은 금지(Rule 0.5) |

### 2.5 localStorage

- **DB Only 정책:** 워크시트 본문·riskData는 localStorage 의존 제거 방향(주석 다수). 탭/visibleSteps 등 일부 메타만 유지.
- **진짜 SSoT:** 프로젝트 스키마 Atomic + `loadAtomicDB` → `atomicToLegacy`.

## 6. URL · E2E와의 정합

- **Playwright `manual-mode-guard.spec.ts`:** `?fmeaNo=...&mode=manual` 로 진입.
- **`useWorksheetState`:** `worksheetFmeaKeyFromSearchParams` — **`id` / `fmeaId` / `fmeaNo` 순으로 동일 취급**(2026-03-29). 프로젝트 목록의 `id`는 API에서 `fmeaNo || id`로 내려오는 경우가 많아, 북마크·가드 URL의 `fmeaNo`와 정합.
- **`mode` 쿼리:** `mode === 'inherit'` 일 때만 **상속(baseId)** 로직이 동작한다. `mode=manual`은 워크시트 **핵심 데이터 분기**가 아니다.
- **단일 PFMEA 자동 리다이렉트:** `id` 없이 진입 시 프로젝트가 1개면 `id`를 붙여 `router.push` — **기존 쿼리 전체(`mode`, `tab` 등)를 유지**하도록 정리됨(2026-03-29). 이전에는 `tab`만 재부착해 `mode=manual`이 떨어질 수 있었다.
- **canonical URL:** 탭 동기화 이펙트가 `id`를 표준 쿼리로 맞추므로, 최종 주소는 보통 `?id=...` 형태로 수렴한다.
