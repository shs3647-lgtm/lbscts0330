# PFMEA 워크시트 수동 모드 정밀 진단

**작성일:** 2026-03-28  
**범위:** DB 저장, 통합 저장 훅, 컨텍스트 메뉴, 모달 입력·추가·삭제, 구조분석 → 3L 기능분석 → 3L 고장분석  
**방법:** 코드 정적 분석 (`src/app/(fmea-core)/pfmea/worksheet`, `useWorksheetSave`, `save-position-import`, E2E 가드 스펙)

---

## 1. 용어 정리 — “수동”이 코드에서 쓰이는 네 가지 층

| 구분 | 위치 | 의미 |
|------|------|------|
| **A. 탭 헤더 자동(Auto) / 수동(Manual)** | `StructureHeader`, `FunctionL1Header`, `FailureL1Header` 등 | 마스터 DB에서 추천 목록을 끌어오는 **자동** vs 셀/우클릭 중심 **수동** 편집 UX. Import 유무와 무관하게 탭별 `isAutoMode` 상태로 토글. |
| **B. 상단 입력모드 `inputMode`** | `page.tsx` → `TopMenuBar` 등 | 주석상 의도: `auto` = 기존 모달 중심, `manual` = 컨텍스트 메뉴 중심. **TopMenuBar에서는 2026-02-05 이후 수동/자동 토글 UI가 제거**되어, `inputMode`는 주로 사이드 패널 등에만 전달되는 형태. |
| **C. Import “수동 템플릿”** | `TemplateGeneratorPanel`, `excel-template.ts` | 엑셀 템플릿 모드. `MANUAL_MODE_EXCLUDE`로 일부 시트(고장·FC 시트 등) 제외. **워크시트 구조 탭의 수동 행 편집과는 별도 축.** |
| **D. API `manualMode` (DB)** | `POST /api/fmea/save-position-import` | `atomicDB`가 비어 있을 때 `buildManualPositionData` 결과를내며 **`manualMode: true`**. L2/L3 구조만 안전하게 동기화(FM 없을 때 구조 클린 리셋 등). |

진단에서 말하는 **“수동 모드”**는 주로 **A(탭별 Auto/Manual) + 구조/기능/고장 탭의 우클릭 메뉴·모달 편집 + D(신규 FMEA 구조 DB 선저장)** 을 가리킨다.

## 2. DB · 저장 파이프라인

### 2.1 이원화 저장 (`useWorksheetSave.ts`)

| 조건 | 동작 |
|------|------|
| `atomicDBRef.current`에 `l2Structures.length > 0` | `syncConfirmedFlags`로 레거시 state와 동기화 후 **`saveAtomicDBDirect`** (일반 Atomic 저장). |
| 그 외(신규·수동으로만 채운 state 등) | `state.l2`에서 이름/번호 있는 공정만 모아 **`buildManualPositionData`** → **`POST /api/fmea/save-position-import`** with `{ manualMode: true }`. |

**시사점:** Import 후 Atomic이 채워지면 **수동 전용 경로(D)는 타지 않고** 상단 동기화 경로로만 간다. 반대로 **DB에 L2가 아직 없는 신규 프로젝트**는 저장 시 D 경로가 핵심이다.

### 2.2 `manualStructureBuilder.ts`

- `WorksheetState.l2[]` → 최소 `PositionAtomicData` (L1/L2/L3 구조, FM/FC/FL/RA 빈 배열).
- 공정·작업요소 **클라이언트 id 보존** (`proc_new_…`, `L2-MAN-…` 등).
- 유효 공정이 0이면 `null` → 저장 스킵.

### 2.3 `save-position-import` — `manualMode && !force`

- **FM 건수가 0이면:** 프로젝트 스키마에서 L3 보조 엔티티 → L3Structure → L2 보조 → L2Structure 순 **구조만 삭제 후 재삽입**에 가까운 동기화(고아 FM/FC 방지 목적, 주석 참고).
- **FM이 이미 있으면:** 구조 전량 삭제는 하지 않고 **skipDuplicates upsert 폴백** 로그만 남김.

**위험도:** 수동으로 구조만 바꾸고 FM이 이미 있는 DB에서는 `manualMode`가 “완전 구조 리셋”을 하지 않을 수 있음 → 기대와 다를 수 있으니 운영 시 시나리오별 확인 필요.

### 2.4 localStorage

- **DB Only 정책:** 워크시트 본문·riskData는 localStorage 의존 제거 방향(주석 다수). 탭/visibleSteps 등 일부 메타만 유지.
- **진짜 SSoT:** 프로젝트 스키마 Atomic + `loadAtomicDB` → `atomicToLegacy`.

## 3. 컨텍스트 메뉴 (`PfmeaContextMenu.tsx`)

공통 구현 파일에 **기능 설명 주석**이 전부 정의되어 있다.

| 메뉴 항목 | 조건 | 비고 |
|-----------|------|------|
| 위로 새 행 추가 / 아래로 새 행 추가 | `onInsertAbove` / `onInsertBelow` 제공 시 | `columnType`이 구조열이면 라벨에 L1/L2/L3 표시 |
| 위/아래 병합 추가 | `onAddMergedAbove` / `onAddMergedBelow` | 탭에서 넘겨줄 때만 |
| 행 삭제 | `onDeleteRow` | |
| 위/아래 행과 병합, 병합 해제 | `isMergeableColumn`(l1,l2,l3,fm,fc) 이고 핸들러 존재 시 | |
| 실행취소 / 다시실행 | `onUndo` / `onRedo`, 카운트 | |
| 고장연결 추가/삭제 | `onAddFailureLink` / `onDeleteFailureLink`, fmId/fcId | 고장연결 탭 위주 |
| 복사/붙여넣기 | `onCopyRow` / `onPasteRow` | 탭별 구현 여부 다름 |

**이벤트:** `stopPropagation` + `type="button"` + 메뉴 `onMouseDown` 차단(2026-03-23 Strict Mode 중복 삽입 방지).

## 4. 탭별 연결 상태 (구조 → 3L 기능 → 3L 고장)

### 4.1 구조분석 `StructureTab.tsx`

| 영역 | 동작 | DB |
|------|------|-----|
| L1 더블클릭 편집 | `setStateSynced` + `saveToLocalStorage` | Atomic 경유 여부는 `atomicDB` 존재 시 `ProcessSelectModal`/`saveNow` 등 |
| 공정 선택 | `ProcessSelectModal` — `atomicDB` 필수 분기(`!atomicDB`면 early return) | `updateL2Structure` / `addL2Structure` + `saveNow` |
| 작업요소 | `WorkElementSelectModal` — 동일하게 `atomicDB` 필요 | `replaceL3Structures` + `saveNow` |
| 우클릭 메뉴 | `PfmeaContextMenu`: **위/아래 삽입, 삭제만** (병합/Undo 미연결) | 핸들러 내부에서 `useAtomicView` + `setStateSynced` + `atomicDB` 갱신 |
| 자동/수동 토글 | 헤더 `StructureHeader` | 마스터 로딩과 연동 |

**주의:** `ProcessSelectModal.onSave` 첫 줄이 `if (!props.atomicDB \|\| !props.setAtomicDB) return;` 이므로, **Atomic이 한 번도 안 잡힌 초기 상태**에서는 모달 저장이 **무동작**일 수 있다. 신규 FMEA는 `useWorksheetDataLoader`가 `createEmptyDB`로 `atomicDB`를 채우는 경로가 있어 보통은 객체가 존재한다.

### 4.2 기능분석 L1 `FunctionL1Tab.tsx`

- `PfmeaContextMenu`: 삽입/삭제 + (구현에 따라) 병합 계열.
- `isAutoMode`에 따라 마스터 자동 채움 vs 수동 표시 범위가 달라짐.

### 4.3 기능분석 L2 `FunctionL2Tab.tsx`

- 컨텍스트 메뉴: **위/아래 삽입, 삭제, 위/아래 병합 추가** (`onAddMergedAbove/Below`).
- 수동 모드 주석: **모든 기능·제품특성 행 표시**(placeholder 포함 slice).

### 4.4 기능분석 L3 `FunctionL3Tab.tsx`

- L2와 유사하게 `PfmeaContextMenu` + 수동 시 전체 행 표시 로직.

### 4.5 고장분석 L1 `FailureL1Tab.tsx`

- `useFailureL1Handlers` + `PfmeaContextMenu`.
- 마스터 연동·확정 플로우가 기능분석과 대칭.

### 4.6 고장분석 L2 `FailureL2Tab.tsx`

- **로컬 `useState(false)`로 `isAutoMode` 별도 관리**(다른 탭과 상태 공유 아님).
- upstream 확정 여부에 따라 마스터 로드 등 분기.

### 4.7 고장분석 L3 `FailureL3Tab.tsx`

- 동일 패턴: `PfmeaContextMenu`, 트리/미리보기, 확정 시 `saveAtomicDB`.

## 5. 수동 모달 · 데이터 입력 (대표)

| 모달 | 파일 | 저장 반영 |
|------|------|-----------|
| 공정 선택 | `ProcessSelectModal` | `atomicDB` + `state.l2` + `saveNow` |
| 작업요소 선택 | `WorkElementSelectModal` | `replaceL3Structures` + state `l2[].l3` |
| 데이터 선택(계층) | `DataSelectModal.tsx` | 수동 입력 시 중복 검사·로그, 상위 콜백으로 워크시트 반영 |
| 특별특성 | `SpecialCharSelectModal` 등 | 탭별 핸들러 + `saveAtomicDB` 디바운스/지연 호출 패턴 다수 |

**삭제:** 구조 탭 공정 삭제는 모달 `onDelete`에서 `deleteL2Structure` + `saveNow` + state 필터. 행 삭제는 컨텍스트 메뉴 → 탭별 cascade 로직.

## 6. URL · E2E와의 정합

- **Playwright `manual-mode-guard.spec.ts`:** `?fmeaNo=...&mode=manual` 로 진입.
- **`useWorksheetState`:** `mode`는 **`inherit` 분기**에만 쓰이고, **`manual` 문자열로 워크시트 모드를 바꾸는 로직은 없음**.
- **실제 선택 FMEA id:** 쿼리는 주로 `id` / `fmeaId` (`useWorksheetState`). 테스트 URL의 `fmeaNo`만으로는 id가 안 잡힐 수 있어, **E2E는 프로젝트 등록·리다이렉트 전제 또는 URL 규약을 맞춰야** 한다.

## 8. 종합 상태 표 (요약)

| 구분 | 상태 | 비고 |
|------|------|------|
| 신규 FMEA 구조만 수동 입력 후 저장 | **동작 설계됨** | `manualMode` + `buildManualPositionData` |
| Import 후 Atomic 풀 로드 후 수동 편집 | **동작** | `syncConfirmedFlags` + `saveAtomicDBDirect` |
| 구조 탭 우클릭 (삽입/삭제) | **연결됨** | 병합/Undo는 미연결 |
| 기능 L2/L3 우클릭 | **삽입·삭제·병합 추가** | 탭별 구현 차이 |
| 고장 L1~L3 우클릭 | **연결됨** | L2는 `isAutoMode` 로컬 상태 |
| `inputMode`(page) | **부분 사용** | TopMenuBar 토글 제거됨 |
| URL `mode=manual` | **워크시트 핵심 분기 아님** | inherit 외 미사용에 가까움 |
| `atomicDB` null 시 구조 모달 저장 | **스킵 가능** | 방어 코드 확인 필요 |

## 9. 권장 후속 (코드 변경 없이 운영/QA 관점)

1. **신규 FMEA:** 공정 추가 → 저장 → `pipeline-verify` 또는 DB에서 L2/L3 존재 확인.  
2. **FM 존재 후 구조 대량 수정:** `manualMode` 폴백 동작(구조 클린 미실행)을 알고 수동 검증.  
3. **E2E:** `fmeaNo` vs `id` 쿼리 통일 여부 확인 후 가드 스펙 유지보수.  
4. **문서:** Import “수동 템플릿”과 워크시트 “수동(Manual)” 헤더를 사용자 매뉴얼에서 용어 분리해 혼동 방지.
