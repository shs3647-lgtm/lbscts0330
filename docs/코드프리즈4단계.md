## CODEFREEZE v3.1.0 — 절대 규칙

이 디렉토리의 파일들은 **코드프리즈 상태**입니다.

### 금지 행위
- 이 디렉토리 내 어떤 파일도 **직접 수정 금지**
- 리팩토링, 최적화, 정리 명목의 수정 **금지**
- import 경로 변경 **금지**
- 타입 정의 변경 **금지**

### 허용 행위
- 파일 **읽기/참조**만 허용
- 이 디렉토리 **외부**에서 이 모듈을 호출하는 것은 허용
- 버그 수정이 필요하면 **반드시 사용자에게 먼저 보고**하고 승인받은 후에만 수정

### 수정이 꼭 필요한 경우
1. 사용자에게 "CODEFREEZE 파일 수정이 필요합니다" 경고
2. 변경 사유와 영향 범위를 설명
3. 사용자 명시적 승인 후에만 수정
4. 수정 전 원본 코드를 주석으로 보존
4단계: 골든 테스트 (최후의 방어선)
AI가 규칙을 무시하고 수정하더라도, 테스트가 깨지면 바로 감지됩니다. JG1 데이터를 기반으로 스냅샷 테스트를 만들어두면 가장 확실합니다:
typescript
// __tests__/golden/jg1-import.test.ts
describe('JG1 골든 테스트 — CODEFREEZE 검증', () => {
  it('FC 시트 파싱: 589 체인', () => {
    const chains = parseFCSheet(jg1Workbook);
    expect(chains.length).toBe(589);
  });

  it('FM 고유: 107건', () => {
    const fmSet = new Set(chains.map(c => `${c.processNo}|${c.fmValue}`));
    expect(fmSet.size).toBe(107);
  });

  it('FC 고유: 251건', () => {
    const fcSet = new Set(chains.map(c => `${c.processNo}|${c.m4}|${c.fcValue}`));
    expect(fcSet.size).toBe(251);
  });

  it('FE 고유: 21건', () => {
    const feSet = new Set(chains.map(c => c.feValue));
    expect(feSet.size).toBe(21);
  });

  it('공정 17개', () => {
    const procSet = new Set(chains.map(c => c.processNo));
    expect(procSet.size).toBe(17);
  });
});
