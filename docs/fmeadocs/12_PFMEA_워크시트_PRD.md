## 1. 개요

### 1.1 목적
PFMEA(Process Failure Mode and Effects Analysis) 워크시트는 공정 잠재적 고장 모드 및 영향 분석을 수행하는 웹 기반 도구입니다.

### 1.2 핵심 기능
- **구조분석**: 완제품 → 메인공정 → 작업요소(4M) 계층 구조 정의
- **기능분석**: 각 레벨별 기능 및 특성 정의
- **고장분석**: 고장영향(FE) → 고장형태(FM) → 고장원인(FC) 분석
- **고장연결**: FE-FM-FC 연결 및 고장 사슬 관리
- **리스크 평가**: S(심각도), O(발생도), D(검출도) 기반 RPN 계산

### 1.3 기술 스택
- **Frontend**: Next.js 14 (App Router), React 18, TypeScript
- **Styling**: TailwindCSS + 인라인 스타일
- **State Management**: React useState + useCallback
- **Storage**: localStorage (실시간) + PostgreSQL/Prisma (확정)
- **Testing**: Playwright E2E, Jest Unit

## 7. 단계별 기능 상세

### 7.1 구조분석 (Structure)

#### 목적
완제품에서 작업요소까지의 계층 구조를 정의합니다.

#### 데이터 구조
```typescript
interface L1Structure {
  id: string;
  name: string;        // 완제품명
  types: L1Type[];     // 구분 (YP/SP/USER)
}

interface L2Structure {
  id: string;
  no: string;          // 공정번호 (10, 20, ...)
  name: string;        // 공정명
  l3: L3Structure[];   // 작업요소 목록
}

interface L3Structure {
  id: string;
  name: string;        // 작업요소명
  m4: string;          // 4M 카테고리 (MN/MC/IM/EN)
}
```

#### 화면 구성
| 열 | 설명 | 너비 |
|----|------|------|
| 완제품공정명 | L1 공정명 | 120px |
| No | 공정번호 | 50px |
| 메인공정명 | L2 공정명 | auto |
| 4M | 4M 카테고리 | 50px |
| 작업요소 | L3 작업요소명 | auto |

| 제품특성 | 특성명 | 주황색 줄무늬 |

### 7.5 고장분석 1L (Failure L1) - 고장영향(FE)

#### 목적
요구사항 미충족 시 고장영향을 정의합니다.

#### 데이터 구조
```typescript
interface FailureEffect {
  id: string;
  reqId: string;       // 요구사항 ID
  effect: string;      // 고장영향
  scope: string;       // 범위 (YP/SP/USER)
  severity: number;    // 심각도 (1-10)
}
```

---

### 7.6 고장분석 2L (Failure L2) - 고장형태(FM)

#### 목적
제품특성 기준으로 고장형태를 정의합니다.

#### 데이터 구조
```typescript
interface FailureMode {
  id: string;
  productCharId: string;  // 제품특성 ID
  name: string;           // 고장형태
  sc?: string;            // 특별특성
}
```

#### ⚠️ 중요 규칙
> **공정별 독립 데이터**: 동일한 제품특성 이름이라도 공정이 다르면 별도 데이터로 관리됩니다. 자동연결 기능이 제거되어 각 공정별로 독립적으로 고장형태를 입력해야 합니다.

## 8. 고장연결 기능

### 8.1 개요
고장연결은 FE(고장영향) ↔ FM(고장형태) ↔ FC(고장원인)를 연결하여 고장 사슬을 완성합니다.

### 8.2 화면 레이아웃
```
┌────────────────────────────────────────────────────────────────────┐
│              P-FMEA 고장 분석(4단계) - 고장연결                    │
├────────────────────────────────────────────────────────────────────┤
│ ┌──────────┬──────────┬──────────┐ ┌────────────────────────────┐ │
│ │ FE(37%) │ FM(26%) │ FC(37%) │ │      고장사슬 다이어그램     │ │
│ │ 고장영향 │ 고장형태 │ 고장원인 │ │                            │ │
│ │          │          │          │ │    [FE] ←→ [FM] ←→ [FC]   │ │
│ │          │          │          │ │                            │ │
│ └──────────┴──────────┴──────────┘ └────────────────────────────┘ │
└────────────────────────────────────────────────────────────────────┘
```

### 8.3 테이블 비율
| 테이블 | 비율 | 설명 |
|--------|------|------|
| FE (고장영향) | 37% | 좌측 |
| FM (고장형태) | 26% | 중앙 |
| FC (고장원인) | 37% | 우측 |

### 8.4 연결 방법
1. **FM 선택**: 중앙 FM 테이블에서 고장형태 클릭
2. **FE 연결**: 좌측 FE 테이블에서 고장영향 클릭 (토글)
3. **FC 연결**: 우측 FC 테이블에서 고장원인 클릭 (토글)
4. **연결확정**: [연결확정] 버튼 클릭 또는 Enter 키

### 8.5 버튼 기능
| 버튼 | 기능 | 상태 |
|------|------|------|
| 연결확정 | 현재 FM 연결 저장 | FE/FC 선택 시 활성화 |
| 연결해제 | 현재 FM 연결 삭제 | 이미 연결된 FM 선택 시 |
| 전체확정 | 전체 연결 DB 저장 | 연결 데이터 존재 시 |
| 완료 | 완료 표시 | 모든 FM 연결 완료 시 |
| 수정 | 수정 모드 전환 | 확정 후 수정 필요 시 |

### 8.6 고장 사슬 다이어그램
```
   [고장영향 1]  ←→  [고장형태]  ←→  [고장원인 1]
   [고장영향 2]  ↗       ↓       ↘  [고장원인 2]
                    [S:8]            [O:6]
```

## 9. 트리뷰 패널

### 9.1 목적
현재 탭의 데이터를 계층적 트리 구조로 시각화합니다.

### 9.2 탭별 트리뷰
| 탭 | 트리뷰 제목 | 계층 구조 |
|----|------------|----------|
| 구조분석 | 구조트리 | 완제품 → 메인공정 → 작업요소 |
| 1L기능 | 1L 기능트리 | 완제품 → 구분 → 기능 → 요구사항 |
| 2L기능 | 2L 기능트리 | 메인공정 → 기능 → 제품특성 |
| 3L기능 | 3L 기능트리 | 메인공정 → 작업요소 → 기능 → 공정특성 |
| 1L고장 | 1L 고장영향 | 완제품 → 구분 → 기능 → 요구사항 → 고장영향 |
| 2L고장 | 2L 고장형태 | 메인공정 → 기능 → 제품특성 → 고장형태 |
| 3L고장 | 3L 고장원인 | 메인공정 → 작업요소 → 공정특성 → 고장원인 |

### 9.3 트리뷰 색상
- **헤더**: 탭별 테마 색상
- **아이템**: 레벨별 색상 (L1 파란, L2 녹색, L3 주황)
- **배지**: 4M/특별특성 표시

