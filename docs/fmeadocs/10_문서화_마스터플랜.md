## 📋 문서화 원칙

### 개발-테스트-배포 사이클

```
┌─────────────────────────────────────────────────────────────────┐
│  1. PRD 작성     →  2. 코드 최적화  →  3. 수동 검증           │
│       ↓                   ↓                  ↓                 │
│  4. 커밋         →  5. 코드프리즈   →  6. 문서 업데이트       │
│       ↓                                      ↓                 │
│  7. Git Push     ←─────────────────────────────                │
└─────────────────────────────────────────────────────────────────┘
```

### 각 화면별 PRD 구성

| 섹션 | 내용 |
|------|------|
| **1. 워크플로우** | 사용자 흐름도, 진입/이탈 경로 |
| **2. 화면 기능 스펙** | 컴포넌트, 상태, API, 데이터 흐름 |
| **3. QA 체크리스트** | 기능별 테스트 항목, PASS/FAIL 기록 |

## 🔄 작업 프로세스

### Phase 1: PRD 작성
```
1. 워크플로우 다이어그램 작성
2. 화면 구성 요소 정의
3. API 및 데이터 흐름 명세
4. QA 체크리스트 초안 작성
```

### Phase 2: 코드 최적화
```
1. 기존 코드 리뷰
2. 컴포넌트 분리/모듈화
3. 중복 코드 제거
4. 성능 최적화
```

### Phase 3: 검증 및 배포
```
1. 수동 화면 테스트
2. QA 체크리스트 검증
3. 버그 수정
4. 커밋 & 코드프리즈
5. 문서 최종 업데이트
6. Git Push
```
