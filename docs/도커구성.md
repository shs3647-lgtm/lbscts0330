## 🐳 Docker 구성 요소

### 1. Dockerfile (멀티스테이지 빌드)

**위치**: `./Dockerfile`

```dockerfile
# Stage 1: Dependencies - 종속성 설치
FROM node:20-alpine AS deps

# Stage 2: Builder - 애플리케이션 빌드
FROM node:20-alpine AS builder
- Prisma 클라이언트 생성
- Next.js 프로덕션 빌드

# Stage 3: Runner - 실행 환경
FROM node:20-alpine AS runner
- 최소한의 파일만 포함
- nextjs 사용자로 실행 (보안)
```

**핵심 특징**:
- ✅ **멀티스테이지 빌드**: 최종 이미지 크기 최소화 (~150MB)
- ✅ **Standalone 모드**: Next.js standalone 출력으로 node_modules 크기 감소
- ✅ **보안**: nextjs 사용자로 실행 (non-root)
- ✅ **메모리 최적화**: NODE_OPTIONS로 4GB 할당

### 2. docker-compose.yml

**위치**: `./docker-compose.yml`

#### 서비스 정의

##### a) PostgreSQL 데이터베이스
```yaml
postgres:
  image: postgres:15-alpine
  environment:
    POSTGRES_USER: ${DB_USER:-postgres}
    POSTGRES_PASSWORD: ${DB_PASSWORD:-postgres}
    POSTGRES_DB: ${DB_NAME:-fmea_db}
  volumes:
    - postgres_data:/var/lib/postgresql/data  # 데이터 영속성
    - ./docker/init-db:/docker-entrypoint-initdb.d  # 초기화 스크립트
  healthcheck:
    test: ["CMD-SHELL", "pg_isready -U postgres"]
```

**중요**:
- Docker Volume으로 데이터 영속성 보장
- healthcheck로 앱 시작 전 DB 준비 완료 확인

##### b) Next.js 애플리케이션
```yaml
app:
  build:
    context: .
    dockerfile: Dockerfile
  environment:
    DATABASE_URL: postgresql://postgres:password@postgres:5432/fmea_db
  depends_on:
    postgres:
      condition: service_healthy  # DB 준비 후 시작
```

**중요**:
- `depends_on`으로 PostgreSQL 준비 완료 후 시작
- DATABASE_URL에서 호스트명을 `postgres` (컨테이너 이름) 사용

### 3. .env 파일 구성

**배포용 .env 예제** (서버용):
```env
# 데이터베이스 설정
DB_USER=postgres
DB_PASSWORD=your_secure_password_here  # 보안 강화 필수
DB_NAME=fmea_db
DB_PORT=5432

# Prisma 데이터베이스 URL (Docker 내부 네트워크)
DATABASE_URL="postgresql://postgres:your_secure_password_here@postgres:5432/fmea_db?schema=public"

# 애플리케이션 포트
APP_PORT=3001  # 또는 원하는 포트

# Redis
REDIS_PASSWORD=your_redis_password

# JWT 시크릿 (운영 환경용으로 변경 필수)
JWT_SECRET=production_jwt_secret_change_this

# 세션 시크릿
SESSION_SECRET=production_session_secret_change_this

# 환경
NODE_ENV=production
```

⚠️ **주의사항**:
- **로컬 개발**: DATABASE_URL에 `localhost` 사용
- **Docker 배포**: DATABASE_URL에 `postgres` (컨테이너 이름) 사용
- 운영 환경에서는 반드시 강력한 비밀번호 설정

### 2. healthcheck.js 파일 생성 (필수)

**위치**: `./healthcheck.js`

```javascript
// healthcheck.js
const http = require('http');

const options = {
  host: 'localhost',
  port: 3000,
  path: '/api/health',
  timeout: 2000,
};

const request = http.request(options, (res) => {
  console.log(`STATUS: ${res.statusCode}`);
  if (res.statusCode === 200) {
    process.exit(0);
  } else {
    process.exit(1);
  }
});

request.on('error', (err) => {
  console.log('ERROR:', err.message);
  process.exit(1);
});

request.on('timeout', () => {
  console.log('TIMEOUT');
  request.destroy();
  process.exit(1);
});

request.end();
```

**Dockerfile 수정** (이미 포함되어 있음):
```dockerfile
# healthcheck.js 복사
COPY --from=builder /app/healthcheck.js ./healthcheck.js

# Healthcheck 설정
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node healthcheck.js
```

### 3. API Health Check 엔드포인트 생성

**위치**: `./src/app/api/health/route.ts`

```typescript
// src/app/api/health/route.ts
import { NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';

export async function GET() {
  try {
    // 데이터베이스 연결 확인
    await prisma.$queryRaw`SELECT 1`;

    return NextResponse.json({
      status: 'ok',
      timestamp: new Date().toISOString(),
      database: 'connected',
    });
  } catch (error) {
    return NextResponse.json(
      {
        status: 'error',
        timestamp: new Date().toISOString(),
        database: 'disconnected',
        error: error instanceof Error ? error.message : 'Unknown error',
      },
      { status: 503 }
    );
  }
}
```

### 4. Prisma 마이그레이션 파일 준비

**현재 문제**: `prisma/migrations` 폴더가 Docker 이미지에 포함되지 않음

**해결 방법**:

#### 옵션 A: 초기화 스크립트 사용 (권장)

**파일 생성**: `./docker/init-db/01-init.sql`

```sql
-- PostgreSQL 초기화 스크립트
-- 필요한 extension 설치
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 기본 설정
ALTER SYSTEM SET timezone = 'Asia/Seoul';
```

**Dockerfile 수정**:
```dockerfile
# Prisma migrations 폴더 복사
COPY --from=builder /app/prisma/migrations ./prisma/migrations

# 또는 entrypoint 스크립트에서 마이그레이션 실행
```

**entrypoint.sh 스크립트** (선택사항):
```bash
#!/bin/sh
set -e

echo "Waiting for PostgreSQL..."
while ! nc -z postgres 5432; do
  sleep 1
done

echo "Running database migrations..."
npx prisma migrate deploy

echo "Starting application..."
exec node server.js
```

#### 옵션 B: prisma db push 사용

Dockerfile에서:
```dockerfile
# 운영 환경에서는 권장하지 않음 (마이그레이션 히스토리 부재)
RUN npx prisma db push --skip-generate
```

```bash
# PostgreSQL 데이터베이스 백업

# Volume 백업
docker run --rm -v sdd_fmea_postgres_data:/data -v $(pwd):/backup \
  alpine tar czf /backup/postgres_data_backup.tar.gz -C /data .
```

```bash
# SQL 파일에서 복원
cat backup_20260130.sql | docker exec -i fmea-postgres psql -U postgres -d fmea_db

# Volume 복원
```

**증상**:
```

```

**해결**:
```bash

lsof -i :3000                 # Linux/Mac

# 2. docker-compose.yml에서 포트 변경
ports:
  - "3001:3000"  # 호스트:컨테이너

# 3. 서비스 재시작
docker-compose up -d
```

```bash
# 로컬: postgresql://user:pass@localhost:5432/db

# 5. 네트워크 확인
docker network inspect sdd_fmea_fmea-network
```

### 문제 5: 메모리 부족

**증상**:
```
FATAL ERROR: Reached heap limit Allocation failed
```

**해결**:
```bash

# docker-compose.yml
services:
  app:
    deploy:
      resources:
        limits:
          memory: 4G

# 3. 서버 메모리 확인
free -h  # Linux
```
