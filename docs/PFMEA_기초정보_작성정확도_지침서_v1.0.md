## 0. 전체 구조 — 항목 간 역할 분리 원칙

각 항목은 고유한 역할을 가진다. 항목 간 경계가 무너지면 고장사슬 매핑이 실패하고, Control Plan 연계가 불가능해진다.

| PFMEA 항목 | 역할 | 연결 대상 | 작성 핵심 |
|---|---|---|---|
| A3 공정기능 | 공정의 행위 + 결과 | A4 제품특성 | 공정이 무엇을 해서 어떤 결과를 만드는가 |
| A4 제품특성 | 제품에서 측정하는 특성 | A5 고장형태 | Wafer/반제품에서 직접 측정되는 결과값 |
| A5 고장형태 | A4 특성의 이탈 현상 | FC 고장사슬 | 결과가 어떻게 잘못되었는가 |
| B1 작업요소 | 설비·재료·인원 고유 명칭 | B2, FC(WE) | 공정 수행 주체의 명칭 |
| B2 요소기능 | B1이 수행하는 기능 | B3 공정특성 | 요소가 무엇을 해서 어떤 결과를 제공하는가 |
| B3 공정특성 | 설비·약품에서 관리하는 파라미터 | B4 고장원인 | 입력 조건, 설정값, 관리값 |
| B4 고장원인 | B3 이탈의 직접 원인 | B5 예방관리 | 왜 파라미터 이탈이 발생했는가 |
| B5 예방관리 | 발생 전 방지 | CP Control Method | 원인 발생 전에 막는 현재 관리 |
| A6 검출관리 | 발생 후 검출 | CP Measurement Technique | 발생 후 출하 전 발견하는 검사 |

### 핵심 구분 원칙

- **A4 제품특성**: 제품(Wafer)에서 측정하는 결과값
- **B3 공정특성**: 설비·약품에서 설정·모니터링하는 파라미터(입력값)
- **A5 고장형태**: A4가 이탈한 현상  
  → 원인 절대 포함 금지
- **B4 고장원인**: B3가 이탈한 이유  
  → 현상 절대 포함 금지
- **B5 예방관리**: 원인 발생 전 방지
- **A6 검출관리**: 발생 후 검출

### 판단 우선순위 6원칙

1. 제품에서 측정하는 결과값이면 **A4**
2. 설비·약품·조건의 입력값이면 **B3**
3. A4가 기준에서 벗어난 현상이면 **A5**
4. B3가 벗어나게 된 직접 이유면 **B4**
5. 원인 발생 전에 막는 현재 관리면 **B5**
6. 발생 후 잡아내는 검사면 **A6**

## 1. 공정기능 (A3) — 공정이 수행하는 행위와 결과

공정기능은 **“이 공정이 무엇을 해서 어떤 결과를 만드는가”**를 한 문장으로 기술한다.  
파라미터 수치, 검사 절차, 불량 방지 내용은 포함하지 않는다.

### 정의 및 형식

> **[대상물]을 [처리방법]으로 [처리]하여 [결과]를 확보한다**

- 공정의 기여(기능)만 기술
- 결과는 제품 상태·품질로 표현
- 동사형으로 마무리

### 올바른 예

- `Wafer 표면의 이물을 DIW로 세정하여 청정도를 확보한다`
- `PR을 균일하게 도포하여 목표 두께를 확보한다`
- `입고 Wafer의 표면 상태 및 Lot 정보를 검증한다`

### 금지 패턴

- `1. 서류심사 입고 Lot에 대해서 라벨 및 Run Sheet, 전산, 실물 일치화`  
  → 절차서 문장을 공정기능으로 사용한 경우
- `UBM Sputter 공정을 정상 진행한다`  
  → “정상 진행한다”는 기능 기술이 아님
- `불량 발생을 방지하기 위해 세정한다`  
  → 고장 조건 포함 금지

## 2. 제품특성 (A4) — 제품(Wafer)에서 측정하는 품질특성

A4는 공정 출력물인 Wafer·반제품에서 **직접 측정·확인 가능한 품질 특성의 명사형**이다.  
설비·약품의 파라미터(온도, 농도, 에너지 등)는 모두 **B3 공정특성**이다.

### 정의 및 형식

> **Wafer·반제품 표면·형상·물성에서 측정 가능한 품질 특성 명사**  
> 형식: **[특성 명사] ([영문 용어, 선택])**

- 측정 대상 = 제품
- 단위 포함 가능
- 명사형만 기재

### 올바른 예

- `Bump 높이 (Bump Height)`
- `PR 도포 두께 (PR Thickness)`
- `TiW 박막 두께`
- `Wafer 표면 외관 결함 부재 (Visual Defect Absence)`
- `Wafer 표면 청정도 (Particle 농도)`

### 금지 패턴 — 공정특성 혼입

- `DI Resistivity` → **B3로 이동**
- `RF Etch amount` → **B3로 이동**
- `Bake Temp` → **B3로 이동**
- `UV Energy` → **B3로 이동**

### 금지 패턴 — 행위·동사형

- `Wafer OCR Check` → `OCR ID 인식 정확도`로 수정
- `Particle Check` → `Wafer 표면 Particle 농도`로 수정
- `Wafer` → 측정 불가, 너무 추상적이므로 특성명으로 세분화

### 실제 오류 → 올바른 기재 방법

| 실제 오류 | 올바른 기재 방법 |
|---|---|
| `Wafer` | `Wafer 표면 외관 결함 부재 (Visual Defect Absence)` / `Wafer 표면 스크래치 부재` / `Wafer 파손·크랙 부재` |
| `Wafer OCR Check` | `OCR ID 인식 정확도 (OCR Matching Accuracy)` |
| `Particle Check` | `Wafer 표면 청정도 (Particle 농도, ea/wfr)` |

## 3. 고장형태 (A5) — A4 제품특성이 이탈한 현상

고장형태는 A4 제품특성이 요구 기준에서 이탈한 **결과적 현상**이다.  
“어떻게 되었는가(What)”만 기술하며, 원인(Why), 행위, 공정특성 이탈은 포함하지 않는다.

### 정의 및 형식

> **[A4 특성명] + [이탈 유형 어휘]**

이탈 유형 예:
- 규격 이탈
- 미달
- 초과
- 미형성
- 부재
- 잔류
- 불만족
- 미검출

### 올바른 예

- `Bump 높이 규격 이탈 (Bump Height Spec Out)`
- `PR 도포 두께 미달 (PR Thickness Under-spec)`
- `Au Bump 미형성 (Missing Bump)`
- `Wafer 표면 외관 결함 미검출 (Visual Defect Escape)`
- `Wafer Lot 혼입 (Lot Mix)`

### 금지 패턴 — 원인 혼입

- `작업표준 미준수` → **B4 고장원인**
- `설비 PM 미실시` → **B4 고장원인**
- `Robot Teaching 불량` → **B4 고장원인**
- `Recipe 오설정` → **B4 고장원인**

### 금지 패턴 — 공정특성 이탈 혼입

- `DIW 비저항 이탈` → **B4: DI Resistivity 규격 이탈**
- `Bake 온도 이탈` → **B4: Bake 온도 규격 이탈**
- `UV Energy 부족` → **B4: UV Energy 기준 미달**

### 금지 패턴 — 동사·문장형

- `Wafer Scratch`  
  → `Wafer 표면 스크래치 발생`처럼 이탈 현상 명확화
- `원자재 불량 유출`  
  → A5가 아닌 **C4 고장영향**  
  → A5는 `Wafer 표면 결함 미검출`

### 실제 오류 → 올바른 기재 방법

| 실제 오류 | 올바른 기재 방법 |
|---|---|
| `원자재 불량 유출` | `Wafer 표면 외관 결함 미검출 (Visual Defect Escape)` |
| `Wafer Chpping, Crack` | `Wafer 파손·크랙 발생 (Chipping/Crack)` |
| `Lot Mix(WAFER MIX)` | `Wafer Lot 혼입 (Lot Mix)` + A4 `Lot 식별 정확도`와 대응 |

### A5와 C4 구분 원칙

- **A5**: 제품특성의 이탈 현상 자체
- **C4**: 그 이탈로 인해 다음 공정/고객/사용자에게 나타나는 결과

예:
- **A5**: `Wafer 표면 외관 결함 미검출`
- **C4**: `불량 Wafer 유출로 후공정 품질 문제 발생`

## 4. 작업요소 (B1) — 공정에 직접 관여하는 설비·재료·인원 명칭

작업요소는 공정 수행에 직접 관여하는 **구체적 요소의 고유 명칭**이다.  
B2 요소기능, FC 작업요소(WE) 컬럼과 **완전히 동일한 표기**를 사용해야 시스템 매핑이 성립한다.

### 4M 작성 기준

| 4M | 작성 기준 | 예시 | 금지 예 |
|---|---|---|---|
| MC 설비·도구 | 설비 고유명 또는 모델명 사용 | `HIGH POWER SCOPE`, `DNS SS-3000`, `Stratus 300`, `토크렌치` | `세정 설비`, `도금 장비` |
| MN 사람 | 직무·역할명으로 특정 | `품질검사원`, `셋업엔지니어`, `보전원`, `Plating공정 Eng'r` | `작업자` |
| IM 재료 | 재료·약품·가스 고유명 사용 | `Au Etchant`, `TMAH 현상액`, `DIW`, `PR (TSMR-8900)` | `화학약품`, `재료` |

### 보완 판단 규칙

- 사람이 역할을 수행하면 **MN**
- 반복 사용 장비·치공구면 **MC**
- 소모성 약품·가스·자재면 **IM**
- 온도·습도·청정도 같은 외부 조건이면 **환경 항목으로 구분 검토**

### 실제 오류 → 수정

- `TEST 장비Technician`  
  → 설비가 아니라 사람(MN)  
  → `HIGH POWER SCOPE`는 MC, `TEST장비 Technician`은 MN으로 분리
- `Particle`  
  → 작업요소가 아님  
  → 측정항목(A4) 또는 장비(MC)로 정리 필요

### B1·B2·FC(WE) 3곳 일치 규칙

B1에 `HIGH POWER SCOPE`가 있으면,  
B2의 ★작업요소(B1) 컬럼과 FC의 작업요소(WE) 컬럼도 반드시 **`HIGH POWER SCOPE`**로 동일하게 기재해야 한다.

> 공백, 대소문자, 표기 차이가 있으면 시스템 매핑이 끊어진다.

## 5. 요소기능 (B2) — 작업요소가 공정에서 수행하는 고유 기능

요소기능은 B1 작업요소가 공정에서 수행하는 기능을 **동사문**으로 기술한다.  
수치, 고장 조건은 포함하지 않고, 주어(B1)와 결과를 명확히 써야 한다.

### 정의 및 형식

> **[B1 작업요소]이/가 [행위]하여 [결과]를 제공/확보/판정한다**

- 주어 = B1 명칭
- 결과 명시 필수

### 올바른 예 — MC

- `HIGH POWER SCOPE가 Wafer를 Auto Loading하여 외관 검사를 수행한다`
- `DNS SS-3000이 DIW를 분사하여 Wafer 표면 이물을 제거한다`
- `Stratus 300이 전류를 인가하여 Au 도금층을 형성한다`

### 올바른 예 — MN

- `품질검사원이 검사기준에 따라 수입검사를 진행하고 합부를 판정한다`
- `셋업엔지니어가 공정 파라미터를 설정하여 초기품을 승인한다`
- `보전원이 설비 점검 결과를 확인하고 예방보전 적합성을 판정한다`

### 금지 패턴

- 절차서 문장 나열
- 결과가 없는 불완전한 문장
- 작업요소가 아닌 항목을 주어로 사용

예:
- `[HIGH POWER SCOPE] Wafer를 High Scope에 Auto Loading하여 Chuck에 Wafer를 위치시킨후 외관검사 진행`
- `[품질검사원] 검사기준에 따라 수입검사 진행`
- `[Particle] 라인 Particle 모니터링을 통해 라인환경 유지 관리`

## 6. 공정특성 (B3) — 설비·약품에서 관리하는 파라미터 (입력값)

공정특성은 B1 작업요소가 공정 중 **설정·모니터링하는 입력 파라미터**이다.  
제품에서 측정하는 결과값은 A4이고, 설비·약품에서 관리하는 조건값이 B3이다.

### 정의 및 형식

> **B1 작업요소가 관리하는 공정 조건의 명칭**  
> 형식: **[파라미터 명] + [(단위) 또는 관리 기준/코드]**

- 측정 대상 = 설비/약품
- 단위 권장
- 단위가 없는 경우 관리 기준/코드로 표현 가능

### 올바른 예

- `TMAH 농도 (wt%)`
- `도금 전류 (A·hr)`
- `Bake 온도 (°C)`
- `DI Resistivity (MΩ·cm)`
- `UV Energy (mJ/cm²)`
- `RF Etch 량 (nm)`
- `Recipe No. (Rev.)`
- `Lens 배율 (×)`
- `AQL 등급 (Level)`

### A4 vs B3 구분

- **A4** = 제품(Wafer)에서 측정하는 결과값
- **B3** = 설비·약품에서 설정·모니터링하는 조건값

예:
- Wafer에서 측정한 `TiW 두께` = **A4**
- 스퍼터 장비의 `파워`, `시간 조건` = **B3**

### 금지 패턴

- B2 요소기능 내용이 B3에 들어가는 경우
- A4 제품특성이 B3에 들어가는 경우
- 측정 불가한 행위 표현

예:
- `[품질검사원] 검사기준에 따라 수입검사 진행`
- `[HIGH POWER SCOPE] Visual Defect`
- `유지보수 정확도`, `취급 조건`

### 실제 오류 → 올바른 기재 방법

| 실제 오류 | 올바른 기재 방법 |
|---|---|
| `[품질검사원] 검사기준에 따라 수입검사 진행` | `검사 합격 기준 (AQL 등급)` 또는 파라미터 없으면 공란 |
| `[HIGH POWER SCOPE] Visual Defect` | A4 `Wafer 표면 외관 결함 부재` / B3 `고배율 렌즈 배율 설정 (×배수)` |
| `[TEST 장비Technician] 주기에 맞춰 PM 진행 및 장비고장시 BM 처리` | `PM 주기 (월)` 또는 파라미터 없으면 공란 |

## 7. 고장원인 (B4) — B3 공정특성이 이탈한 직접적 원인

고장원인은 B3 공정특성 파라미터가 기준에서 이탈하게 된 **직접적 원인**이다.  
고장형태(A5)나 대책·행동을 기술하지 않는다.

### 정의 및 형식

> **[B3 파라미터] [이탈 유형] + 원인 메커니즘**  
> 또는  
> **[B1 요소]의 [물리적 변화]에 의한 [파라미터 이탈]**

- 파라미터 이탈 원인
- 명사형 기술 권장
- 행동 기술 금지

### 올바른 예 — MC

- `Robot Teaching 위치 이탈에 의한 반송 오류`
- `Wafer 카세트 Loading 위치 편차`
- `설비 PM 미실시에 의한 Pump 성능 저하`
- `Lamp Life Time 초과에 의한 UV 조사량 저하`

### 올바른 예 — MN

- `Recipe 오적용 (제품별 조건 설정 혼동)`
- `공정 누락 (PR Coating Skip)`
- `작업표준 미숙지에 의한 처리 조건 이탈`
- `Run Sheet/Wafer 라벨 매칭 실수에 의한 Lot 혼입`

### 금지 패턴 — 고장형태 혼입

- `Wafer Scratch`
- `Bump 미형성`
- `Lot Mix`

→ 위 항목은 모두 **A5 고장형태**

### 금지 패턴 — 행동 기술

- `[품질검사원] 실물 Wafer와 Run Sheet/ Wafer 카세트 라벨 Mix`
- `작업자 작업 절차 평가 진행`
- `분기 1회 PM 진행`

→ 마지막 두 항목은 **B5 예방관리**

### 실제 오류 → 올바른 기재 방법

| 실제 오류 | 올바른 기재 방법 |
|---|---|
| `[품질검사원] 실물 Wafer와 Run Sheet/ Wafer 카세트 라벨 Mix` | `Run Sheet와 Wafer 카세트 라벨 육안 매칭 실수에 의한 Lot 혼입` |
| `[품질검사원] 제품 구분 보관 Mix` | `제품 구분 보관 기준 미준수에 의한 Lot 위치 혼동` |
| `[HIGH POWER SCOPE] Robeot Teaching 상태 불량` | `Robot Teaching 위치 이탈에 의한 Wafer 반송 오류` |

## 8. 예방관리 (B5) / 검출관리 (A6) — 발생 전 방지 vs 발생 후 검출

예방관리(B5)는 고장원인 발생을 **사전에 방지**하는 시스템·장치·절차이다.  
검출관리(A6)는 고장이 발생한 후 **출하 전에 발견**하는 검사·측정 방법이다.

### B5 예방관리 (Prevention)

#### 정의
B4 고장원인 발생을 사전에 방지하기 위한 현재 관리 방법

#### 형식
> **[구체적 장치명·시스템명·절차명] + [작동 방식]**

#### 올바른 예

- `Robot Teaching 주기적 점검 (분기 1회)`
- `Run Sheet 라벨·Wafer 캐리어 라벨 바코드 매칭 시스템`
- `공정 Recipe 이중 확인 (MES 전산 인터록)`
- `작업자 인증 시험 (정기 숙련도 평가)`

### A6 검출관리 (Detection)

#### 정의
고장 발생 후 출하 전 검출을 위한 검사·측정 방법

#### 형식
> **[검출 장비명] + [검사/측정 방법] + [검출 대상] + [빈도/방식]**

#### 올바른 예

- `Machine Interlock (자동 검출, 실시간)`
- `MES Interlock (전산 자동 차단, 실시간)`
- `HIGH POWER SCOPE 전수 외관검사 (Wafer 표면 결함, Every lot)`
- `Micro Scope 육안검사 (AQL 1.5 G-Ⅰ, Every lot)`
- `4Point Probe 측정 (TiW Thickness, 13 points/wfr, Daily)`

### 혼재 오류 구분 기준

- **B5** = 원인이 발생하기 전에 막는 것
- **A6** = 이미 발생한 것을 잡는 것

#### 잘못된 예

- A6에 `작업자 교육`, `SOP 게시` 기재 → **B5로 이동**
- A6에 `예방보전 계획 수립` 기재 → **B5로 이동**
- B5에 `캘리퍼스로 측정` 기재 → **A6로 이동**

### 실제 오류 보완

- `작업자 작업 절차 평가 진행`
  → `작업자 인증 시험 및 정기 숙련도 평가`
- `분기 점검 시 Robot Teaching 상태 check`
  → `Robot Teaching 상태 분기 정기 점검`
- `Wafer 검사`
  → `HIGH POWER SCOPE 전수 외관검사 (Every lot)`
- `Visual Inspection`
  → `Micro Scope 육안검사 (AQL 1.5 G-Ⅰ, Every lot)`

## 9. Control Plan 연계 — PFMEA 항목이 CP에 반영되는 방법

PFMEA 기초정보를 정확하게 작성하면, Control Plan의 CHARACTERISTICS, 측정기법, 관리방법, 이상반응계획이 PFMEA에서 직접 도출된다.  
중복 작성 없이 PFMEA-CP 양방향 추적이 가능해진다.

### PFMEA → CP 연동 구조

| PFMEA 기초정보 | Control Plan 연동 | 효과 |
|---|---|---|
| A4 제품특성 (CC/SC) | CP Product 특성 컬럼 | PFMEA 특성 자동 반영 |
| B3 공정특성 (CC/SC) | CP Process 특성 컬럼 | 파라미터 중복 작성 제거 |
| A6 검출관리 | CP 측정기법 컬럼 | 검사 장비/방법 누락 방지 |
| B5 예방관리 | CP 관리방법 컬럼 | 관리 시스템 직접 반영 |
| FC AP (H/M/L) | CP 이상 반응계획 | 반응 강도 일관성 확보 |

### PFMEA 항목별 CP 컬럼 연계 예시

| PFMEA 항목 | CP 컬럼명 | AU Bump 적용 예시 | 측정·관리 방법 | 연계 원칙 |
|---|---|---|---|---|
| A4 제품특성 (CC/SC) | CHARACTERISTICS Product | Visual Defect / TiW Thickness (◇) / Au Thickness (◇) / Bump Height / Bump Co-planarity | Micro Scope / 4Point Probe / GTX8 | A4의 CC/SC 항목을 CP Product 컬럼으로 그대로 이관 |
| B3 공정특성 (CC/SC) | CHARACTERISTICS Process | DI Resistivity / RF Etch amount (◇) / Bake Temp / TMAH Concentration / UV Energy | Machine Interlock / Nanospec / COA / Intensity Meter | B3의 CC/SC 항목을 CP Process 컬럼으로 그대로 이관 |
| A6 검출관리 | EVALUATION / MEASUREMENT TECHNIQUE | Microscope / Naked eye / Machine interlock / 4Point Probe / AVI | 직접 반영 | A6 장비명 + 방법을 CP 측정기법 컬럼으로 직접 연동 |
| B5 예방관리 | Control Method | Run Sheet 라벨 매칭 / SPC Chart / Trend Chart / MES Interlock / PM Sheet | 직접 반영 | B5 관리 시스템을 CP Control Method로 직접 연동 |
| FC AP | REACTION PLAN | AP=H: Lot Hold / AP=M: 공정 중단 검토 / AP=L: 이상 보고 후 모니터링 강화 | 직접 반영 | AP 등급이 CP 이상 반응 강도 결정 |

### CP 작성 원칙

1. **A4의 CC·SC 항목 → CP Product 컬럼 그대로 이관**
2. **B3의 CC·SC 항목 → CP Process 컬럼 그대로 이관**
3. **A6 장비명 + 방법 → CP 측정기법 컬럼 그대로 이관**
4. **B5 관리시스템 → CP 관리방법 컬럼 그대로 이관**

이 원칙을 따르면 PFMEA-CP 간 항목 중복 작성과 누락을 동시에 방지할 수 있다.

## 11. 한 줄 기준 요약

> **A4는 제품 결과값, B3는 공정 입력값, A5는 결과값의 이탈 현상, B4는 입력값 이탈의 직접 원인, B5는 사전 예방, A6는 사후 검출이다.**

이 원칙을 유지하면 PFMEA 기초정보의 작성 정확도를 높일 수 있고, FC 및 Control Plan과의 양방향 추적성을 안정적으로 확보할 수 있다.
