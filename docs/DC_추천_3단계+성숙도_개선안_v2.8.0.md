## 2. AIAG-VDA 검출도(D) 성숙도 기준표

| D | 레벨 | 검출방법 성숙도 | 검출기회 | DC 사례 |
|---|------|---------------|---------|---------|
| **10** | 매우 낮음 | 시험/검사 방법이 수립되지 않음 | 검출 불가 | (없음) |
| **9** | 매우 낮음 | 검출 가능성이 낮음 | 무작위 심사로 검출 어려움 | 비정기 감사 |
| **8** | 낮음 | 방법이 효과적으로 **입증되지 않음** (경험 부족, Gage R&R 한계적) | **사람의 검사**(시각/촉각/청각) 또는 **수동 게이지** | 육안검사(미입증), 수동 게이지(미입증) |
| **7** | 낮음 | 방법이 효과적으로 **입증되지 않음** | **기계기반 검출**(자동/반자동, 알림) 또는 **CMM 등 검사장비** | 바코드 스캐너(미입증), CMM(미입증) |
| **6** | 중간 | 방법이 효과적·신뢰할 수 있는 것으로 **입증됨** (경험 있음, Gage R&R 수용) | **사람의 검사** 또는 **수동 게이지** (제품 샘플 포함) | **육안검사(입증됨)**, 수동 게이지(입증됨) |
| **5** | 중간 | 방법이 효과적·신뢰할 수 있는 것으로 **입증됨** | **기계기반 검출**(반자동) 또는 **CMM 등** (샘플 포함) | EOL검사기(미입증), 비전검사(미입증), CMM(입증됨) |
| **4** | 높음 | 시스템이 효과적·신뢰할 수 있는 것으로 **입증됨** | **하류 부문** 기계기반 자동 검출 + 유출 방지 | **바코드 스캐너(입증됨)**, PDA 스캔(입증됨) |
| **3** | 높음 | 시스템이 효과적·신뢰할 수 있는 것으로 **입증됨** | **스테이션 내** 기계기반 자동 검출 + 유출 방지 | **EOL검사기(입증됨)**, 핀휨검사기, 토크검사, 비전검사 |
| **2** | 높음 | 검출 방법이 입증됨 (실수방지 검증 등) | **원인 검출** + 고장 형태 생산 방지 | **검교정**, MSA 검증, Gage R&R, Poka-Yoke |
| **1** | 매우 높음 | 고장이 물리적으로 생산될 수 없거나, 항상 검출 입증 | 물리적 방지 / 100% 검출 | 설계적 방지, 완전자동 100% |

### 2.1 핵심 판별 기준

```
D = 6~8: "사람의 검사" (육안, 수동 게이지)
  → 입증됨 = D=6, 미입증 = D=8

D = 5~7: "기계기반 + 미입증" 또는 "검사장비 + 입증"
  → CMM 입증 = D=5, 바코드 미입증 = D=7

D = 3~4: "기계기반 자동 + 입증"
  → 인스테이션 = D=3, 하류 = D=4

D = 1~2: "원인 검출/물리적 방지"
  → 실수방지 = D=2, 물리적 불가 = D=1
```

### 3.1 JG1 HUD 문제 발견

```
현재 상태: 589건 중 588건이 D=1

★ 문제: 육안검사(562건)가 D=1로 평가됨
  AIAG 기준: 육안검사 → D=6(입증됨) ~ D=8(미입증)
  D=1 = "물리적으로 생산 불가 또는 항상 100% 검출"
  → 육안검사는 D=1이 될 수 없음 (심각한 과소평가)

영향: D값이 과소평가되면 → AP(Action Priority)도 과소평가
  → 실제로는 High인 리스크가 Low로 분류되어 개선 조치 누락
```

## 5. 추천2: PC→DC 연계 + 성숙도 기반 개선

### 5.1 원리

추천1에 PC 연계 DC를 합집합으로 추가하고,
**성숙도 등급으로 D값 적정성 평가** + **개선 방향 제시**.

```
추천2 = 추천1 ∪ { 같은 PC의 다른 FC DC }

성숙도 평가:
  현재DC의 적정D vs 추천2 DC의 적정D 비교
  → 추천2에 더 좋은(낮은) D의 DC가 있으면 "↓개선" 표시
  → 현재D < 적정D이면 "⚠ 과소평가" 경고
```

### 5.2 통계

| 항목 | 값 |
|------|-----|
| 총 PC | 80개 |
| DC 2+ PC | 8개 |
| 추천2 DC 추가 행 | 129건 (22%) |

### 5.3 성숙도 개선 사례

**■ 사례1: "MC_마스터 유효기간 관리" PC → D값 개선**

| 공정 | 현재DC | 현재D | 적정D | 추천2 DC | 추천2 적정D | 효과 |
|------|--------|-------|-------|---------|-----------|------|
| 20번 | 버전 검사기 | D=1 | D≥3 | + EOL, 핀휨, 육안 | D≥3 | D=1→3 ⚠과소 |

| 120번 | EOL 검사기 | D=1 | D≥3 | + 버전, 핀휨, 육안 | D≥3 | D=1→3 ⚠과소 |

**■ 사례3: 성숙도 기반 D값 과소평가 경고**

```

  ⚠⚠⚠ 육안검사 562건: 현재 D=1, 적정 D=6~8 → 과소평가 5~7등급
  ⚠    바코드 스캐너 7건: 현재 D=1, 적정 D=4~7 → 과소평가 3~6등급

  ⚠    검교정 3건: 현재 D=1, 적정 D=2~4 → 과소평가 1~3등급

  → 전체 589건 중 588건이 D값 과소평가!
```

### 5.4 추천2 알고리즘 (성숙도 통합)

```python
DC_MATURITY = {
    '육안검사': {'proven': 6, 'unproven': 8},
    '바코드 스캐너': {'proven': 4, 'unproven': 7},
    'EOL 검사기': {'proven': 3, 'unproven': 5},
    '검교정': {'proven': 2, 'unproven': 4},
    # ... (전체 DC별 매핑)
}

def recommend_dc_2_with_maturity(fm_value, pc_value, current_dc, current_d):
    # 1. DC 후보 수집
    dc_set = set(fm_dc_map.get(fm_value, {'육안검사'}))
    if pc_value and pc_value in pc_dc_map:
        dc_set |= pc_dc_map[pc_value]
    
    # 2. DC별 성숙도 부여
    candidates = []
    for dc in dc_set:
        mat = DC_MATURITY.get(dc, {'proven': 8, 'unproven': 10})
        candidates.append({
            'dc': dc,
            'd_proven': mat['proven'],
            'd_unproven': mat['unproven'],
        })
    
    # 3. 현재 D값 적정성 평가
    cur_mat = DC_MATURITY.get(current_dc, {'proven': 8})
    d_gap = current_d - cur_mat['proven']  # 음수 = 과소평가
    
    # 4. 최적 DC (가장 낮은 D 가능)
    best = min(candidates, key=lambda x: x['d_proven'])
    
    return {
        'candidates': sorted(candidates, key=lambda x: x['d_proven']),
        'd_gap': d_gap,           # 과소평가 경고
        'd_warning': d_gap < 0,   # True면 과소평가
        'best_dc': best['dc'],
        'best_d': best['d_proven'],
        'improvement': cur_mat['proven'] - best['d_proven'],  # 양수면 개선
    }
```
