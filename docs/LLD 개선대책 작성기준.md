## 1. 개요

LLD(Lessons Learned Database, 습득교훈 데이터베이스)의 개선대책은 PFMEA 워크시트의 **위험분석·최적화 단계(5~6단계)**에서 활용된다. 본 문서는 개선대책 작성 시 준수해야 할 기준과 절차를 정의한다.

### 1.1 개선대책의 목적

| 목적 | 설명 |
|------|------|
| **재발 방지** | 과거 품질 이슈(RMA/ABN/CIP 등)의 근본 원인 대책을 표준화 |
| **수평전개** | 유사 공정/제품에 동일 대책을 선제 적용 |
| **AP 등급 개선** | H(High) → M(Medium) → L(Low)로 위험도 저감 |
| **O/D 목표값 달성** | 발생도(O)·검출도(D)를 정량적 목표까지 개선 |

## 2. AP(Action Priority) 등급 기준

### 2.1 AP 판정 테이블

AP는 심각도(S), 발생도(O), 검출도(D)의 조합으로 결정된다.

| S 범위 | O 범위 | D=7~10 | D=5~6 | D=2~4 | D=1 |
|--------|--------|--------|-------|-------|-----|
| **9~10** | 8~10 | **H** | **H** | **H** | **H** |
| **9~10** | 6~7 | **H** | **H** | **H** | **H** |
| **9~10** | 4~5 | **H** | **H** | **H** | M |
| **9~10** | 2~3 | **H** | **H** | M | L |
| **9~10** | 1 | **H** | M | L | L |
| **7~8** | 8~10 | **H** | **H** | **H** | M |
| **7~8** | 6~7 | **H** | **H** | M | M |
| **7~8** | 4~5 | **H** | M | M | L |
| **7~8** | 2~3 | M | M | L | L |
| **7~8** | 1 | M | L | L | L |
| **4~6** | 8~10 | **H** | M | M | L |
| **4~6** | 6~7 | M | M | M | L |
| **4~6** | 4~5 | M | M | L | L |
| **4~6** | 2~3 | M | L | L | L |
| **4~6** | 1 | L | L | L | L |
| **2~3** | 8~10 | M | M | L | L |
| **2~3** | 6~7 | M | L | L | L |
| **2~3** | 4~5 | L | L | L | L |
| **2~3** | 2~3 | L | L | L | L |
| **2~3** | 1 | L | L | L | L |
| **1** | (전체) | L | L | L | L |

- **H (High, 빨강)**: 반드시 개선대책 수립 필수
- **M (Medium, 주황)**: 개선대책 수립 권장 (경영층 판단)
- **L (Low, 초록)**: 현 수준 유지 가능

### 2.2 개선 우선순위

```
1순위: AP = H (즉시 대책 수립)
2순위: AP = M, S ≥ 9 (안전/규제 관련)
3순위: AP = M, O ≥ 8 (높은 발생빈도)
4순위: AP = M (기타)
```

## 3. 심각도(S) 평가 기준

> S값은 고장영향(FE)의 심각성을 1~10 등급으로 평가한다.
> **S값은 설계 변경 없이 저감 불가** — 개선대책의 대상이 아니다.

| S | 등급명 | 자사공장(YP) 관점 영향 | 핵심 키워드 |
|---|--------|----------------------|------------|
| 10 | 매우 심각 (안전) | 근로자 건강/안전 리스크 | 사망, 중상, 화재, 폭발, 감전, 질식 |
| 9 | 매우 심각 (규제) | 공장 내 규제 미준수 | 규제, 법규, 인증, 환경규제, 유해물질 |
| 8 | 심각 | 100% 폐기, 1shift+ 라인 중단 | 폐기, 라인중단, 출하중단, 전수불량 |
| 7 | 높음 | 선별/일부 폐기, 기준 이탈 | 선별, 기준이탈, 라인속도저하, CPK미달 |
| 6 | 보통 (높음) | 100% 라인 밖 재작업 | 라인밖재작업, 전수재작업, 외주수리 |
| 5 | 보통 | 일부 라인 밖 재작업 | 일부재작업, 부분수리, 교육훈련부족 |
| 4 | 보통 (낮음) | 100% 스테이션 내 재작업 | 스테이션재작업, 조정, 미세조정 |
| 3 | 경미 | 일부 스테이션 내 재작업 | 일부조정, 소폭재작업, 경미한결함 |
| 2 | 매우 경미 | 작업자 약간의 불편 | 불편, 외관경미, 소음미세 |
| 1 | 없음 | 식별 가능한 영향 없음 | 영향없음, 무해, 감지불가 |

## 4. 발생도(O) 평가 및 개선 기준

> O값은 고장원인(FC)의 발생 빈도를 1~10 등급으로 평가한다.
> **예방관리(PC) 강화로 O값을 저감**하는 것이 개선대책의 핵심이다.

### 4.1 발생도 등급표

| O | 등급명 | 예방관리 수준 | 발생 빈도 |
|---|--------|-------------|----------|
| 10 | 매우 높음 | 예방관리 없음 | ≥100/1,000건 (≥10%) |
| 9 | 높음 | 예방관리 미흡 | 50/1,000건 (5%) |
| 8 | 높음 | 비효과적 예방관리 | 20/1,000건 (2%) |
| 7 | 보통 (높음) | 미완성 예방관리 | 10/1,000건 (1%) |
| 6 | 보통 | 불완전 예방관리 | 2/1,000건 (0.2%) |
| 5 | 보통 (낮음) | 제한적 예방관리 | 0.5/1,000건 (0.05%) |
| 4 | 낮음 | 부분적 예방관리 | 0.1/1,000건 (0.01%) |
| 3 | 낮음 | 효과적 예방관리 | 0.01/1,000건 |
| 2 | 매우 낮음 | 매우 효과적 예방관리 | ≤0.001/1,000건 |
| 1 | 극히 낮음 | 예방관리 완벽 | 발생 사실상 불가 |

### 4.2 O값 개선 전략 (예방관리 강화)

| 현재 O | 목표 O | 필요한 예방관리 수준 | 대표 개선대책 |
|--------|--------|-------------------|-------------|
| 8~10 | 5 이하 | 공정 파라미터 관리 + Poka-Yoke | SPC 도입, 자동 인터락, 작업표준 강화 |
| 6~7 | 4 이하 | 주기적 점검 + 예방보전 | PM 계획 수립, 4M 변경관리, 교육 강화 |
| 4~5 | 2~3 | 자동화 예방 + 통계적 관리 | 자동화 설비 도입, X-R Chart 적용 |
| 2~3 | 유지 | 현 수준 유지 | 지속적 모니터링 |

### 4.3 예방관리(PC) 3단계 매칭 체계

시스템이 자동 추천하는 예방관리는 다음 우선순위를 따른다:

```
1순위: Import FC↔PC 직접 매칭 (사용자 정의 데이터)
       → 고장원인(FC)에 대해 Import 마스터의 failureChains.pcValue 직접 매칭
       → 가장 정확한 결과

2순위: B5 항목 공정번호 + 4M 기반 키워드 랭킹
       → 해당 공정의 예방관리 항목 중 4M(Man/Machine/Method/Material) 매칭
       → 공정 특화 추천

3순위: 산업DB + 전체풀 폴백
       → 한국산업DB(kr-industry) 표준 예방관리 활용
       → 마스터 데이터 미비 시 최종 폴백
```

## 5. 검출도(D) 평가 및 개선 기준

> D값은 검출관리(DC)의 검출 능력을 1~10 등급으로 평가한다.
> **검출관리(DC) 강화로 D값을 저감**하는 것이 개선대책의 또 다른 축이다.

### 5.1 검출도 등급표 (AIAG-VDA 보수적 기준 v3.0)

| D | 등급명 | 검출관리 수준 | 대표 장비/방법 |
|---|--------|-------------|--------------|
| 1 | 거의 확실 | 자동 인터락/Poka-Yoke | 기계 자동 차단 (사람 개입 없음) |
| 2 | 매우 높음 | 기계 검출 + 사람 확인 | 반자동 검사 (자동검출 + 작업자 확인) |
| 3 | 높음 | 자동측정/SPC/바코드 | 자동측정기, SPC, 바코드스캐너 |
| 4 | 보통 (높음) | 전용 검사기/게이지 | 전수 또는 샘플링 검사 (전용 장비) |
| 5 | 보통 | 검사기 + 주기적 점검 | 검교정/MSA 보증된 계측기 |
| 6 | 낮음 | 전수 육안검사 | 숙련 작업자 육안/촉감 전수검사 |
| 7 | 매우 낮음 | 일반 육안검사 | 비전검사(MSA 미입증), 일반 육안 |
| 8 | 거의 불가 | 간접적 확인 | 간접 모니터링, 샘플 육안검사 |
| 9 | 불가 | 검출관리 미흡 | 검출 방법 부재 |
| 10 | 검출 불가 | 검출관리 없음 | 해당 고장을 검출할 수단 전무 |

### 5.2 특수 규칙 (필수 준수)

| 검출 방법 | D값 | 비고 |
|----------|-----|------|
| 비전검사 (단독) | **D=7** | MSA 미입증 기계검사 → 보수적 평가 |
| 비전검사 + 유출방지 복합 | **D=3** | 이중 검출 효과로 승격 |
| 검교정/MSA 보증 계측기 | **D=5** | 간접 보증 수단 |
| 육안검사 (사람 의존) | **D=6~7** | D=1~5 절대 불가 (사람 의존 = 검출 한계) |
| Poka-Yoke (자동 차단) | **D=1** | 물리적 검출 차단 (최고 등급) |

### 5.3 검출관리(DC) 개선대책 작성 핵심 원칙 ★ NEW

> **DC 개선대책 = 고장을 직접 찾아내는 검사장치·검사방법 도입/강화**
> "검교정 주기 단축", "알림 시스템 도입" 같은 **관리 프로세스 개선은 예방관리(PC)** 에 해당한다.

#### DC와 PC의 명확한 구분 기준

| 구분 | 정의 | 핵심 질문 | 작성 예시 |
|------|------|----------|---------|
| **예방관리(PC)** | 고장원인(FC)이 **발생하지 않도록** 예방 | "고장이 생기지 않게 막는 것인가?" | SPC, 검교정 주기 단축, 교육 강화, 예방보전 |
| **검출관리(DC)** | 발생한 고장/불량을 **직접 찾아내는** 검사 | "불량품을 잡아내는 검사 수단인가?" | 버니어캘리퍼스, 외관검사, 비전검사, AOI, 토크센서 |

#### DC 작성 시 반드시 포함할 4가지 요소

```
① 검사 장치/도구명    : 버니어캘리퍼스, 마이크로미터, 비전검사, AOI, 토크센서 등
② 검사 대상 특성     : 치수, 외관, 토크값, 경도, 도막두께, 색상 등
③ 검사 빈도/범위     : 전수검사 / Lot당 n점 / AQL Level II 샘플링 등
④ 불합격 처리 방법   : 자동 NG 배출 / 후공정 이동 불가 인터락 / 즉시 격리 등
```

#### DC 작성 금지 패턴 (예방관리와 혼동되는 표현)

| ❌ 잘못된 DC 작성 (PC 성격) | ✅ 올바른 DC 작성 (실제 검출 수단) |
|--------------------------|--------------------------------|
| "검교정 이력 전산화 및 만료 7일 전 자동알림 시스템 도입" | "마이크로미터(MSA 검증) Lot별 전수 치수검사 → 기준 초과 시 NG 격리" |
| "Gage R&R 정기 실시 및 재교정 프로세스 수립" | "인라인 자동 토크센서(±3% 정밀도) 전수검사 → 미달 시 다음공정 이동 불가 인터락" |
| "DOE 기반 양산 조건 최적화" | "로크웰 경도계(HRC) Lot당 5점 경도검사 → 기준 미달 시 즉시 공정 정지" |
| "나사산 게이지 검사 빈도 증가(빈도 관리)" | "나사산 게이지(전용 GO/NO-GO 게이지) 전수 검사 → NG 시 자동 배출" |
| "검사 주기를 단축하여 관리 강화" | "자동 비전검사 시스템(고해상도 카메라) 전수 외관검사 → 기준 편차 자동 NG" |

### 5.4 공정별 DC 검사장치 가이드 (표준 추천)

> DC 개선대책 작성 시 아래 표를 참고하여 고장형태에 맞는 **실제 검사장치**를 선택한다.

| 고장형태 유형 | 권장 검사장치/방법 | 검출 D값 | 검사 방식 |
|-------------|-----------------|---------|---------|
| **치수 불량** (길이, 외경, 두께) | 버니어캘리퍼스, 마이크로미터, 에어게이지, 인라인 레이저 센서 | D=3~5 | 전수/샘플링 |
| **외관 불량** (크랙, 긁힘, 이물) | 비전검사 시스템(카메라+조명), AOI, 전수 육안검사 | D=4~7 | 전수 |
| **재질/성분 불량** (이종재질) | XRF 합금분석기, 성분검사기 | D=3~4 | 전수/Lot |
| **강도/경도 불량** | 로크웰/비커스 경도계, UTM 인장시험기 | D=4~5 | Lot별 샘플 |
| **체결 토크 부족** | 인라인 자동 토크센서, 토크렌치(검교정) | D=3~5 | 전수 |
| **도막 두께 부족** | 마그네틱/와전류 도막두께계 | D=3~4 | 전수/샘플링 |
| **색상 편차** | 색차계 (L*a*b* 정량 판정) | D=4 | 전수/샘플링 |
| **용접 불량** (크랙, 기공) | 초음파탐상(UT), X-ray 비파괴검사, 비전검사 | D=3~4 | 전수/샘플링 |
| **무게/중량 불량** | 자동 중량선별기, 전자저울(검교정) | D=3~5 | 전수 |
| **흔들림/진동** (휠 런아웃) | 런아웃 자동측정기 (다이얼게이지+회전지그) | D=4 | 전수 |
| **제동력 부족** | 제동력 시험기 (롤러식/플랫폼식) | D=3~4 | 전수 |
| **기능 이상** (전기/전자) | 기능시험기, ICT(In-Circuit Test), FCT | D=3~4 | 전수 |

### 5.5 D값 개선 전략 (검출관리 강화)

| 현재 D | 목표 D | 필요한 검출관리 수준 | 대표 개선대책 |
|--------|--------|-------------------|-------------|
| 8~10 | 5 이하 | 전용 검사장비 도입 | 자동 검사기, 센서, 게이지 도입 |
| 6~7 | 3~4 | 자동화 검출 전환 | 비전검사 → 자동측정기, SPC 적용 |
| 4~5 | 2~3 | 인터락/Poka-Yoke | 자동 인터락 설치, 실시간 SPC |
| 2~3 | 유지 | 현 수준 유지 | 검교정 주기 관리 |

### 5.6 검출관리(DC) 3단계 매칭 체계

```
1순위: A6(관리계획서) Import 마스터 → 공정번호 기반 직접 매칭
       → 가장 정확한 결과

2순위: 산업DB (KrIndustryDetection)
       → methodType='자동' 조건 필터 (수동검사 제외)
       → 자동검출 방법만 추천

3순위: FM 키워드 기반 DC 자동 생성 (폴백)
       → 고장형태(FM) 키워드 → detectionKeywordMap 규칙 적용
       → 기본 폴백: 비전검사 (D=7, 가장 범용적)
```

> ⚠️ **3순위 폴백 주의**: FM 키워드 기반 자동생성 결과는 반드시 사람이 검토 후 적용
> 폴백으로 생성된 "검교정 이력 전산화", "MSA 정기 실시" 등의 내용은 **PC 성격**이므로
> DC 필드에 그대로 등록하면 안 되며, 5.4항 표를 참고하여 실제 검사장치로 교체해야 한다.

## 6. 개선대책 작성 절차

### 6.1 개선대책 수립 프로세스

```
┌─────────────────────────────────────────────────┐
│ Step 1: AP 등급 확인                              │
│   → AP=H: 반드시 개선대책 수립                     │
│   → AP=M: 권장 (경영층 판단)                       │
│   → AP=L: 현 수준 유지                            │
└──────────────┬──────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────┐
│ Step 2: 개선 경로 선택 (3가지)                     │
│   ① O만 개선 (예방 집중)                          │
│   ② D만 개선 (검출 집중)                          │
│   ③ O+D 동시 개선 (균형)                          │
│   → 최소 총변화량 경로 권장                        │
└──────────────┬──────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────┐
│ Step 3: 목표값 설정                               │
│   → 목표 O값 (예방 강화 후)                       │
│   → 목표 D값 (검출 강화 후)                       │
│   → 개선 후 AP = L 이 되는 조합 선택               │
└──────────────┬──────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────┐
│ Step 4: 구체적 대책 기술                           │
│   → 예방관리(PC): 고장원인 발생 억제 수단           │
│   → 검출관리(DC): 실제 불량을 잡아내는 검사장치/방법 │  ← ★ DC는 반드시 검사장치 명시
│   → 담당자 / 완료기한 / 상태                      │
└──────────────┬──────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────┐
│ Step 5: LLD 등록                                  │
│   → 구분: RMA/ABN/CIP/ECN/FieldIssue/DevIssue    │
│   → 적용: 예방관리 / 검출관리                      │
│   → 상태: R(미완료) → Y(진행중) → G(완료)          │
└─────────────────────────────────────────────────┘
```

### 6.2 개선 경로별 특성

| 경로 | 장점 | 단점 | 권장 상황 |
|------|------|------|----------|
| **① O만 개선** | 근본 원인 제거, 장기 효과 | 비용/시간 큼, 설비 투자 필요 | 반복 발생 이슈, 설비 노후 |
| **② D만 개선** | 빠른 적용, 비용 적음 | 원인 미해결, 검사 의존 | 긴급 대응, 단기 개선 |
| **③ O+D 동시** | 균형 잡힌 개선, 총변화 최소 | 2개 활동 동시 관리 | 대부분의 경우 권장 |

### 6.3 개선대책 작성 예시

#### 예시 1: AP=H, S=8, O=7, D=8 → 개선 후 O=4, D=3

| 항목 | 현재 | 개선 후 |
|------|------|---------|
| **고장형태(FM)** | 용접 불량 | - |
| **고장원인(FC)** | 용접전류 변동 | - |
| **예방관리(PC)** | 작업표준서 교육 (O=7) | SPC 관리 + 자동전류보정 인터락 (O=4) |
| **검출관리(DC)** | 샘플 육안검사 (D=8) | 인라인 초음파탐상(UT) 전수검사 → 이상 시 라인 정지 (D=3) |
| **AP** | **H** | **L** |

#### 예시 2: AP=H, S=10, O=5, D=6 → 개선 후 O=3, D=3

| 항목 | 현재 | 개선 후 |
|------|------|---------|
| **고장형태(FM)** | 절연 파괴 | - |
| **고장원인(FC)** | 코팅 두께 부족 | - |
| **예방관리(PC)** | 주기적 두께 측정 (O=5) | 자동 코팅기 + 실시간 SPC (O=3) |
| **검출관리(DC)** | 전수 육안검사 (D=6) | 와전류 도막두께계 전수 자동측정 → ±10% 이탈 시 자동 NG 처리 (D=3) |
| **AP** | **H** | **L** |

#### 예시 3: 체결 토크 불량 (PC/DC 구분 명확화)

| 항목 | ❌ 잘못된 예 (DC에 PC 혼입) | ✅ 올바른 예 |
|------|--------------------------|-----------|
| **예방관리(PC)** | 토크렌치 검교정 주기 단축 | 토크렌치 검교정 주기 1M→2W 단축 + SPC X-R Chart 적용 (O=4) |
| **검출관리(DC)** | ~~Gage R&R 정기 실시~~ | 인라인 자동 토크센서(±3% 정밀도) 전수 체결검사 → 미달 시 다음공정 이동 불가 인터락 (D=3) |

## 7. LLD 추천 엔진 활용

### 7.1 LLD 매칭 점수 체계 (8가지 가중치)

시스템은 현재 FMEA 항목과 LLD 데이터 간 유사도를 다음 8가지 기준으로 자동 계산한다:

| # | 매칭 기준 | 가중치 | 설명 |
|---|----------|--------|------|
| 1 | FM ↔ LLD 고장형태 | **3.0** | 고장형태(FM) 키워드 직접 매칭 |
| 2 | FC ↔ LLD 고장원인 | **3.0** | 고장원인(FC) 키워드 직접 매칭 |
| 3 | FM ↔ LLD 고장원인 (교차) | 1.0 | FM에서 LLD.cause 관련성 확인 |
| 4 | FC ↔ LLD 고장형태 (교차) | 1.0 | FC에서 LLD.failureMode 관련성 확인 |
| 5 | FM/FC ↔ LLD 개선대책 | 0.5 | LLD.improvement 참고 (폴백) |
| 6 | 현재 PC/DC 정합성 | **1.5** | 예방/검출 카테고리 일치 시 우대 |
| 7 | 반영 대상 일치 | 0.5 | 예방/검출 맥락 일치 시 가점 |
| 8 | 완료 사례 우선 | 가변 | G(+0.4) > Y(+0.15) > R(-0.1) |

### 7.2 DC 추천 품질 검증 기준 ★ NEW

LLD 시스템이 DC로 추천한 내용을 적용하기 전, 다음 체크리스트를 반드시 확인한다:

```
□ 추천 내용에 구체적인 "검사장치명"이 포함되어 있는가?
   예: 버니어캘리퍼스, 마이크로미터, 비전검사, AOI, 토크센서, 경도계 등
   → 없으면 5.4항 표에서 해당 고장형태에 맞는 검사장치 선택 후 교체

□ 추천 내용이 "관리 프로세스 개선" 성격인가? (PC 혼입 여부 판단)
   예: "검교정 주기 단축", "알림 시스템 도입", "MSA 정기 실시" → PC 성격
   → PC 성격이면 DC로 사용 금지. PC 필드로 이동하고 DC는 별도 작성

□ 불합격 처리 방법이 명시되어 있는가?
   예: "NG 자동 배출", "후공정 이동 불가 인터락", "즉시 격리" 등
   → 없으면 추가 기술

□ D값이 검사방법과 일치하는가? (5.1항 등급표 참조)
   사람 의존 검사 → D=6~7 / 자동측정기 → D=3~5 / Poka-Yoke → D=1~2
```

### 7.3 LLD 구분 코드

| 코드 | 의미 | 설명 |
|------|------|------|
| **RMA** | 반품분석 | 고객 반품에서 발견된 이슈 |
| **ABN** | 이상품 | 공정 중 이상품 발생 사례 |
| **CIP** | 지속개선 | 지속적 개선 활동 결과 |
| **ECN** | 설계변경 | 설계 변경 통보에 따른 대책 |
| **FieldIssue** | 필드이슈 | 시장/현장에서 발생한 이슈 |
| **DevIssue** | 개발이슈 | 개발 단계에서 발견된 이슈 |

### 7.4 LLD 상태 관리

| 상태 | 색상 | 의미 | LLD 매칭 가중치 |
|------|------|------|----------------|
| **G** (Green) | 초록 | 완료 — 대책 실행 및 효과 확인 완료 | +0.4 (최우선 추천) |
| **Y** (Yellow) | 노랑 | 진행중 — 대책 실행 중 | +0.15 |
| **R** (Red) | 빨강 | 미완료 — 대책 미수립 또는 미실행 | -0.1 (비추천) |

## 8. 개선대책 작성 시 주의사항

### 8.1 필수 준수 사항

| # | 항목 | 설명 |
|---|------|------|
| 1 | **S값은 변경 불가** | 심각도는 설계로만 변경 가능 — 개선대책 대상이 아님 |
| 2 | **O 최소값 = 2** | 발생도 1은 "발생 사실상 불가"이므로 일반 개선으로 도달 불가 |
| 3 | **D 최소값 = 2~3** | 검출도 1은 Poka-Yoke 수준 — 일반 검사 장비로는 D=3이 한계 |
| 4 | **FC의 D ≤ 8** | 고장원인(FC) 검출관리는 D=1~8만 허용 (D=9~10은 FM 전용) |
| 5 | **육안검사 ≠ D=1~5** | 사람 의존 검사는 최소 D=6~7 — 과대 평가 금지 |
| 6 | **비전검사 = D=7** | MSA 미입증 기계검사 — 보수적 평가 적용 |
| 7 | **구체성 필수** | "관리 강화" 같은 추상적 표현 금지 → 구체적 방법/장비 명시 |
| 8 | **정량 목표 필수** | 목표 O/D값을 반드시 숫자로 명시 |

### 8.2 금지 사항

- **추상적 개선대책**: "품질 향상", "관리 강화", "주의" 등 → 구체적 행동으로 기술
- **근거 없는 O/D 저감**: 예방/검출관리 변경 없이 O/D만 낮추는 행위
- **S값 임의 변경**: 설계 변경 없이 S값을 조정하는 행위
- **육안검사 D=1~5 평가**: 사람 의존 검사를 과대 평가하는 행위
- **상태=R인 LLD 무조건 적용**: 미완료 대책을 검증 없이 적용

### 8.3 올바른 작성 예시 vs 잘못된 작성 예시

| 항목 | 잘못된 예시 | 올바른 예시 |
|------|-----------|-----------|
| **예방관리** | "작업자 교육 강화" | "토크렌치 검교정 주기 1M→2W 단축 + SPC X-R Chart 적용" |
| **검출관리** | ~~"검사 강화"~~ ← 추상적 | "인라인 자동 토크센서(±3% 정밀도) 전수검사 도입 → 미달 시 다음공정 이동 불가 인터락" |
| **검출관리** | ~~"검교정 이력 전산화 및 알림 도입"~~ ← PC 성격 | "마이크로미터(MSA 검증 완료) Lot당 전수 치수검사 → 기준 초과 시 NG 격리" |
| **목표 O** | "낮춘다" | "O=7 → O=4 (SPC 관리 후)" |
| **목표 D** | "개선" | "D=8 → D=3 (자동 토크센서 D=3)" |
| **담당자** | 미기재 | "품질보증팀 김OO / 2026-04-30" |

### 8.4 PC와 DC 혼동 방지 판별 규칙 ★ NEW

다음 질문에 **YES**면 PC, **NO**면 DC로 분류한다:

```
Q1. 이 대책은 고장원인(FC)이 애초에 발생하지 않도록 막는 것인가?
    YES → 예방관리(PC) / NO → Q2로 이동

Q2. 이 대책에 "검사장치명" 또는 "측정 방법"이 명시되어 있는가?
    YES → 검출관리(DC) / NO → 구체적 검사장치 추가 후 DC로 작성

---
【PC로 분류되는 키워드】
  주기 단축, 예방보전, 교육, 작업표준, SPC 도입, 인터락(가동 전),
  4M 변경관리, 검교정 주기, Poka-Yoke(설비 중단), DOE, 조건 최적화

【DC로 분류되는 키워드】
  외관검사, 버니어캘리퍼스, 마이크로미터, 자동저울, 수동저울,
  비전검사, AOI, UTM, 경도계, 도막두께계, 색차계, 토크센서,
  초음파탐상(UT), X-ray, 게이지, 전수검사, 샘플링검사, AQL
```

## 9. 시스템 자동 추천 활용 가이드

### 9.1 개선추천 모달 사용법

1. **ALL 탭** → 메뉴바 **[개선추천]** 버튼 클릭
2. AP=H 항목이 상단에 표시 → 행별 **체크박스**로 적용 대상 선택
3. 시스템이 3가지 개선 경로(O만/D만/O+D) 제시 → **최소 총변화량** 경로 권장
4. PC/DC 개선안 확인 → **DC 항목은 5.3항 체크리스트 검토 후** 필요 시 수정 → **[적용]** 클릭

### 9.2 AP 셀 직접 클릭

1. ALL 탭에서 **AP 셀** 직접 클릭
2. 해당 행의 3가지 개선 경로 즉시 표시
3. 경로 선택 → 자동으로 O'/D'/PC'/DC' 업데이트

### 9.3 LLD 자동 매칭

1. LLD 페이지에서 사전 등록된 습득교훈 데이터 확인
2. ALL 탭에서 개선대책 입력 시 → LLD 추천 점수 상위 항목 자동 제시
3. 완료(G) 상태의 LLD가 최우선 추천
4. **추천된 DC 항목은 7.2항 검증 기준으로 반드시 검토** — 추천 내용이 PC 성격이면 교체
