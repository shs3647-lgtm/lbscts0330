# 3. 기초정보 Import

## 3.1 Import 화면

> 📸 **화면 목업**: [09_Import.html](화면캡처/09_Import.html)

**화면 설명**

엑셀 파일로 작성한 공정 정보를 시스템에 가져오는 화면입니다.

**사용 순서**

1. **템플릿 다운로드**: 상단 "템플릿 다운로드(Download Template)" 버튼을 클릭하여 엑셀 양식을 받습니다
2. **엑셀 작성**: 다운로드한 양식에 공정 정보를 입력합니다
3. **파일 선택**: "파일 선택(Select File)" 영역에 엑셀 파일을 끌어다 놓거나 클릭하여 선택합니다
4. **미리보기 확인**: 가져온 데이터가 표에 표시됩니다
5. **Import 실행**: "Import" 버튼을 클릭하면 데이터가 시스템에 저장됩니다

**주의사항**
- 엑셀 양식을 변경하지 마세요 (행/열 구조를 그대로 사용)
- 공정번호와 공정명은 반드시 입력해야 합니다

## 5.2 리스크 평가 (S/O/D)

> 📸 **화면 목업**: [20_SOD선택.html](화면캡처/20_SOD선택.html)

**화면 설명**

S(심각도), O(발생도), D(검출도) 점수를 선택하는 팝업입니다.

| 항목 | 의미 | 점수 범위 |
|------|------|----------|
| S (심각도/Severity) | 고장이 발생하면 얼마나 심각한가 | 1(무시)~10(매우 심각) |
| O (발생도/Occurrence) | 고장이 얼마나 자주 발생하는가 | 1(거의 안남)~10(매우 자주) |
| D (검출도/Detection) | 고장을 출하 전에 발견할 수 있는가 | 1(확실히 발견)~10(못 찾음) |

**AP (리스크) 자동 계산**

| AP 등급 | 의미 | 색상 | 필요한 조치 |
|---------|------|------|-----------|
| **H** (High) | 높은 리스크 | 빨강 | 반드시 개선 필요 |
| **M** (Medium) | 보통 리스크 | 노랑 | 개선 권장 |
| **L** (Low) | 낮은 리스크 | 초록 | 현재 상태 유지 |

**SOD 재평가 (O-Rec / D-Rec) — v6.0 신규**

5ST 리스크분석 헤더의 O-Rec / D-Rec 버튼으로 발생도(O)와 검출도(D)를 일괄 재평가합니다.

| 버튼 | 프로세스 | 결과 |
|------|---------|------|
| **O-Rec** | 1순위: 산업DB 매칭 (kr_industry_prevention 25건) → 2순위: 키워드 추론 | 발생도 일괄 재산정 |
| **D-Rec** | 1순위: 산업DB 매칭 (kr_industry_detection 25건) → 2순위: 키워드 추론 | 검출도 일괄 재산정 |
| **S-Rec** | FE 텍스트와 SOD 평가기준 유사도 기반 | 심각도 예비 재산정 |

> **핵심 규칙**: O=1, D=1 자동추천은 절대 금지 — 엔지니어만 판단 가능. 최소 추천값은 2.

## 5.2a S추천 (심각도 추천 매핑) — v6.1 신규

**화면 설명**

고장영향(FE)에 대한 심각도(S)를 과거 사례 기반으로 추천하는 기능입니다.
FMEA 작성 시 결정한 심각도가 자동으로 DB에 누적되어, 다른 프로젝트에서 비슷한 고장영향을 작성할 때 추천됩니다.

**핵심 개념**: 모든 프로젝트가 동일한 S추천 DB를 공유합니다. 한 프로젝트에서 추가한 사례가 모든 프로젝트에서 활용됩니다.

**S추천 탭 열기**: 워크시트 상단 메뉴바에서 **"S추천(S-Rec)"** 버튼 클릭

| 항목 | 하는 방법 |
|------|----------|
| 데이터 확인 | S추천 탭 열기 시 DB에서 자동 로드 |
| YIELD 추천 추가 | "YIELD 6·7 추천 추가" 버튼 → YIELD 관련 심각도 기본 데이터 DB 저장 |
| 엑셀 Import | "엑셀 Import" → .xlsx 파일 선택 → DB에 일괄 저장 |
| 엑셀 Export | "엑셀 Export" → 현재 데이터를 .xlsx로 다운로드 |
| 템플릿 받기 | "템플릿 받기" → 빈 엑셀 양식 다운로드 |
| 일괄 적용 | **4ST 고장분석** 헤더의 **"S-Rec"** 배지 클릭 → 미리보기 후 적용 |

**자동 누적 프로세스**

```
1. 엔지니어가 ALL 탭에서 심각도(S) 값을 설정
2. 시스템이 고장영향 텍스트 + S값을 DB에 자동 기록
3. 다른 프로젝트에서 같은 고장영향 작성 시 S값 추천
```

**매핑표 컬럼 설명**

| 컬럼 | 의미 |
|------|------|
| Scope | 적용범위 (YP/SP/USER) |
| 제품기능(C2) | 관련 공정 |
| 요구사항(C3) | 관련 요구사항 |
| 고장영향(FE) | 고장 영향 텍스트 |
| 심각도(S) | 1~10 심각도 점수 |
| AIAG-VDA 근거 | 사용 이력 및 기준 근거 |

> **참고**: S추천 데이터는 온프레미스 서버의 중앙 DB에 저장됩니다.
> 별도 백업 없이 DB 백업에 포함됩니다.

## 5.4a SRP 프로세스 (SOD Reduction Path) — v6.0 신규

6ST 최적화의 핵심 프로세스입니다. H/M 등급 항목을 L로 내리기 위한 **최소 개선경로**를 자동 산출합니다.

**SRP 경로 유형**

| 배지 | 색상 | 의미 | N/A 처리 |
|------|------|------|---------|
| **O:N** | 주황 | O만 개선하면 L 도달 가능 | D측 = N/A |
| **D:N** | 파랑 | D만 개선하면 L 도달 가능 | O측 = N/A |
| **O&D:N** | 보라 | O와 D 둘 다 개선 필요 | 양측 모두 추천 |

**최소경로 알고리즘**
- S(심각도)는 고정 — 공정 고유 특성이므로 변경 불가
- O(발생도)와 D(검출도)만 개선 가능 (최소값 = 2)
- Cost = |ΔO| + |ΔD| — 총 변화량이 최소인 경로를 자동 선택
- AIAG-VDA AP 테이블 기준으로 L 도달 여부 판정

**6ST 최적화 프로세스 순서**

```
1단계: LLD추천 클릭
  → LLD DB에서 FC/FM 매칭 → H/M 항목에 우선 배정
  → 매칭 등급: 정확(녹) > 공정(청) > 수평(보라) > Miss(회)

2단계: 개선추천 클릭
  → L레벨 제외 (이미 목표 달성)
  → LLD 매칭 항목 제외 (이미 1단계에서 배정)
  → 나머지 H/M 항목에 산업DB 기반 개선안 배정
  → SRP 최소경로 기반 O/D 목표 산출
  → 불필요한 측면은 자동 N/A 처리
```

**예시: S=9, O=8, D=6 → AP=H**

| 경로 | 목표 | Cost | 선택 |
|------|------|------|------|
| O경로 | O=8→2, D 유지 | 6 | |
| D경로 | O 유지, D=6→2 | 4 | **최소경로** |
| O&D | O=8→6, D=6→4 | 4 | |

→ D경로 선택: 검출관리 개선 추천, 예방관리는 N/A

## 5.5 개선추천 기능

| 체크박스 | 적용할 항목을 선택/해제 |

| 3 | 키워드 추론 | pcOccurrenceMap / detectionRatingMap |

| 항목 | 하는 방법 |
|------|----------|
| 적용 | 체크된 항목만 "선택 적용" 버튼 클릭 |

| 선택 해제 | 적용하고 싶지 않은 행의 체크박스를 해제 |

> **참고**: L등급 항목과 LLD 매칭 완료 항목은 목록에 표시되지 않습니다.

# 8. 문서 연동 (CP↔PFD↔PFMEA)

## 8.1 연동 개요

PFMEA, 관리계획서(CP), 공정흐름도(PFD) 세 문서는 서로 데이터를 주고받을 수 있습니다.

| 항목 | 설명 |
|------|------|
| ID 개정 접미사 `-r00` | 모든 문서 ID 끝에 개정 번호가 자동 포함됩니다 (r00→r01→r02) |
| 6방향 연동 | CP↔PFD, PFD↔PFMEA, CP↔PFMEA 세 쌍이 양방향으로 연동 |
| 개정 출처 마커 | 연동된 항목에 ᶠ(FMEA), ᶜ(CP), ᵖ(PFD) 마커가 붙어 출처를 추적 |
| 4M 원본 보존 | 연동 시 원본 4M 데이터(사람/설비/재료/환경)가 보존됩니다 |

| 항목 | 하는 방법 |
|------|----------|
| 연동 실행 | 각 워크시트 메뉴바의 "연동(Sync)" 버튼 클릭 |
| 출처 확인 | 셀 내 위첨자 마커(ᶠ/ᶜ/ᵖ) 색상으로 출처 확인 |
| 개정 번호 확인 | 문서 ID 끝의 `-r00`, `-r01` 등으로 현재 개정 버전 확인 |

> **상세 가이드**: [연동 및 개정 상세 가이드](연동_개정_가이드.md)

| ECN | 변경점 | 설계/공정 변경 관련 교훈 (Engineering Change Notice) |
| Field Issue | 필드 이슈 | 양산/출하 후 현장에서 발생한 문제 |
| DevIssue | 개발 이슈 | 개발 단계에서 발생한 이슈 기록 |

### 상태 관리

| 상태 | 색상 | 의미 |
|------|------|------|

| Y | 노랑 | 진행중 — 개선 조치를 진행 중입니다 |
| R | 빨강 | 미완료 — 아직 착수하지 않았습니다 |

|------|------|

| 제품명 | 해당 제품명 |
| 공정명 | 해당 공정명 |
| 고장형태(FM) | Failure Mode — 어떤 고장이 발생했는지 |
| 고장원인(FC) | Failure Cause — 고장의 근본 원인 |
| S/O/D | 심각도/발생도/검출도 (1~10) |
| 예방관리 개선 | 예방관리 관점의 개선 내용 |
| 검출관리 개선 | 검출관리 관점의 개선 내용 |
| 개선일자 | 개선이 완료된 날짜 |
| 상태 | G(완료)/Y(진행중)/R(미완료) |
| 담당자 | 개선 담당자 이름 |
| FMEA | 연결된 FMEA 프로젝트 ID |
| 첨부 | 근거서류 URL (드라이브/위키 링크) |

### 워크시트 자동 추천 연동

LLD에 등록된 교훈은 워크시트 ALL탭에서 활용됩니다:

1. **자동 추천**: FM/FC 입력 시 LLD에서 유사한 교훈을 자동으로 추천합니다
2. **수평전개**: "⇄H.P." 버튼으로 다른 프로젝트에 동일 교훈을 전파합니다
3. **FMEA 연결**: 교훈이 적용된 FMEA 프로젝트가 FMEA 컬럼에 표시됩니다

### 일반적인 작업 흐름

1. **수동 등록**: "+Add Row" → 셀 클릭하여 정보 입력 → "⊞Save"
2. **Excel Import**: "📋양식"으로 템플릿 다운로드 → 데이터 작성 → "↑Import" → "⊞Save"
3. **수평전개**: 교훈 행 선택 → "⇄H.P." → 대상 프로젝트 선택
4. **조회/필터**: 상단 구분 필터 또는 검색창으로 원하는 교훈 찾기

## 11.2 결재 흐름

**결재 3단계**

| 단계 | 역할 | 버튼 | 설명 |
|------|------|------|------|
| 1. 작성(Draft) | 작성자 | **상신(Submit)** | 검토를 요청합니다 |
| 2. 검토(Review) | 검토자 | **검토 승인** / **반려** | 내용을 검토합니다 |
| 3. 승인(Approve) | 승인자 | **최종 승인** / **반려** | 최종 결재합니다 |

**결재 흐름**

```
[작성자] 상신 → [검토자] 검토 승인 → [승인자] 최종 승인 → 결재 완료
                       ↓ 반려                    ↓ 반려
                  [작성자] 수정 후 재상신     [작성자] 수정 후 재상신
```

**담당자 지정**: 각 단계에서 이름 부분을 클릭하면 사용자 선택 팝업이 열립니다. 이미 지정된 담당자도 재변경 가능합니다.

**회수 기능**: 상신 후 검토자가 미처리 시 작성자가 "회수(Withdraw)" 버튼으로 취소 가능합니다.
