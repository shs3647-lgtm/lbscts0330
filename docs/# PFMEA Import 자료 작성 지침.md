# PFMEA 엑셀 Import 자료 작성 지침

> **버전**: v6.3 (2026-03-28)  
> **목적**: PFMEA 시스템에 Import할 엑셀 파일의 정확한 작성 규칙

---

## 1. 파일 구조 — 필수 4개 시트

| 시트명 | 역할 | 필수 컬럼 |
|--------|------|-----------|
| **L1 통합(C1-C4)** | 고장영향(FE) 정의 | C1, C2, C3, C4 |
| **L2 통합(A1-A6)** | 공정/고장형태(FM) 정의 | A1, A2, A3, A4, SC, A5, A6 |
| **L3 통합(B1-B5)** | 작업요소/고장원인(FC) 정의 | 공정번호, 4M, B1, B2, B3, SC, B4, B5 |
| **FC 고장사슬** | FE↔FM↔FC 연결 정의 | FE구분, FE, 공정번호, FM, 4M, WE, FC, PC, DC, O, D, AP |

> ⚠️ 시트명에 반드시 `L1`, `L2`, `L3`, `FC`와 `통합` 또는 `고장사슬`이 포함되어야 합니다.

---

## 2. L1 통합(C1-C4) — 고장영향 시트

### 컬럼 정의

| 컬럼 | 헤더 | 설명 | 필수 |
|------|------|------|------|
| 1 | C1. FE구분 | `YP` / `SP` / `USER` | ✅ |
| 2 | C2. 공정기능 | 해당 공정의 기능 설명 | |
| 3 | C3. 제품요구사항 | 제품이 만족해야 할 요구사항 | |
| 4 | C4. 고장영향(FE) | 고장으로 인한 영향 | ✅ |

### 작성 규칙

1. **C1 FE구분**: 반드시 `YP`(생산), `SP`(특성), `USER`(사용자) 중 하나
2. **C4 고유성**: 동일 FE 텍스트가 중복되면 안 됨
3. **FC 교차검증**: FC 시트에서 참조하는 FE만 작성 (FC에 없는 FE는 삭제)

### 예시

```
| C1   | C2          | C3        | C4                                    |
|------|-------------|-----------|---------------------------------------|
| YP   | Wafer 세정   | 이물 제거  | Particle 오염으로 인한 제품 불량        |
| SP   | Au Bump 도금 | 접합 신뢰성 | IMC 과성장에 의한 접합부 열화           |
| USER | 최종검사     | 품질 보증   | 고객 라인 정지, 클레임                 |
```

## 3. L2 통합(A1-A6) — 공정/고장형태 시트

### 컬럼 정의

| 컬럼 | 헤더 | 설명 | 필수 |
|------|------|------|------|
| 1 | A1. 공정번호 | 숫자 (10, 20, 30...) | ✅ |
| 2 | A2. 공정명 | 공정 이름 | ✅ |
| 3 | A3. 공정기능 | 공정이 수행하는 기능 | |
| 4 | A4. 제품특성 | 제품이 가져야 할 특성 | |
| 5 | SC | 특별특성 (CC/SC) | |
| 6 | A5. 고장형태(FM) | 잠재적 고장 모드 | ✅ |
| 7 | A6. 검출관리 | 검출 방법 | |

### 작성 규칙

1. **A1 공정번호**: 숫자만 사용 (01, 10, 20, 30...). 같은 공정번호에 여러 FM 가능
2. **A5 고장형태**: 같은 공정번호 내에서 고유해야 함
3. **★★★ FC 교차검증 필수 ★★★**: 
   - **L2의 모든 FM은 FC 시트에 반드시 존재해야 함**
   - FC에 없는 FM이 있으면 Import 오류 발생
   - `forge-validate-excel.js`로 사전 검증

### 예시

```
| A1  | A2             | A3        | A4        | SC | A5                | A6          |
|-----|----------------|-----------|-----------|----|--------------------|-------------|
| 10  | Wafer Cleaning | 이물 제거  | 파티클 수준 |    | Particle 초과       | 파티클 카운터 |
| 10  | Wafer Cleaning | 이물 제거  | 세정 시간   |    | 세정 시간 부족       | 타이머 모니터링 |
| 30  | UBM Sputter    | UBM 두께  | 막 두께    | CC | UBM 두께 부족       | XRF 측정     |
```

## 4. L3 통합(B1-B5) — 작업요소/고장원인 시트

### 컬럼 정의

| 컬럼 | 헤더 | 설명 | 필수 |
|------|------|------|------|
| 1 | 공정번호 | L2 A1과 동일 | ✅ |
| 2 | 4M | Machine/Material/Method/Man | |
| 3 | B1. 작업요소 | 작업 요소명 | |
| 4 | B2. 작업기능 | 작업 요소의 기능 | |
| 5 | B3. 공정특성 | 공정이 가진 특성 | |
| 6 | SC | 특별특성 (CC/SC) | |
| 7 | B4. 고장원인(FC) | 잠재적 고장 원인 | ✅ |
| 8 | B5. 예방관리 | 예방 관리 방법 | |

### 작성 규칙

1. **행수 = FC 시트 행수**: L3 행수와 FC 시트 행수는 반드시 동일해야 함 (1:1 매핑)
2. **공정번호**: L2에 존재하는 공정번호만 사용
3. **B4 고장원인**: FC 시트의 FC 컬럼과 동일 텍스트

### 예시

```
| 공정번호 | 4M       | B1           | B2        | B3        | SC | B4              | B5            |
|---------|----------|--------------|-----------|-----------|----|--------------------|---------------|
| 10      | Machine  | DI Water 공급 | 순수 공급   | 순수 저항  |    | DI Water 오염       | 저항 모니터링    |
| 10      | Machine  | 초음파 세정기  | 세정 효율   | 초음파 출력 |    | 초음파 출력 저하      | 출력 교정       |
| 30      | Machine  | Sputter 챔버  | 증착 균일성 | 타겟 수명  | CC | Target 마모         | 교체 주기 관리   |
```

## 5. FC 고장사슬 — 고장연결 시트 ★핵심★

### 컬럼 정의

| 컬럼 | 헤더 | 설명 | 필수 |
|------|------|------|------|
| 1 | FE구분 | YP/SP/USER (L1 C1과 동일) | ✅ |
| 2 | FE(고장영향) | L1 C4와 동일 텍스트 | ✅ |
| 3 | L2-1.공정번호 | L2 A1과 동일 | ✅ |
| 4 | FM(고장형태) | L2 A5와 동일 텍스트 | ✅ |
| 5 | 4M | Machine/Material/Method/Man | |
| 6 | 작업요소(WE) | L3 B1과 동일 | |
| 7 | FC(고장원인) | L3 B4와 동일 텍스트 | ✅ |
| 8 | B5.예방관리 | L3 B5와 동일 | |
| 9 | A6.검출관리 | L2 A6과 동일 | |
| 10 | O | Occurrence (1-10) | |
| 11 | D | Detection (1-10) | |
| 12 | AP | Action Priority (H/M/L) | |

### ★★★ Forward-Fill 규칙 ★★★

FC 시트에서 **FE구분, FE, 공정번호, FM**은 **첫 등장 행에만 기재**합니다.
같은 그룹의 나머지 행은 **공란**으로 두면 시스템이 자동으로 위 값을 채웁니다.

```
| FE구분 | FE(고장영향)           | 공정번호 | FM(고장형태)    | 4M       | WE           | FC(고장원인)         |
|--------|----------------------|---------|---------------|----------|--------------|---------------------|
| YP     | Particle 오염 불량     | 10      | Particle 초과  | Machine  | DI Water 공급 | DI Water 오염        |
|        |                      |         |               | Machine  | 초음파 세정기  | 초음파 출력 저하       |  ← FE/공정/FM 공란
|        |                      |         | 세정 시간 부족  | Method   | 세정 레시피    | 세정 시간 미준수      |  ← FM만 변경
| YP     | 정렬 불량 수율 손실     | 20      | Wafer 정렬 불량 | Machine  | Aligner      | Aligner 구동 오차     |  ← FE/공정/FM 모두 변경
```

### 매칭 규칙

```
FC 각 행 = 하나의 FailureLink(FL)

  FE(고장영향)  ←→  L1 C4 (scope+FE 복합키)
  FM(고장형태)  ←→  L2 A5 (processNo+FM 복합키)  
  FC(고장원인)  ←→  L3 B4 (위치 인덱스 1:1 매핑)
```

## 6. 교차검증 체크리스트

Import 전에 반드시 확인해야 할 사항:

### 필수 검증 (error)

| # | 검증 항목 | 설명 |
|---|----------|------|
| 1 | **L2 FM ⊆ FC FM** | L2의 모든 FM이 FC에 존재해야 함 |
| 2 | **L3 행수 = FC 행수** | L3와 FC의 유효 행수가 동일해야 함 |
| 3 | **FC B4 ⊆ L3 B4** | FC의 FC텍스트가 L3 B4에 존재해야 함 |
| 4 | **A1 일관성** | L2, L3, FC의 공정번호가 일관되어야 함 |

### 권장 검증 (warning)

| # | 검증 항목 | 설명 |
|---|----------|------|
| 5 | L1 FE ⊆ FC FE | L1의 FE가 FC에 참조되지 않으면 경고 |
| 6 | FC FM ⊆ L2 FM | FC의 FM이 L2에 없으면 cross-process 매칭 시도 |
| 7 | SC 일관성 | L2와 L3의 특별특성(CC/SC) 일치 |

### 검증 명령어

```bash
node scripts/forge-validate-excel.js
```

## 7. 자주 하는 실수

### ❌ 실수 1: L2에 FM을 추가했지만 FC에 연결 안 함

```
L2에 "Dimple 규격 초과" FM 추가 → FC 시트에 해당 FM 행 없음
→ Import 시 FM=28 이지만 FL=115 (2건 미연결)
→ 해결: FC 시트에 해당 FM의 고장사슬 행 추가 또는 L2에서 삭제
```

### ❌ 실수 2: FC Forward-Fill 순서 오류

```
잘못된 예:
| YP | Particle 오염 | 10 | FM-A | ... |
| SP |               | 20 | FM-B | ... |  ← FE만 변경하려면 FE구분+FE 모두 기재!

올바른 예:
| YP | Particle 오염     | 10 | FM-A | ... |
| SP | IMC 접합부 열화     | 20 | FM-B | ... |  ← FE구분+FE+공정번호+FM 모두 기재
```

### ❌ 실수 3: L3 행수 ≠ FC 행수

```
L3: 115행, FC: 110행 → 5행 불일치
→ FC[i] = L3[i] 위치 매핑 실패
→ 해결: L3과 FC의 행수를 반드시 동일하게 맞춤
```

## 8. 엔티티 관계도

```
L1 (고장영향 FE)          L2 (공정/고장형태 FM)          L3 (작업요소/고장원인 FC)
┌───────────────┐      ┌──────────────────┐        ┌──────────────────┐
│ C1: FE구분     │      │ A1: 공정번호      │        │ 공정번호          │
│ C2: 공정기능    │      │ A2: 공정명        │        │ 4M               │
│ C3: 제품요구사항 │      │ A5: 고장형태(FM)  │        │ B1: 작업요소       │
│ C4: 고장영향(FE)│      │ A6: 검출관리      │        │ B4: 고장원인(FC)   │
└───────┬───────┘      └────────┬─────────┘        └────────┬─────────┘
        │                       │                           │
        │    FC 고장사슬 시트 (FailureLink)                    │
        │    ┌─────────────────────────────────────────┐    │
        └────┤  FE구분 + FE  │  공정번호 + FM  │  FC    ├────┘
             │  (scope+FE    │  (processNo+FM  │  (위치  │
             │   복합키 매칭)  │   복합키 매칭)   │  인덱스)│
             └─────────────────────────────────────────┘
```

### 매칭 방식

| 엔티티 | 매칭 방법 | 설명 |
|--------|----------|------|
| **FC → L3** | **위치 인덱스** | FC[i] = L3[i] (1:1 물리적 행 매핑) |
| **FM → L2** | **복합키** | processNo + FM텍스트 → L2 UUID |
| **FE → L1** | **복합키** | scope + FE텍스트 → L1 UUID |

> ⚠️ **이름(텍스트) 매칭 금지**: FC↔L3은 반드시 위치 인덱스로 매칭. 텍스트 비교 사용 금지.

## 9. Import 실행 절차

1. **엑셀 작성** → 4개 시트 구성
2. **사전 검증** → `node scripts/forge-validate-excel.js` 실행
3. **오류 수정** → 교차검증 오류 해결
4. **Import** → UI에서 엑셀 업로드
5. **결과 확인** → FM/FE/FC/FL 카운트 확인
6. **워크시트** → 고장연결 탭에서 ●/⚠️ 아이콘 확인
