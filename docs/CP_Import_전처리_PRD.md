# CP Import 전처리 PRD (Python 전처리기 스펙)

> **목적**: 레거시 관리계획서(CP) Excel → 시스템 Import용 5시트 Excel로 변환하는 Python 전처리기의 입출력 스펙
> **작성일**: 2026-03-05
> **최종 목표**: 전처리된 CP Excel → 시스템 CP Import → CP에서 PFMEA 기초자료 자동 생성

---

## 1. 전체 흐름

```
레거시 CP Excel (다양한 형식)
    │
    ▼
[Python 전처리기] ← 이 PRD가 정의하는 범위
    │
    ▼
시스템 Import용 Excel (.xlsx, 5시트 or 단일시트)
    │
    ▼
[시스템 CP Import] parseCPExcel() — 기존 파서가 자동 처리
    │
    ▼
[CP → PFMEA 역변환] — 시스템이 자동 생성
    │  공정번호, 공정명, 공정기능, 설비, 제품특성, 공정특성 → FMEA 기초자료
    ▼
PFMEA 워크시트 (사용자가 고장분석 FM/FC/FE만 추가 작성)
```

## 4. 공정번호 규칙 (CRITICAL)

```python
# ✅ 올바른 공정번호
"10", "20", "30", "100", "110"

# ❌ 잘못된 공정번호 (파서가 무시함)
"10번", "공정10", "No.10", "10.", "010"
```

| 규칙 | 설명 |
|---|---|
| **숫자만** | `/^\d+$/` 패턴만 허용 (파서의 `parseInt` 검증) |
| **정수** | 소수점 불허 |
| **10 단위 권장** | 10, 20, 30... (중간 삽입 여유) |
| **중복 불허** | 같은 공정번호는 하나의 공정으로 병합됨 |
| **정렬** | 공정번호 숫자순으로 자동 정렬됨 |

**전처리 시 정규화 예시**:
```python
def normalize_process_no(raw: str) -> str:
    """레거시 공정번호를 시스템 형식으로 정규화"""
    # "10번" → "10", "No.20" → "20", "공정30" → "30"
    digits = re.findall(r'\d+', str(raw))
    return digits[0] if digits else ''
```

## 5. 특별특성 규칙

시스템에서 인식하는 특별특성 값:

| 입력 가능 값 | 시스템 내부 값 | 의미 |
|---|---|---|
| `CC`, `◇` | CC | Critical Characteristic (핵심 특성) |
| `SC`, `◆` | SC | Significant Characteristic (주요 특성) |
| `IC`, `△` | IC | Important Characteristic (중요 특성) |
| 빈값, `-`, `N/A` | (없음) | 특별특성 아님 |

**전처리 시 정규화**:
```python
def normalize_special_char(raw: str) -> str:
    raw = str(raw).strip().upper()
    if raw in ('CC', '◇', 'C/C'): return 'CC'
    if raw in ('SC', '◆', 'S/C'): return 'SC'
    if raw in ('IC', '△', 'I/C'): return 'IC'
    return ''
```

## 6. 설비/금형/지그 규칙

하나의 셀에 복수 설비가 들어갈 수 있음:

```python
# 입력 예시
"지게차, 입고대"
"MB Mixer\n믹싱롤"
"프레스/금형A"

# 구분자: 쉼표(,), 줄바꿈(\n), 슬래시(/)
```

**PFMEA 변환 시 4M 자동 분류**:

| 4M 코드 | 분류 | 키워드 예시 |
|---|---|---|
| MC | Machine (설비/금형/지그) | 프레스, 용접기, 가공기, 금형, 지그, JIG, 로봇, Mixer |
| MN | Man (사람) | 작업자, 검사원, 기술자 |
| EN | Environment (환경) | 온도, 습도, 청정, 이물 |
| IM | Indirect Material (부자재) | 그리스, 윤활유, 오일, 접착제 |

- **기본값**: MC (대부분의 CP 설비는 기계류)
- **MN은 PFMEA CP 연동 시 제외됨** (사람은 관리계획서 관리 대상이 아님)

## 7. 파서 동작 상세 (시스템 내부)

Python 전처리기 작성 시 알아야 할 시스템 파서 동작:

### 7.1 시트명 자동 감지
```
"공정현황"     → processInfo  (부분 매칭: "공정" 포함)
"검출장치"     → detector     (부분 매칭: "검출" 포함)
"관리항목"     → controlItem  (부분 매칭: "관리항목" 포함)
"관리방법"     → controlMethod (부분 매칭: "관리방법" 포함)
"대응계획"     → reactionPlan (부분 매칭: "대응" 또는 "reaction" 포함)
```

→ 시트명에 해당 키워드만 포함되면 자동 인식됨.

### 7.2 데이터 시작 행 자동 탐지
```
1행~5행 스캔 → 첫 번째 열이 숫자인 행 = 데이터 시작
또는 "공정번호", "번호", "No" 포함 행 다음 = 데이터 시작
기본값: 4행 (1행 메타 + 2행 그룹 + 3행 헤더 + 4행~ 데이터)
```

→ **1행 헤더 + 2행 데이터** 형식도 지원됨. 유연하게 처리.

### 7.3 공정번호 기준 병합
- 같은 공정번호의 데이터는 모든 시트에서 하나로 병합됨
- 관리항목에서 같은 공정번호에 여러 행 = 복수 제품특성/공정특성

## 8. 레거시 CP → 5시트 변환 가이드

### 8.1 레거시 단일시트 CP (가장 흔한 형식)

레거시 관리계획서는 보통 **단일 시트에 모든 정보**가 있고 **병합셀**이 많음:

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│ 공정 │ 공정 │ 설비 │ 제품 │ 공정 │ 특별 │ 스펙 │ 평가 │ 관리 │...
│ 번호 │  명  │      │ 특성 │ 특성 │ 특성 │ /공차│ 방법 │ 방법 │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  10  │      │      │ 이물 │입고  │      │온도  │육안  │검수  │
│ (병합)│입고  │지게차│ 질   │ 온도 │      │20~25 │검사  │기록  │
│      │      │      │보관  │      │      │      │      │      │
│      │      │      │상태  │ 습도 │      │60%↓  │      │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  20  │수입  │Mooney│Mooney│검사  │ CC   │50±5  │Mooney│성적서│
│      │검사  │Visco.│Visco.│정밀도│      │      │Visco.│관리  │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

**전처리 핵심 작업**:

1. **병합셀 해소**: 병합된 공정번호/공정명을 아래 행으로 복사 (carry-forward)
2. **컬럼 매핑**: 레거시 헤더명 → 시스템 표준 헤더명
3. **5시트 분리**: 단일시트 → 5개 시트로 분배
4. **공정번호 정규화**: 비표준 번호 → 숫자만
5. **특별특성 정규화**: 다양한 표기 → CC/SC/IC

|---|---|---|

```python
def validate_output(wb):
    errors = []

    # 1. 시트 존재 확인
    required_sheets = ['공정현황', '관리항목']  # 최소 필수
    optional_sheets = ['검출장치', '관리방법', '대응계획']

    # 2. 공정번호 검증
    for sheet in wb.sheetnames:
        for row in get_data_rows(sheet):
            pno = str(row[0]).strip()
            if pno and not re.match(r'^\d+$', pno):
                errors.append(f"[{sheet}] 행{row.idx}: 공정번호 '{pno}' 숫자가 아님")

    # 3. 공정번호 일관성 (공정현황의 공정번호가 다른 시트에도 존재)
    process_nos = set(get_process_nos('공정현황'))
    for sheet in ['관리항목', '관리방법', '대응계획']:
        sheet_nos = set(get_process_nos(sheet))
        orphans = sheet_nos - process_nos
        if orphans:
            errors.append(f"[{sheet}] 공정현황에 없는 공정번호: {orphans}")

    # 4. 필수 컬럼 확인
    # 공정현황: A열(공정번호) + B열(공정명) 필수
    # 관리항목: A열(공정번호) + B열(공정명) 필수

    # 5. 특별특성 값 검증
    valid_special = {'CC', 'SC', 'IC', ''}
    for row in get_data_rows('관리항목'):
        sc = str(row[4]).strip().upper()
        if sc and sc not in valid_special:
            errors.append(f"[관리항목] 행{row.idx}: 특별특성 '{sc}' 미인식")

    return errors
```

## 11. CP → PFMEA 자동 생성 매핑 (시스템 내부 — 참고용)

전처리된 CP가 Import되면, 시스템이 아래 매핑으로 PFMEA 기초자료를 자동 생성:

| CP 필드 | → | PFMEA 필드 | 설명 |
|---|---|---|---|
| 공정번호 | → | A1 (공정번호) | 그대로 |
| 공정명 | → | A2 (공정명) | 그대로 |
| 공정설명 | → | A3 (공정기능) | 없으면 자동생성 |
| 제품특성 | → | A4 (제품특성) | 복수 가능, specialChar 포함 |
| _(빈값)_ | → | A5 (고장형태) | 사용자가 FMEA에서 작성 |
| 설비/금형/지그 | → | B1 (작업요소) | 4M=MC 자동 분류. 예: `Developer 항온수` |
| _(빈값)_ | → | B2 (작업요소기능) | 사용자가 FMEA에서 작성. 예: `유지`, `관리`, `검사` 등 작업요소의 기능 |
| 공정특성 | → | B3 (공정특성) | 복수 가능, specialChar 포함 |
| _(빈값)_ | → | B4 (고장원인) | 사용자가 FMEA에서 작성 |
| 평가방법 | → | DC 자동추천 참고 | 메타데이터로 저장 |
| 관리방법 | → | PC 자동추천 참고 | 메타데이터로 저장 |

**결과**: CP에서 공정 구조 + 특성 데이터가 FMEA로 전달되어, 사용자는 **고장분석(FM/FC/FE)만 작성**하면 고품질 FMEA가 완성됨.
