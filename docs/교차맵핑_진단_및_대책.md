# B1↔B2↔B3 교차매핑 종합 진단 및 대책

> **작성일**: 2026-02-23
> **대상 시스템**: Smart FMEA (PFMEA 기초정보 Import → PFD/CP 연동)
> **영향 범위**: PFD 공정흐름도, CP 관리계획서, PFMEA 워크시트

---

## 1. 문제 요약

### 1.1 현상

PFMEA 기초정보를 Import한 후 PFD/CP에 데이터를 연동할 때, **작업요소(B1)와 공정특성(B3)/요소기능(B2)이 엉뚱한 WE에 배정**되는 교차매핑 발생.

| 항목 | 건수 | 비율 |
|------|------|------|
| B3↔B1 교차매핑 | **59건** | 46% (전체 B3 129건 중) |
| B2↔B1 교차매핑 | **6건** | 5% (전체 B2 127건 중) |
| FA 시트 내 불일치 | **86행** | 14% (전체 FA 589행 중) |
| 순서 불일치 그룹 | **22개** | 73% (다중WE 그룹 30개 중) |

### 1.2 영향 전파 경로

```
PFMEA Import → B3 교차매핑 (잘못된 WE에 배정)
    ├── PFMEA 워크시트: 고장원인-공정특성 매핑 오류
    ├── CP 연동: processNo+workElement+processChar 부정합
    │   └── 관리항목-설비-관리방법 연결 오류
    └── PFD 연동: 공정특성이 엉뚱한 설비에 표시
        └── 강제 병합으로 L3 상세 손실
```

## 4. 대책 C: 빌더 보강 (구현 1순위)

### 4.1 B2 belongsTo 지원 추가

**파서 (excel-parser.ts)**:
- B3와 동일한 complex template 감지 로직을 B2에도 추가
- `b2WECol` 감지: m4 다음 헤더가 '작업요소'이면 complex template
- `elementFuncsWE: string[]` 필드를 ProcessRelation에 추가
- 행 읽기 시 workElement 값을 `row.extra`로 전달

**flat 변환 (useImportFileHandlers.ts, useImportHandlers.ts, useDataCompare.ts)**:
- B2 flat 변환에 `belongsTo: p.elementFuncsWE?.[i]` 추가

### 4.2 폴백 개선 (nameMatchDistribute)

**현재**: 매칭 안 되면 빈 슬롯에 균등 배분 → 빈 슬롯 없으면 최소 WE에 배분
**개선**: 매칭 안 된 항목에 대한 **4단계 퍼지 매칭** 추가

```
1단계: belongsTo 확정 매칭 (B2/B3 Extended)
2단계: 이름 접두사 매칭 (B2/B3 Simple)
3단계: 장비명 추출 매칭 (번호-장비명-특성 패턴)
4단계: 빈 슬롯 균등 배분 (경고 로그 출력)
```

### 4.3 마지막 WE 몰아넣기 제거

```typescript
// BEFORE: 빈 슬롯 없으면 → 최소 WE에 강제 배치
else {
  const sortedByCount = wes.map((_, idx) => idx)
    .sort((a, b) => result[a].length - result[b].length);
  for (let i = 0; i < unmatched.length; i++) {
    result[sortedByCount[i % sortedByCount.length]].push(unmatched[i]);
  }
}

// AFTER: 빈 슬롯 없으면 → 경고만 (강제 배치 금지)
// 대신 3단계 퍼지 매칭으로 최대한 매칭 시도
```
