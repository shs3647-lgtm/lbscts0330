## 1. 문서 연동 구조 (6.1 구조 연동 기준)

### 1.1 구조 연동 방향 (6개 케이스)

|-------|------|------|----------|

## 8. 핵심 검증 공식

```
[원자성 검증]
CP.rows.length === PFD.rows.length === 108

[공정 매핑 검증]  
Set(CP.processNo) === Set(PFD.processNo) === Set(FMEA.L2.no) === {10, 20, 30}

[특성 개수 검증]
CP.productChar.filter(v => v).length === 21
CP.processChar.filter(v => v).length === 9

[특성 위치 검증]
CP[i].productChar → FMEA L2Function
CP[i].processChar → FMEA L3Function
(절대 반대로 매핑되면 안 됨)

[행별 동일성 검증]
∀i: CP[i].processNo === PFD[i].processNo
∀i: CP[i].productChar === PFD[i].productChar
∀i: CP[i].processChar === PFD[i].processChar
```

## 9. 결론

이 테스트 계획서에 따라 검증을 수행하면:
1. **108행 = 108행** 원자성 보장
2. **8개 필드** 양방향 동기화 확인
3. **제품특성/공정특성** 올바른 위치 확인
4. **rowSpan** 렌더링 정확성 확인

모든 검증 통과 시에만 **연계성 100% 보장** 선언 가능.
