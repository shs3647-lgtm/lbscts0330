## 0. 전체 구조 — 항목 간 역할 분리 원칙

각 항목은 고유한 역할을 가진다. 항목 간 경계가 무너지면 고장사슬 매핑이 실패하고 CP 연계가 불가능해진다.

```
A3 공정기능          →  A4 제품특성          →  A5 고장형태          ⇄  FC 고장사슬          ⇄  B4 고장원인          ←  B3 공정특성
(공정의 행위+결과)      (제품에서 측정하는 특성)  (A4 특성의 이탈 현상)    (FM↔FE↔FC 연결 중심)    (B3 파라미터 이탈 원인)   (설비에서 관리하는 파라미터)

B1 작업요소          →  B2 요소기능          →  B3 공정특성              B5 예방관리  /  A6 검출관리
(설비·재료·인원 고유명칭)  (B1이 수행하는 기능 동사문)                          (발생 전 방지 장치·시스템)  (발생 후 검출 장비+방법+빈도)
```

**핵심 구분 원칙**

- **A4 제품특성**: 제품(Wafer)에서 측정하는 결과값
- **B3 공정특성**: 설비·약품에서 설정·모니터링하는 파라미터(입력값)
- **A5 고장형태**: A4가 이탈한 현상 — 원인 절대 포함 금지
- **B4 고장원인**: B3가 이탈한 이유 — 현상 절대 포함 금지
- **B5 예방관리**: 원인 발생 전 방지
- **A6 검출관리**: 발생 후 검출

## 1. 공정기능 (A3) — 공정이 수행하는 행위와 결과

공정기능은 "이 공정이 무엇을 해서 어떤 결과를 만드는가"를 한 문장으로 기술한다. 파라미터 수치·검사 절차·불량 방지 내용은 포함하지 않는다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의** | `[대상물]을 [처리방법]으로 [처리]하여 [결과]를 확보한다` | 공정의 기여(기능)만 기술. 결과는 제품 상태·품질로 표현 |
| **올바른 예** | ✅ "Wafer 표면의 이물을 DIW로 세정하여 청정도를 확보한다"<br>✅ "PR을 균일하게 도포하여 목표 두께를 확보한다"<br>✅ "입고 Wafer의 표면 상태 및 Lot 정보를 검증한다" | 동사 마무리. 수치·기준 없음 |
| **금지 패턴** | ❌ "1. 서류심사 입고 Lot에 대해서 라벨 및 Run Sheet, 전산, 실물 일치화" → 절차서 문장 나열 금지<br>❌ "UBM Sputter 공정을 정상 진행한다" → "정상 진행한다"는 기능 기술이 아님<br>❌ "불량 발생을 방지하기 위해 세정한다" → 고장 조건 포함 금지 | v6 파일 실제 오류 패턴 |

## 2. 제품특성 (A4) — 제품(Wafer)에서 측정하는 품질특성

A4는 공정 출력물인 Wafer·반제품에서 직접 측정·확인 가능한 품질 특성의 명사다. 설비·약품의 파라미터(온도·농도·에너지 등)는 모두 B3 공정특성이다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의** | Wafer·반제품 표면·형상·물성에서 측정 가능한 품질 특성 명사<br>형식: `[특성 명사] ([영문 용어, 선택])` | 측정 대상 = 제품. 단위 포함 가능 |
| **올바른 예** | ✅ "Bump 높이 (Bump Height)"<br>✅ "PR 도포 두께 (PR Thickness)"<br>✅ "TiW 박막 두께"<br>✅ "Wafer 표면 외관 결함 부재 (Visual Defect Absence)"<br>✅ "Wafer 표면 청정도 (Particle 농도)" | CC: Bump Height, Shear, TiW/Au 두께<br>SC: PR 두께, 청정도 |
| **금지 패턴 — 공정특성 혼입** | ❌ A4: "DI Resistivity" → B3로 이동 (설비 파라미터)<br>❌ A4: "RF Etch amount" → B3로 이동 (공정 파라미터)<br>❌ A4: "Bake Temp" → B3로 이동 (설비 파라미터)<br>❌ A4: "UV Energy" → B3로 이동 (장비 파라미터) | v6 실제 오류: A4에 공정파라미터 혼입 |
| **금지 패턴 — 행위·동사형** | ❌ A4: "Wafer OCR Check" → "OCR ID 인식 정확도"로 수정<br>❌ A4: "Particle Check" → "Wafer 표면 Particle 농도"로 수정<br>❌ A4: "Wafer" (너무 추상적) → 구체적 특성명으로 세분화 | 명사형 특성만 기재 |

**실제 오류 vs 올바른 기재 (v6 파일 기준)**

| ❌ 실제 오류 | ✅ 올바른 기재 |
|------------|--------------|
| A4: "Wafer" (측정 불가, 너무 추상적) | A4: "Wafer 표면 외관 결함 부재 (Visual Defect Absence)"<br>A4: "Wafer 표면 스크래치 부재"<br>A4: "Wafer 파손·크랙 부재" (특성별로 행 분리) |
| A4: "Wafer OCR Check" (검사 행위 → 명사화 필요) | A4: "OCR ID 인식 정확도 (OCR Matching Accuracy)" |
| A4: "Particle Check" (검사 행위 → 명사화 필요) | A4: "Wafer 표면 청정도 (Particle 농도, ea/wfr)" |

## 3. 고장형태 (A5) — A4 제품특성이 이탈한 현상

고장형태는 A4 제품특성이 요구 기준에서 이탈한 결과적 현상이다. "어떻게 되었는가(What)"만 기술하며, 원인(Why)·행위·공정특성 이탈을 포함하지 않는다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의** | `[A4 특성명] + [이탈 유형 어휘]`<br>이탈 유형: 규격 이탈·미달·초과·미형성·부재·잔류·불만족·미검출 | 반드시 A4와 1:1 대응. 원인 절대 포함 금지 |
| **올바른 예** | ✅ "Bump 높이 규격 이탈 (Bump Height Spec Out)"<br>✅ "PR 도포 두께 미달 (PR Thickness Under-spec)"<br>✅ "Au Bump 미형성 (Missing Bump)"<br>✅ "Wafer 표면 외관 결함 미검출 (Visual Defect Escape)"<br>✅ "Wafer Lot 혼입 (Lot Mix)" | 이탈 유형이 텍스트에 내포됨 |
| **금지 패턴 — 원인 혼입** | ❌ "작업표준 미준수" → B4 고장원인으로 이동<br>❌ "설비 PM 미실시" → B4 고장원인으로 이동<br>❌ "Robot Teaching 불량" → B4 고장원인으로 이동<br>❌ "Recipe 오설정" → B4 고장원인으로 이동 | v6 실제 오류: 원인이 A5에 혼입 |
| **금지 패턴 — 공정특성 이탈 혼입** | ❌ "DIW 비저항 이탈" → B4 고장원인 "DI Resistivity 규격 이탈"로<br>❌ "Bake 온도 이탈" → B4 고장원인 "Bake 온도 규격 이탈"로<br>❌ "UV Energy 부족" → B4 고장원인 "UV Energy 기준 미달"로 | 공정특성 이탈 = 원인 → B4 |
| **금지 패턴 — 동사·문장형** | ❌ "Wafer Scratch" → "Wafer 표면 스크래치 발생"으로 이탈 현상 명확화<br>❌ "원자재 불량 유출" → A5가 아닌 C4 고장영향. A5: "Wafer 표면 결함 미검출" | 영문 약어만 쓰면 이탈 유형 불명확 |

**실제 오류 vs 올바른 기재 (v6 파일 기준)**

| ❌ 실제 오류 | ✅ 올바른 기재 |
|------------|--------------|
| A5: "원자재 불량 유출" (고장영향에 해당 → A5 오기재) | A5: "Wafer 표면 외관 결함 미검출 (Visual Defect Escape)"<br>(미검출이 고장형태, 유출은 고장영향 C4) |
| A5: "Wafer Chpping, Crack" (이탈 유형 불명확) | A5: "Wafer 파손·크랙 발생 (Chipping/Crack)" |
| A5: "Lot Mix(WAFER MIX)" (허용 — 단 A4 대응 명확해야 함) | A5: "Wafer Lot 혼입 (Lot Mix)" + A4: "Lot 식별 정확도"와 대응 |

## 4. 작업요소 (B1) — 공정에 직접 관여하는 설비·재료·인원 명칭

작업요소는 공정 수행에 직접 관여하는 구체적 요소의 고유 명칭이다. B2 요소기능·FC 작업요소(WE) 컬럼과 완전히 동일한 표기를 써야 시스템 매핑이 성립한다.

| 4M | 작성 기준 | 실제 오류 → 수정 |
|----|---------|----------------|
| **MC (설비·도구)** | 설비 고유명 또는 모델명 사용<br>✅ "HIGH POWER SCOPE" / "DNS SS-3000" / "Stratus 300"<br>✅ "토크렌치" / "4Point Probe장비"<br>❌ "세정 설비" / "도금 장비" (특정 불가) | ❌ B1: "TEST 장비Technician" → MN(사람)에 해당<br>→ B1: "HIGH POWER SCOPE" (설비)<br>→ "TEST장비 Technician" → MN으로 분류 |
| **MN (사람)** | 직무·역할명으로 특정<br>✅ "품질검사원" / "셋업엔지니어" / "보전원"<br>✅ "Plating공정 Eng`r" / "공정기술자"<br>❌ "작업자" (공통 외 개별 공정 세분화 권장) | ❌ B1: "Particle" → 재료(IM)에 해당하거나 측정 항목(A4)에 해당<br>→ B1: "파티클 카운터 장비"(MC)로 수정 |
| **IM (재료)** | 재료·약품·가스 고유명 사용<br>✅ "Au Etchant" / "TMAH 현상액" / "DIW"<br>✅ "PR (TSMR-8900)" / "CO2 Gas"<br>❌ "화학약품" / "재료" (특정 불가) | ❌ B1에 재료명 없이 파라미터만 기재<br>→ 재료 고유명으로 특정 |

> **B1·B2·FC(WE) 3곳 일치 규칙**: B1에 "HIGH POWER SCOPE"가 있으면 → B2의 ★작업요소(B1) 컬럼과 FC의 작업요소(WE) 컬럼도 반드시 "HIGH POWER SCOPE"로 동일 기재해야 한다. 표기 차이(공백·대소문자 포함)가 있으면 시스템 매핑이 끊긴다.

## 5. 요소기능 (B2) — 작업요소가 공정에서 수행하는 고유 기능

요소기능은 B1 작업요소가 공정에서 수행하는 기능을 동사문으로 기술한다. 수치·고장 조건은 포함하지 않고, 주어(B1)와 결과를 명확히 써야 한다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의·형식** | `[B1 작업요소]이/가 [행위]하여 [결과]를 제공/확보/판정한다` | 주어 = B1 명칭. 결과 명시 필수 |
| **MC 올바른 예** | ✅ "HIGH POWER SCOPE가 Wafer를 Auto Loading하여 외관 검사를 수행한다"<br>✅ "DNS SS-3000이 DIW를 분사하여 Wafer 표면 이물을 제거한다"<br>✅ "Stratus 300이 전류를 인가하여 Au 도금층을 형성한다" | 설비 행위 + 결과 |
| **MN 올바른 예** | ✅ "품질검사원이 검사기준에 따라 수입검사를 진행하고 합부를 판정한다"<br>✅ "셋업엔지니어가 공정 파라미터를 설정하여 초기품을 승인한다"<br>✅ "보전원이 설비 점검 결과를 확인하고 예방보전 적합성을 판정한다" | 직무 행위 + 판정 |
| **금지 패턴** | ❌ "[HIGH POWER SCOPE] Wafer를 High Scope에 Auto Loading하여 Chuck에 Wafer를 위치시킨후 외관검사 진행" → 절차서 문장 나열 금지. 핵심 기능 1문장으로 요약<br>❌ "[품질검사원] 검사기준에 따라 수입검사 진행" → 결과(합부 판정)가 없음<br>❌ "[Particle] 라인 Particle 모니터링을 통해 라인환경 유지 관리" → "Particle"은 B1 작업요소가 될 수 없음 (측정 항목) | v6 실제 오류 패턴 |

## 6. 공정특성 (B3) — 설비·약품에서 관리하는 파라미터 (입력값)

공정특성은 B1 작업요소가 공정 중 설정·모니터링하는 입력 파라미터다. "제품에서 측정하는 결과값"은 A4 제품특성이고, "설비·약품에서 관리하는 조건값"이 B3이다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의·형식** | B1 작업요소가 관리하는 공정 조건의 명칭<br>형식: `[파라미터 명] ([단위])` | 측정 대상 = 설비/약품. 단위 필수 |
| **올바른 예** | ✅ "TMAH 농도 (wt%)"<br>✅ "도금 전류 (A·hr)"<br>✅ "Bake 온도 (°C)"<br>✅ "DI Resistivity (MΩ·cm)"<br>✅ "UV Energy (mJ/cm²)"<br>✅ "RF Etch 량 (nm)" | 공정 레시피·약품·설비 파라미터 |
| **A4 vs B3 구분** | A4 = 제품(Wafer)에서 측정하는 결과값<br>B3 = 설비·약품에서 설정·모니터링하는 조건값<br><br>예) 같은 "두께"라도:<br>→ Wafer에서 측정한 TiW 두께 = A4 제품특성<br>→ 스퍼터 장비의 파워·시간 조건 = B3 공정특성 | 결과 vs 조건으로 구분 |
| **금지 패턴** | ❌ B3: "[품질검사원] 검사기준에 따라 수입검사 진행" → B2 요소기능 내용이 B3에 혼입. B3은 파라미터 명사만 기재<br>❌ B3: "[HIGH POWER SCOPE] Visual Defect" → Visual Defect는 A4 제품특성<br>❌ B3: "유지보수 정확도", "취급 조건" → 측정 불가한 행위 기술. 가능하면 구체화하거나 삭제 | v6 실제 오류: B2 내용 및 A4 항목이 B3에 혼입 |

**실제 오류 vs 올바른 기재 (v6 파일 기준)**

| ❌ 실제 오류 | ✅ 올바른 기재 |
|------------|--------------|
| B3: "[품질검사원] 검사기준에 따라 수입검사 진행" (B2 요소기능 내용이 B3에 혼입) | B3: "검사 합격 기준 (AQL 등급)"<br>또는 해당 공정에 관리해야 할 파라미터 없으면 공란 처리 |

## 7. 고장원인 (B4) — B3 공정특성이 이탈한 직접적 원인

고장원인은 B3 공정특성 파라미터가 기준에서 이탈하게 된 직접적 원인이다. 고장형태(A5)나 대책·행동을 기술하지 않는다.

| 항목 | 기준 및 형식 | 비고 |
|------|-------------|------|
| **정의·형식** | `[B3 파라미터] [이탈 유형] + 원인 메커니즘`<br>또는 `[B1 요소]의 [물리적 변화]에 의한 [파라미터 이탈]` | 파라미터 이탈 원인. 행위 기술 금지 |
| **MC 올바른 예** | ✅ "Robot Teaching 위치 이탈에 의한 반송 오류"<br>✅ "Wafer 카세트 Loading 위치 편차"<br>✅ "설비 PM 미실시에 의한 Pump 성능 저하"<br>✅ "Lamp Life Time 초과에 의한 UV 조사량 저하" | 물리적 원인. 명사형 기술 |
| **MN 올바른 예** | ✅ "Recipe 오적용 (제품별 조건 설정 혼동)"<br>✅ "공정 누락 (PR Coating Skip)"<br>✅ "작업표준 미숙지에 의한 처리 조건 이탈"<br>✅ "Run Sheet/Wafer 라벨 매칭 실수에 의한 Lot 혼입" | 행동이 아닌 판단·절차 오류 |
| **금지 패턴 — 고장형태 혼입** | ❌ B4에 "Wafer Scratch" → A5 고장형태<br>❌ B4에 "Bump 미형성" → A5 고장형태<br>❌ B4에 "Lot Mix" → A5 고장형태<br>→ B4는 "왜 그 현상이 발생했는가"의 원인만 기재 | v6 실제 오류: 고장형태가 B4에 혼입 |

## 8. 예방관리 (B5) / 검출관리 (A6) — 발생 전 방지 vs 발생 후 검출

예방관리(B5)는 고장원인 발생을 사전에 방지하는 시스템·장치·절차다. 검출관리(A6)는 고장이 발생한 후 출하 전에 발견하는 검사·측정 방법이다. 두 항목이 혼재되는 것이 가장 흔한 오류다.

| 구분 | 정의 및 작성 기준 | 실제 오류 → 수정 |
|------|----------------|----------------|
| **B5 예방관리 (Prevention)** | 정의: B4 고장원인 발생을 사전에 방지하기 위한 현재 관리 방법<br>형식: `[구체적 장치명·시스템명·절차명] + [작동 방식]`<br><br>✅ "Robot Teaching 주기적 점검 (분기 1회)"<br>✅ "Run Sheet 라벨·Wafer 캐리어 라벨 바코드 매칭 시스템"<br>✅ "공정 Recipe 이중 확인 (MES 전산 인터록)"<br>✅ "작업자 인증 시험 (정기 숙련도 평가)" | ❌ B5: "작업자 작업 절차 평가 진행" → 올바름 (단, 구체화)<br>→ ✅ B5: "작업자 인증 시험 및 정기 숙련도 평가"<br><br>❌ B5: "분기 점검 시 Robot Teaching 상태 check"<br>→ ✅ B5: "Robot Teaching 상태 분기 정기 점검" |
| **A6 검출관리 (Detection)** | 정의: 고장 발생 후 출하 전 검출을 위한 검사·측정 방법<br>형식: `[검출 장비명·방법] + [검사 빈도/방식]`<br><br>✅ "Machine Interlock (자동 검출, 실시간)"<br>✅ "MES Interlock (전산 자동 차단)"<br>✅ "Micro Scope 전수 육안검사 (AQL 1.5 G-Ⅰ)"<br>✅ "4Point Probe 13points/wfr Daily" | ❌ A6: "Wafer 검사" → 장비명·빈도 없음<br>→ ✅ A6: "HIGH POWER SCOPE 전수 외관검사 (Every lot)"<br><br>❌ A6: "Visual Inspection" → 장비명·빈도 없음<br>→ ✅ A6: "Micro Scope 육안검사 (AQL 1.5 G-Ⅰ, Every lot)" |
| **혼재 오류 (B5↔A6)** | 구분 기준:<br>B5 = 원인이 발생하기 **전에** 막는 것<br>A6 = 이미 발생한 것을 **잡는** 것<br><br>❌ A6에 "작업자 교육, SOP 게시" 기재 → B5로 이동<br>❌ A6에 "예방보전 계획 수립" 기재 → B5로 이동<br>❌ B5에 "캘리퍼스로 측정" 기재 → A6로 이동 | v6 실제 오류:<br>❌ B5: "작업대기 Shelf의 측면 배치를 통한 작업 시야각 확보 및 작업동선 최소화 개선"<br>→ 이것은 B5 예방관리 맞음 (개선 조치). 단, 간결하게 요약 필요 |

## 9. Control Plan 연계 — PFMEA 항목이 CP에 반영되는 방법

PFMEA 기초정보를 정확하게 작성하면, Control Plan의 CHARACTERISTICS·측정기법·관리방법·이상반응계획이 PFMEA에서 직접 도출된다. 중복 작성 없이 PFMEA-CP 양방향 추적이 가능해진다.

```
PFMEA 기초정보                      →  Control Plan 연동                →  효과
1. A4 제품특성 (CC/SC)                 1. CP Product 특성 컬럼             1. PFMEA 특성 → CP 자동 반영
2. B3 공정특성 (CC/SC)                 2. CP Process 특성 컬럼             2. 파라미터 측정 방법 중복 제거
3. A6 검출관리 (장비+방법+빈도)         3. CP 측정기법 컬럼                 3. 관리 항목 누락 방지
4. B5 예방관리 (시스템+장치)            4. CP 관리방법 컬럼                 4. 이상 조치 기준 일관성 확보
5. FC AP (H/M/L)                       5. CP 이상 반응계획 컬럼            5. PFMEA-CP 양방향 추적 가능
```

| PFMEA 항목 | CP 컬럼명 | AU Bump 적용 예시 | 측정·관리 방법 | 연계 원칙 |
|-----------|---------|----------------|--------------|---------|
| **A4 제품특성 (CC/SC)** | CHARACTERISTICS Product 컬럼 | Visual Defect<br>TiW Thickness (◇)<br>Au Thickness (◇)<br>Bump Height<br>Bump Co-planarity | Micro Scope<br>4Point Probe<br>4Point Probe<br>GTX8<br>GTX8 | A4의 CC/SC 항목 → CP Product 컬럼 그대로 이관 |
| **B3 공정특성 (CC/SC)** | CHARACTERISTICS Process 컬럼 | DI Resistivity<br>RF Etch amount (◇)<br>Bake Temp<br>TMAH Concentration<br>UV Energy | Machine interlock<br>Nanospec<br>Machine interlock<br>Document (COA)<br>Intensity Meter | B3의 CC/SC 항목 → CP Process 컬럼 그대로 이관 |
| **A6 검출관리** | EVALUATION/MEASUREMENT TECHNIQUE | Microscope, Naked eye<br>Machine interlock<br>4Point Probe<br>AVI (Spotlight)<br>Nanospec | → CP 측정기법 컬럼과 직접 연동 | A6 장비명·방법 → CP 측정기법. 중복 작성 불필요 |
| **B5 예방관리** | Control Method | Run Sheet 라벨 매칭<br>SPC Chart (X-bar R)<br>Trend Chart<br>MES Interlock<br>PM Sheet | → CP 관리방법 컬럼과 직접 연동 | B5 관리 시스템 → CP Control Method. 중복 작성 불필요 |
| **FC AP (조치우선순위)** | REACTION PLAN Action 컬럼 | AP=H: Lot Hold → 즉시 격리 + 고객 통보<br>AP=M: 공정 중단 검토 + ABN 발행<br>AP=L: 이상 보고 후 모니터링 강화 | → CP 이상 반응계획 기준이 됨 | AP 등급 기준이 CP 이상 반응 강도 결정 |

> **CP 작성 원칙**: ① A4의 CC·SC 항목 → CP Product 컬럼 그대로 이관 ② B3의 CC·SC 항목 → CP Process 컬럼 그대로 이관 ③ A6 장비명+방법 → CP 측정기법 컬럼 그대로 이관 ④ B5 관리시스템 → CP 관리방법 컬럼 그대로 이관. 이 원칙을 따르면 PFMEA-CP 간 항목 중복 작성과 누락을 동시에 방지할 수 있다.

### 13. PFMEA→CP 직접 연동 (syncPfmeaCP.ts)

`syncPfmeaToCP` 함수에서 공정특성 행 생성 시 riskData의 B5/A6 값을 CP에 직접 매핑.

| PFMEA 필드 | riskData 키 | CP 필드 | 설명 |
|-----------|------------|---------|------|
| B5 예방관리 | `prevention-{idx}` | `controlMethod` | PFMEA B5 → CP 관리방법 |
| A6 검출관리 | `detection-{idx}` | `measureMethod` | PFMEA A6 → CP 평가측정방법 |

> 기존 CP→PFMEA 역방향(`controlMethod` → `prevention`/`detection`)은 그대로 유지.  
> 이제 양방향 자동 동기화로 PFMEA-CP 간 중복 작성 제거.
