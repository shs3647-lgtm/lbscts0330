## 1. 개요

### 1.1 문서 목적

PFMEA와 DFMEA의 기초정보 입력 시트 구조, 관계형 데이터 정의, Excel 다운로드 시트 명명 규칙을 정의합니다.

### 1.2 핵심 차이점

| 구분 | PFMEA (공정FMEA) | DFMEA (설계FMEA) |
|------|------------------|------------------|
| **분석 대상** | 제조 공정 | 제품 설계 |
| **L2 기준번호** | 공정번호 | 초점요소번호 |
| **L3 기준번호** | 공정번호 (L2와 동일) | 초점요소번호 (L2와 동일) |
| **L1 구분** | YOUR PLANT / SHIP TO PLANT / USER | 법규 / 기능 / 보조 / 관능 |
| **L3 내용** | 작업요소 (4M: Man/Machine/Material/Environment) | 부품/컴포넌트 |

### 1.3 데이터 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 기초정보 입력 (Excel Import)                                 │
│    └── Flat 데이터 (processNo/focusNo 기준)                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. 구조분석 → 기능분석 → 고장분석                               │
│    └── 1:n:n 계층 관계 설정                                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. 리스크분석                                                   │
│    └── 예방관리 + 검출관리 + S/O/D → AP                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. 관계형 기초정보 다운로드                                     │
│    └── 분석 결과 기반 관계형 Excel 생성                         │
└─────────────────────────────────────────────────────────────────┘
```

## 2. PFMEA 기초정보 정의

### 2.1 레벨별 항목 구성

#### L2 레벨 (공정 - Focus Element)

| 코드 | 시트명 | 항목명 | 설명 | 필수 |
|------|--------|--------|------|------|
| A1 | L2-1 공정번호 | 공정번호 | 공정 식별 번호 (기준번호) | ✅ |
| A2 | L2-2 공정명 | 공정명 | 공정 이름 | ✅ |
| A3 | L2-3 공정기능 | 공정기능(설명) | 공정의 기능/목적 | |
| A4 | L2-4 제품특성 | 제품특성 | 공정 결과물의 품질 특성 | |
| A5 | L2-5 고장형태 | 고장형태 | 공정 기능 미수행/부분수행 | |
| A6 | L2-6 검출관리 | 검출관리 | 고장 검출 방법 | |

#### L3 레벨 (작업요소 - Next Lower Level)

|------|--------|--------|------|------|
| B1 | L3-1 작업요소 | 작업요소(설비) | 4M 작업요소 (Man/Machine/Material/Environment) | |
| B2 | L3-2 요소기능 | 요소기능 | 작업요소의 기능 | |

### 2.3 PFMEA L1 구분값

| 구분 | 코드 | 설명 | 예시 |
|------|------|------|------|
| **YP** | YP | Your Plant (자사 공장 내 영향) | 재작업, 폐기, 라인 정지 |
| **SP** | SP | Ship to Plant (고객사/조립공장 영향) | 반품, 클레임, 라인 정지 |
| **USER** | USER | User (최종 사용자 영향) | 안전 사고, 성능 저하, 불만족 |

## 4. 관계형 데이터 구조

### 4.1 계층 관계 (1:n:n)

```
┌─────────────────────────────────────────────────────────────────┐
│ L1 (상위수준)                                                   │
│ ├── 구분1: YOUR PLANT / 법규                                    │
│ │   ├── 기능1                                                   │
│ │   │   └── 요구사항1 → 고장영향1                               │
│ │   └── 기능2                                                   │
│ │       └── 요구사항2 → 고장영향2                               │
│ └── 구분2: SHIP TO PLANT / 기능                                 │
│     └── 기능3                                                   │
│         └── 요구사항3 → 고장영향3                               │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼ (1:n)
┌─────────────────────────────────────────────────────────────────┐
│ L2 (초점요소/공정)                                              │
│ ├── 공정10 / 초점요소10                                         │
│ │   ├── 기능1 → 특성1 → 고장형태1 → 검출관리1                   │
│ │   ├── 기능2 → 특성2 → 고장형태2 → 검출관리2                   │
│ │   └── 기능3 → 특성3 → 고장형태3 → 검출관리3                   │
│ └── 공정20 / 초점요소20                                         │
│     └── 기능4 → 특성4 → 고장형태4 → 검출관리4                   │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼ (1:n)
┌─────────────────────────────────────────────────────────────────┐
│ L3 (작업요소/부품)                                              │
│ ├── 작업요소1 / 부품1                                           │
│ │   ├── 기능1 → 특성1 → 고장원인1 → 예방관리1                   │
│ │   └── 기능2 → 특성2 → 고장원인2 → 예방관리2                   │
│ └── 작업요소2 / 부품2                                           │
│     └── 기능3 → 특성3 → 고장원인3 → 예방관리3                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 고장 연결 (FC → FM → FE)

```
고장원인(FC) ──────→ 고장형태(FM) ──────→ 고장영향(FE)
   (L3)                 (L2)                 (L1)
                          │
                          ▼
                   ┌─────────────┐
                   │ FailureLink │
                   ├─────────────┤
                   │ fcId        │
                   │ fmId        │
                   │ feId        │
                   └─────────────┘
                          │
                          ▼
                   ┌─────────────┐
                   │ RiskAnalysis│
                   ├─────────────┤
                   │ severity    │
                   │ occurrence  │
                   │ detection   │
                   │ ap          │
                   │ prevention  │
                   │ detection   │
                   └─────────────┘
```
