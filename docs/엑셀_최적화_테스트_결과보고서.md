| 5 | `page.tsx` | exportFunctionL3() ①번 호출 | `await` 추가 |

**효과**: Dynamic import(`await import()`) 반환값이 정상적으로 resolve 되어야 함수가 실행됨. `await` 없으면 Promise가 무시되어 "다운 안 돼" 현상 발생.

### 3-2. try-catch 에러 핸들링 (7건) ✅

| # | 파일 | 함수 | 에러 시 동작 |
|---|------|------|------------|
| 1 | `excel-export.ts` | exportFunctionL1/L2/L3 (호출부) | `console.error` + `alert` |
| 2 | `excel-export-all.ts` | exportAllSheetsExcel() | `console.error` + `alert` |
| 3 | `exportFmea4Excel.ts` | exportFmea4Excel() | `console.error` + `alert` |
| 4 | `exportCPExcel.ts` | exportCPExcel() | `console.error` + `alert` |
| 5 | `exportChainErrors.ts` | exportChainErrorsExcel() | `console.error` + `alert` |
| 6 | `page.tsx` | Excel 내보내기 switch 전체 | `console.error` |
| 7 | `useExcelHandlers.ts` | handleWorksheetExport/handleFunctionExport | `console.error` + `alert` |

**효과**: Dynamic import 실패, ExcelJS 초기화 에러, 데이터 변환 에러 등 모든 예외를 포착하여 사용자에게 알림.

### 3-3. Worker 폴백 메커니즘 ✅

```
exportAllViewExcel() 실행
  │
  ├─ Phase 1: 데이터 준비 (메인 스레드, ~750ms)
  │
  ├─ Phase 3: Worker 시도
  │   ├─ ✅ 성공 → buffer 수신 → saveExcelFile()
  │   │
  │   └─ ❌ 실패 (Worker 미지원 / 타임아웃 60초 / ExcelJS 로드 실패)
  │       └─ buildWorkbookFromData() 폴백 (메인 스레드)
  │           ├─ 공유 스타일 객체 사용 (GC 최소화)
  │           ├─ yieldToMain() 주기적 호출
  │           └─ writeBuffer() 후 saveExcelFile()
  │
  └─ ❌ 전체 실패 → console.error + alert
```

**폴백 시나리오**:
- IE11 / 구형 브라우저 (Worker 미지원) → 메인 스레드 폴백
- Worker 내부 ExcelJS 로드 실패 → 에러 메시지 + 메인 스레드 폴백
- Worker 60초 타임아웃 → 폴백
- **모든 경우에 사용자에게 결과 제공 보장**
