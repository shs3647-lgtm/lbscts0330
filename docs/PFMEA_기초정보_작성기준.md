> **문서 버전**: v1.3 (2026-03-12)
> **적용 표준**: AIAG-VDA FMEA Handbook 1st Edition (2차 정오표)
> **시스템**: FMEA OnPremise v5.4.0+ (ctscore 최적화 버전)

## 1. 기초정보 시트 구조 개요

PFMEA 기초정보는 **3개 레벨 × 15개 시트**로 구성된다.

```
C레벨 (L1) — 고객/시스템 관점
  L1-1(C1) 구분           : YP / SP / USER 분류
  L1-2(C2) 제품기능        : 레벨별 제품(반)기능
  L1-3(C3) 요구사항 ★      : 레벨별 요구사항 (한 행 = 요구사항 하나)
  L1-4(C4) 고장영향        : 고장영향 기술

A레벨 (L2) — 공정 관점
  L2-1(A1) 공정번호        : 공정번호 + 공정명 + 공정유형코드
  L2-2(A2) 공정명          : 공정명 목록
  L2-3(A3) 공정기능        : 공정기능 설명
  L2-4(A4) 제품특성        : 제품특성 + 특별특성 기호
  L2-5(A5) 고장형태        : 공정별 고장형태 목록

B레벨 (L3) — 작업요소/원인 관점
  L3-1(B1) 작업요소        : 설비·도구·인원 목록 (4M 분류)
  L3-2(B2) 요소기능 ★      : 작업요소 기능 + ★작업요소(B1) 매핑
  L3-3(B3) 공정특성        : 공정특성(파라미터) + ★작업요소(B1) 매핑
  L3-4(B4) 고장원인        : 고장원인 목록

고장사슬/검증
  FC 고장사슬 ★            : FM × FC 조합 완전 고장사슬 (12열, B5·A6 포함)
  VERIFY                   : I→D 검증 시트 (Import vs DB 카운트 일치 확인)
```

> **v1.1 변경**: FA 통합분석 미리보기 탭 삭제 → Import SA/FC 2단계 확정으로 간소화.
> SOD·AP 분석은 워크시트 ALL탭에서 직접 수행.
> **v1.2 변경**: FC 컬럼 13→12열 (S 삭제, v5.1 반영). 모드별 템플릿 시트 구성 추가.
> **v1.3 변경**: B5(예방관리)·A6(검출관리) Import 필수 명시. 탭 명칭 통일 (수동템플렛/자동템플렛/기존데이타). 모드별 샘플 자동 연동 반영.

## 2. 모드별 템플릿 시트 구성

### 2.1 수동템플렛 (Manual)

구조+기능+고장영향까지만 포함. 고장분석·FC는 워크시트에서 직접 입력.

| 시트 | 포함 |
|------|------|
| A1 공정번호, A2 공정명, A3 공정기능 | ✅ |
| B1 작업요소, B2 요소기능 | ✅ |
| C1 구분, C2 제품기능, C3 요구사항, C4 고장영향 | ✅ |
| **A4 제품특성, A5 고장형태** | ❌ 제외 |
| **B3 공정특성, B4 고장원인** | ❌ 제외 |
| **FC 고장사슬** | ❌ 제외 |
| **FA 통합분석** | ❌ 제외 |

> **총 9시트** — 공정 구조와 기본 기능만 정의
> **샘플Down**: 9시트 샘플만 다운로드됨

### 2.2 자동템플렛 / 기존데이타 (Auto / Existing)

전체 시트 포함. **FC 고장사슬이 반드시 포함**되어야 한다.

| 시트 | 포함 |
|------|------|
| A1~A5 (공정번호~고장형태) | ✅ |
| B1~B4 (작업요소~고장원인) | ✅ |
| C1~C4 (구분~고장영향) | ✅ |
| **FC 고장사슬** (B5·A6 포함) | ✅ **필수** |
| VERIFY | ✅ |

> **총 15시트** — FC 없이는 고장사슬 자동 구축 불가
> **샘플Down**: 15시트 전체 샘플이 다운로드됨

### 2.3 모드별 샘플 자동 연동 (v1.3 신규)

```
★ 탭 전환 시 샘플Down 내용이 자동으로 변경된다:
  - 수동템플렛 탭 → 샘플Down = 9시트 (FC 제외)
  - 자동템플렛 탭 → 샘플Down = 15시트 (FC 포함)
  - 기존데이타 탭 → DB 데이터 있으면 기존 Export, 없으면 15시트 샘플
```

### 2.4 다운로드 시 주의사항

```
⚠️ 수동템플렛에서 다운로드하면 FC 시트가 빠진다!
   고장사슬이 필요하면 반드시 "기존데이타" 또는 "자동템플렛" 탭에서 다운로드할 것.

⚠️ 기존 데이터 Export 시 failureChains 파라미터가 전달되어야 FC 시트가 포함된다.
```

## 3. B5(예방관리)·A6(검출관리) 작성 및 Import 기준 (v1.3 신규)

### 3.1 핵심 원칙

> **B5(예방관리)와 A6(검출관리)는 FC 고장사슬 시트의 8번·9번 컬럼에 직접 작성한다.**
> 별도 시트가 아니라 FC 시트 내 컬럼으로 통합되어 있으므로,
> **FC 작성 시 반드시 함께 기입**해야 한다.

### 3.2 B5(예방관리) 작성 기준

| 항목 | 설명 |
|------|------|
| **정의** | 해당 고장원인(FC)의 발생을 **사전에 방지**하기 위한 현재 관리 방법 |
| **작성 위치** | FC 고장사슬 시트 8번 컬럼 (B5.예방관리) |
| **작성 원칙** | 구체적 장치명·시스템명·절차명을 명시 (추상적 표현 금지) |

**예시**:
| 고장원인(FC) | ❌ 나쁜 예 | ✅ 올바른 예 |
|-------------|-----------|------------|
| 절단 속도 불균일 | 관리한다 | 절단 속도 자동 제어 시스템, 실시간 모니터링 알람 |
| 금형 위치 이탈 | 점검한다 | 포카요케 가이드 핀 설치, 금형 고정 상태 점검 |
| 작업 실수 | 교육 | 작업자 숙련도 평가 및 정기 교육, SOP 현장 게시 |

### 3.3 A6(검출관리) 작성 기준

| 항목 | 설명 |
|------|------|
| **정의** | 고장이 발생한 후 **출하 전에 검출**하기 위한 현재 검사·시험 방법 |
| **작성 위치** | FC 고장사슬 시트 9번 컬럼 (A6.검출관리) |
| **작성 원칙** | 검출 장비명 + 검사 방법(전수/샘플링/Lot별) 명시 |

**예시**:
| 고장형태(FM) | ❌ 나쁜 예 | ✅ 올바른 예 |
|-------------|-----------|------------|
| 소재 길이 부적합 | 측정 | 측정기기(캘리퍼스, 줄자) 전수 측정 |
| BURR 과다 | 외관검사 | 육안검사 전수 + 자동광학검사(AOI) |
| 용접강도 미달 | 시험 | 인장시험기 Lot별 파괴검사 + 비파괴검사(NDT) |

### 3.4 Import 시 반영 원칙

```
★★★ B5·A6는 Import 시 FC 시트와 함께 반드시 반영되어야 한다 ★★★

1. FC 고장사슬 시트 작성 시 B5(8번 컬럼)·A6(9번 컬럼)을 반드시 기입
2. Import 시 B5·A6 값이 FC 시트에서 자동으로 읽혀 DB에 저장됨
3. 워크시트 ALL탭에서 O(발생도)·D(검출도) 자동 추천의 근거가 됨
4. B5·A6가 비어 있으면 O·D 자동 추천이 불가하므로 반드시 작성할 것

⚠️ B5·A6는 별도 시트가 아님 — FC 고장사슬 시트의 컬럼이므로
   FC 시트를 작성할 때 함께 채워야 한다.
```

### 3.5 B5·A6 작성 체크리스트

```
[ ] 모든 FC 행에 B5(예방관리)가 기입되어 있는가?
[ ] 모든 FC 행에 A6(검출관리)가 기입되어 있는가?
[ ] B5에 구체적 장치명·시스템명이 명시되어 있는가? (추상적 표현 아닌지)
[ ] A6에 검출 장비명 + 검사 방법(전수/샘플링/Lot별)이 명시되어 있는가?
[ ] O(발생도) 값이 B5 예방관리 수준과 부합하는가?
[ ] D(검출도) 값이 A6 검출관리 수준과 부합하는가?
```

## 4. L1-3(C3) 요구사항 작성기준

### 4.1 핵심 원칙

> **한 행 = 요구사항 하나**
> 여러 요구사항을 한 셀에 쉼표로 나열하지 않는다.
> 각 요구사항은 독립적으로 추적·관리 가능하도록 행을 분리한다.

### 4.2 구분(C1) 별 요구사항 분류 기준

| 구분 | 정의 | 작성 대상 |
|------|------|---------|
| **YP** | 자사(YourPlant) 내부 요구사항 | 공정 내 품질 기준, 작업 기준, 내부 규격 |
| **SP** | 고객(Supplier/완성차업체) 요구사항 | 납품 제품 규격, 고객 도면 요건, 완성차 강도·외관 기준 |
| **USER** | 최종사용자·법적·산업표준 요구사항 | 안전법규, KS/ISO 표준, 환경 요건, 암시적 요구사항 |

### 4.3 산업표준·법적요구사항 → USER 매핑 원칙

> **법적 요구사항과 산업표준은 반드시 USER 구분으로 등록**한다.

| 요구사항 유형 | 구분 | 예시 |
|-------------|------|------|
| 법적 안전 기준 | **USER** | KS R ISO 4210 자전거 안전요건 |
| 법적 내구 기준 | **USER** | KS R ISO 4210-6 피로시험 |
| 산업 규범 | **USER** | ISO 9001, VDA 6.3, SAE J1739 |
| 기능 안전 | **USER** | ISO 26262 |
| 환경 규제 | **USER** | 재활용 적합 설계, 유해물질 제한 |
| 암시적 요구사항 | **USER** | 금지 재료로부터의 자유, 오용 안전성 |
| 명시적 고객 시방 | **USER** | 고객이 명시한 모든 특정 조건 |

### 4.4 요구사항 작성 예시 (자전거 프레임)

| 구분 | 요구사항 (1행 1건) | 출처 |
|------|----------------|------|
| YP | 이종 재질 혼입 방지 (재질 성적서 확인) | 내부 요구사항 |
| YP | 입고 자재 치수 오차 ±허용공차 이내 | 내부 요구사항 |
| YP | 용접 불량(크랙·기공) 발생 방지 | 내부 요구사항 |
| YP | 열처리 온도·시간 조건 이탈 방지 | 내부 요구사항 |
| SP | 완성차 프레임 강도 규격 준수 | 고객 요구사항 |
| SP | 완성차 외관 품질 규격 준수 | 고객 요구사항 |
| USER | 안전 기준 KS R ISO 4210 준수 (자전거 안전요건) | 법적 요구사항 / 산업표준 |
| USER | 내구 기준 KS R ISO 4210-6 피로시험 합격 | 법적 요구사항 / 산업표준 |
| USER | ISO 9001 품질경영시스템 요구사항 준수 | 산업표준 |
| USER | VDA 6.3 프로세스 심사 요구사항 준수 | 산업표준 |
| USER | 재활용 적합 환경 친화적 제품 설계 요건 | 법적 요구사항 |
| USER | 작업자 잠재적 오용 발생 시 안전 확보 (불연성 등) | 법적 요구사항 |
| USER | 명시적 고객 시방 준수 (모든 특정 조건) | 고객 요구사항 |
| USER | 암시적 요구사항 — 금지된 재료로부터의 자유 | 고객 요구사항 |

### 4.5 작성 금지 패턴

| 금지 | 올바른 방법 |
|--------|------------|
| `이종 재질, 치수 오차, 셋업 실수` (1행에 여러 요구사항) | 각각 별도 행으로 분리 |
| `안전 기준(KS R ISO 4210), 내구 기준` (USER를 SP로 등록) | USER 구분으로 등록 |
| `완성차 강도·외관 규격` (너무 포괄적) | 강도 규격, 외관 규격 각각 분리 |

## 5. L3-2(B2) 요소기능 — 작업요소(B1) 매핑 기준

### 5.1 작업요소(B1) 컬럼의 역할

> B2 요소기능 시트의 **★작업요소(B1) 컬럼**은
> L3-1(B1) 작업요소 시트의 설비·도구·인원과 **1:1 매핑**되어야 한다.
> 이 매핑이 없으면 고장사슬(FC) 생성 시 **작업요소(WE) 연결이 불가**하다.

### 5.2 매핑 원칙

```
L3-1(B1) 작업요소 시트의 "L3-1.작업요소(설비)" 값
    ↕ 1:1 매핑
L3-2(B2) 요소기능 시트의 "★작업요소(B1)" 값
    ↕ 연결
FC 고장사슬의 "작업요소(WE)" 값
```

- 공정번호가 같은 행은 B1의 **등록 순서대로** 매핑한다
- 공통(MN) 행은 셋업엔지니어·작업자·운반원·보전원·검사원 순서로 매핑한다
- ★작업요소(B1) 값은 B1 시트에 등록된 값과 **완전히 동일**해야 한다

### 5.3 공정별 B1↔B2 매핑표 (자전거 프레임)

| 공정번호 | 4M | B1 작업요소 | B2 요소기능 |
|---------|----|-----------|-----------|
| 10 | MC | 지게차 | 원자재를 하역하고 지정 위치로 운반한다 |
| 10 | MC | 자재창고 랙 | 입고 자재를 지정 위치에 적재 보관한다 |
| 20 | MC | 만능인장시험기 | 알루미늄 파이프의 인장강도를 측정한다 |
| 20 | MC | 마이크로미터 | 파이프 외경 및 두께를 정밀 측정한다 |
| 30 | MC | CNC 절단기 | 파이프를 설계 치수에 맞게 자동 절단한다 |
| 40 | MC | TIG 용접기 | 프레임 접합부를 TIG 용접하여 접합한다 |
| 40 | MC | 용접 지그 | 프레임 파이프를 설계 각도로 고정한다 |
| 50 | MC | 열처리로 | 용접 후 잔류응력을 제거하고 경도를 확보한다 |
| 60 | MC | 분체도장 부스 | 정전 분체도장으로 프레임 표면을 코팅한다 |
| 60 | IM | 분체 도료 | 프레임 표면에 분체 도료를 공급한다 |
| 70 | MC | 토크렌치 | 규정 토크로 볼트를 체결한다 |

## 6. FC 고장사슬 작성기준

### 6.1 고장사슬 구조

```
FE (고장영향, C레벨)          <- L1-4(C4) 고장영향
  +- FM (고장형태, A레벨)     <- L2-5(A5) 고장형태
       +- FC (고장원인, B레벨) <- L3-4(B4) 고장원인 + L3-1(B1) 작업요소
            +- B5 (예방관리)   <- FC 시트 8번 컬럼 ★ 필수 작성
            +- A6 (검출관리)   <- FC 시트 9번 컬럼 ★ 필수 작성
```

> **1개 FM에 복수의 FC가 연결**될 수 있다.
> 각 FC는 해당 **작업요소(WE, B1)와 반드시 연결**되어야 한다.
> **각 FC 행에 B5(예방관리)와 A6(검출관리)를 반드시 기입**해야 한다.

### 6.2 고장사슬 12개 컬럼 정의 (v5.1 — S 삭제)

| # | 컬럼 | 내용 | 출처 시트 | 필수 |
|---|------|------|---------|------|
| 1 | FE구분 | YP / SP / USER | L1-1(C1) | |
| 2 | FE(고장영향) | 고장 발생 시 영향 | L1-4(C4) | |
| 3 | L2-1.공정번호 | 해당 공정번호 | L2-1(A1) | ★ |
| 4 | FM(고장형태) | 고장형태 | L2-5(A5) | ★ |
| 5 | 4M | Man/Machine/Material | B1 4M 분류 | |
| 6 | 작업요소(WE) | 작업요소명 | L3-1(B1) | ★ |
| 7 | FC(고장원인) | 고장원인 | L3-4(B4) | ★ |
| 8 | **B5.예방관리** | **현재 예방관리** | **FC 직접 입력** | **★ 필수** |
| 9 | **A6.검출관리** | **현재 검출관리** | **FC 직접 입력** | **★ 필수** |
| 10 | O | 발생도(1~10) | AIAG-VDA 기준 | |
| 11 | D | 검출도(1~10) | AIAG-VDA 기준 | |
| 12 | AP | 조치우선순위 H/M/L | SOD 조합 | |

> **v5.1 변경**: S(심각도) 컬럼 삭제. S는 FE(고장영향) 레벨에서 평가하므로 FC 시트에서 제외.
> **★ B5(예방관리)·A6(검출관리)는 별도 시트가 아닌 FC 시트 8·9번 컬럼으로 통합.**
> **Import 시 FC 시트에서 자동으로 읽혀 DB에 저장되므로, 반드시 FC 작성 시 함께 기입해야 한다.**

### 6.3 고장사슬 작성 규칙

```
1. FM 하나에 관련된 모든 FC를 빠짐없이 작성한다
2. 각 FC에 반드시 작업요소(WE)를 연결한다 — B1 시트 값과 동일해야 함
3. 같은 FM이라도 FC가 다르면 별도 행으로 작성한다
4. USER 구분 FM은 S=9~10 부여 검토 (안전·규제 관련)
5. ★ 예방관리(B5)에 구체적 장치명·시스템명·절차명 명시 (추상적 표현 금지)
6. ★ 검출관리(A6)에 검출 장비명 + 검사 방법(전수/샘플링/Lot별) 명시
7. 공정번호 기준 정렬 (Process-first) — FM을 중심축으로 FE↔FC 연결
8. B5·A6가 비어 있으면 워크시트에서 O·D 자동 추천이 불가하므로 반드시 작성
```

## 10. 주요 오류 체크리스트

```
L1-3(C3) 요구사항
[ ] 한 행에 여러 요구사항이 쉼표로 나열되어 있지 않은가?
[ ] 법적 요구사항·산업표준이 USER 구분으로 등록되어 있는가?
[ ] KS/ISO/SAE 표준 번호가 명시되어 있는가?
[ ] 암시적 요구사항(금지재료, 오용 안전 등)이 누락되지 않았는가?

L3-2(B2) 요소기능
[ ] ★작업요소(B1) 컬럼이 모두 채워져 있는가?
[ ] B1 시트의 작업요소명과 완전히 동일한 값인가?
[ ] 공정번호별 순서가 B1 시트와 일치하는가?

L3-3(B3) 공정특성
[ ] ★작업요소(B1) 컬럼이 모두 채워져 있는가?
[ ] 공정특성을 관리하는 설비·장치명이 B1 값과 일치하는가?

FC 고장사슬
[ ] 모든 FM에 대해 FC가 최소 1개 이상 연결되어 있는가?
[ ] 각 FC에 작업요소(WE)가 연결되어 있는가?
[ ] 작업요소(WE)가 B1 시트 값과 동일한가?
[ ] USER 구분 FM의 S값이 9~10으로 평가되어 있는가?
[ ] ★ B5(예방관리)에 구체적 장치명·시스템명이 명시되어 있는가?
[ ] ★ A6(검출관리)에 검출 장비명 + 검사 방법이 명시되어 있는가?
[ ] B5·A6가 빈 행이 없는가? (비어 있으면 O·D 자동 추천 불가)
[ ] 공정번호 기준 정렬되어 있는가? (Process-first)
```

## 11. Import 확정 프로세스 (v5.4.0+)

### 11.1 확정 단계

| 단계 | 명칭 | 내용 | DB 저장 |
|------|------|------|---------|
| **SA** | 구조분석 확정 | C/A/B 레벨 기초정보 검증 → buildWorksheetState | Import 시 1회 저장 (replace=true) |
| **FC** | 고장사슬 확정 | FC 시트 기반 고장사슬 검증 → failureChains PATCH | chains만 업데이트 |

> **FA 통합분석 탭은 삭제됨** (v1.1). SOD·AP 분석은 워크시트 ALL탭에서 수행.

### 11.2 DB 저장 원칙

```
1. Import 버튼 → saveMasterDataset(replace=true) : 기존 데이터 전체 삭제 + 새 데이터 삽입
2. SA 확정     → failureChains PATCH만 수행 (flatData 중복저장 방지)
3. 워크시트 생성 → quickCreateWorksheet (원자성 테이블에 저장)
```

> **주의**: Import 시 기존 DB 데이터는 완전히 삭제되고 새 데이터로 교체된다.
> I→D (Import→DB) = 0이 정상 상태이다.

### 11.3 B5·A6 Import 흐름

```
★ FC 시트의 B5(8번 컬럼)·A6(9번 컬럼) → Import 시 자동 파싱
  → MasterFailureChain.pcValue (B5 예방관리)
  → MasterFailureChain.dcValue (A6 검출관리)
  → DB 저장 → 워크시트 ALL탭에서 O·D 자동 추천의 근거로 활용

⚠️ B5·A6가 비어 있으면:
  - O(발생도) 자동 추천 시 예방관리 수준 판단 불가
  - D(검출도) 자동 추천 시 검출 수준 판단 불가
  → 결과적으로 O·D를 수동 입력해야 하므로, 반드시 FC 작성 시 함께 기입할 것
```

### 11.4 I→D 중복 발생 시 대처

```
I→D가 +N (양수)이면 DB에 데이터가 중복 저장된 상태.
대처: 기존 데이터 삭제 후 다시 Import하면 I→D = 0으로 정상화.
```
