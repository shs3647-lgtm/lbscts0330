## 2. PostgreSQL 설치 및 설정

### 📦 PostgreSQL 설치 (Windows)

1. **PostgreSQL 다운로드**
   ```
   https://www.postgresql.org/download/windows/
   ```

2. **설치 시 설정**
   - Port: `5432` (기본값)
   - Username: `postgres` (기본값)
   - Password: **본인이 설정** (예: `postgres` 또는 `fmea_db_password_2026`)
   - Locale: `Korean, Korea` 또는 `C`

3. **설치 확인**
   ```powershell
   # PowerShell에서 실행
   psql --version
   ```

### 🗄️ 데이터베이스 생성

**방법 1: GUI 도구 사용 (pgAdmin)**

1. pgAdmin 실행 (PostgreSQL과 함께 설치됨)
2. 서버 연결 (postgres 사용자로 로그인)
3. 우클릭 > Create > Database
4. Database name: `fmea_db`
5. Save

**방법 2: 명령줄 사용**

```powershell
# PostgreSQL 사용자로 로그인
psql -U postgres

# 데이터베이스 생성
CREATE DATABASE fmea_db;

# 확인
\l

# 종료
\q
```

## 4. 환경 변수 설정

### 📝 .env 파일 생성

**방법 1: 템플릿 사용**

```powershell
# .env.new_laptop 파일을 .env로 복사
Copy-Item .env.new_laptop .env
```

**방법 2: 직접 생성**

`.env` 파일을 프로젝트 루트에 생성하고 아래 내용 입력:

```env
# PostgreSQL 데이터베이스 설정
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=fmea_db
DB_PORT=5432
DB_HOST=localhost

# Prisma 데이터베이스 연결 URL
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/fmea_db?schema=public"

# 애플리케이션 설정
PORT=3000
APP_PORT=3000
APP_URL=http://localhost
NODE_ENV=development
NEXT_PUBLIC_APP_URL=http://localhost:3000

# 보안 설정
JWT_SECRET=JWT@2026#SecretKey$Development&FMEA^SystemAuth!
SESSION_SECRET=Session#2026@FMEADev$SecretKey&Authentication!

# 타임존
TZ=Asia/Seoul
```

### ⚙️ 환경 변수 수정

1. `.env` 파일 열기
2. `DB_PASSWORD` 값을 PostgreSQL 설치 시 설정한 비밀번호로 변경
3. `DATABASE_URL`의 비밀번호 부분도 동일하게 변경

**예시:**
```env
DB_PASSWORD=your_actual_password
DATABASE_URL="postgresql://postgres:your_actual_password@localhost:5432/fmea_db?schema=public"
```

## 7. 문제 해결

### ❌ PostgreSQL 연결 오류

**에러 메시지:**
```
Error: P1001: Can't reach database server at localhost:5432
```

**해결 방법:**

1. **PostgreSQL 서비스 확인**
   ```powershell
   # 서비스 상태 확인
   Get-Service postgresql*
   
   # 서비스 시작
   Start-Service postgresql-x64-14  # 버전에 따라 이름 다를 수 있음
   ```

2. **포트 확인**
   ```powershell
   netstat -ano | findstr :5432
   ```

3. **방화벽 확인** - Windows 방화벽에서 PostgreSQL 허용 확인

### ❌ 데이터베이스 인증 오류

**에러 메시지:**
```
Error: P1001: Authentication failed for user `postgres`
```

**해결 방법:**

1. `.env` 파일의 비밀번호 확인
2. `DATABASE_URL`의 비밀번호 부분 확인
3. PostgreSQL 비밀번호 재설정 (필요시)

### ❌ 포트 충돌 오류

**에러 메시지:**
```
Error: Port 3000 is already in use
```

**해결 방법:**

```powershell
# 다른 포트로 실행
npm run dev:4000

# 또는 사용 중인 프로세스 종료
netstat -ano | findstr :3000
taskkill /PID <PID번호> /F
```

### ❌ Prisma 마이그레이션 오류

**에러 메시지:**
```
Error: Migration engine error
```

**해결 방법:**

```powershell
# 기존 마이그레이션 리셋 (데이터 삭제됨!)
npm run db:push

# 또는 처음부터 다시
npx prisma migrate reset
npm run db:migrate
```

### ❌ Node 모듈 오류

**에러 메시지:**
```
Error: Cannot find module ...
```

**해결 방법:**

```powershell
# node_modules 삭제 후 재설치
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install
```
