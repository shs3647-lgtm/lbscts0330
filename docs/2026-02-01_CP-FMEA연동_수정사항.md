# CP-FMEA 연동 수정사항 (2026-02-01)

## 📋 발견된 문제점 및 해결책

### 1. CP 등록 시 품명/품번 누락
**문제**: BizInfoSelectModal에서 고객정보 선택 시 `partName`, `partNo`가 설정되지 않음

**원인**:
- `onSelect` 핸들러에서 `customerName`, `modelYear`만 업데이트
- Prisma 스키마에 `partName`, `partNo` 필드 누락

**해결**:
1. `control-plan/register/page.tsx` - BizInfoSelectModal의 onSelect에 품명/품번 추가
2. `prisma/schema.prisma` - CpRegistration 모델에 `partName`, `partNo` 필드 추가
3. `prisma generate` + `prisma db push` 실행

**수평전개 대상**: PFD, DFMEA, APQP 등 BizInfoSelectModal 사용하는 모든 모듈

---

### 2. FMEA 목록이 localStorage에만 의존
**문제**: localStorage를 비우면 FMEA 목록이 "목록없음"으로 표시됨

**원인**: `useWorksheetState.ts`에서 FMEA 목록을 localStorage에서만 로드

**해결**:
```typescript
// DB API 우선, localStorage 폴백
const res = await fetch('/api/fmea/projects');
if (res.ok) {
  const data = await res.json();
  if (data.success && data.projects.length > 0) {
    setFmeaList(data.projects);
    localStorage.setItem('pfmea-projects', JSON.stringify(projects)); // 동기화
    return;
  }
}
// localStorage 폴백...
```

**수평전개 대상**: 모든 모듈의 목록 로드 로직 (localStorage → DB API 우선)

---

### 3. FMEA 이동 시 fmeaId 파라미터 누락
**문제**: CP에서 "FMEA 이동" 클릭 시 연동된 FMEA가 아닌 다른 FMEA로 이동

**원인**: `router.push('/pfmea/worksheet')`로 fmeaId 없이 이동

**해결**:
```typescript
const targetUrl = fmeaId ? `/pfmea/worksheet?id=${fmeaId}` : '/pfmea/worksheet';
router.push(targetUrl);
```

**수평전개 대상**: 모든 모듈 간 이동 버튼 (연동 ID 파라미터 전달 필수)

---

### 4. CP 저장 시 FMEA 프로젝트 기초정보 누락
**문제**: CP에서 FMEA 연동 후 이동하면 "기초정보가 없습니다" 경고 표시

**원인**: CP 저장 시 fmeaId만 저장하고 FMEA 프로젝트 자체는 생성하지 않음

**해결**:
```typescript
// /api/control-plan/route.ts
if (parentFmeaId) {
  const existingFmea = await prisma.fmeaProject.findUnique({ where: { fmeaNo: fmeaIdLower } });
  
  if (existingFmea && !existingFmea.subject) {
    // 기초정보 업데이트
    await prisma.fmeaProject.update({
      where: { fmeaNo: fmeaIdLower },
      data: {
        subject: cpInfo?.subject || `CP연동_${cpNoLower}`,
        productName: cpInfo?.partName || '',
        customerName: cpInfo?.customerName || '',
      },
    });
  } else if (!existingFmea) {
    // 신규 생성
    await prisma.fmeaProject.create({ ... });
  }
}
```

**수평전개 대상**: PFD, DFMEA에서 연동 시에도 동일 로직 적용

## 📌 원칙 (교훈)

1. **모든 데이터는 DB가 Single Source of Truth** - localStorage는 임시/캐시용만

3. **연동 생성 시 대상 프로젝트도 함께 생성** - 기초정보 포함
4. **Prisma 스키마 변경 후 반드시 `generate` + `db push`**
