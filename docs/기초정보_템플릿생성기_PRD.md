## 1. 개요

### 1.1 목적
PFMEA Import 페이지의 **기존 미리보기 패널을 활용**하여, 템플릿 생성 → 미리보기 → 편집 → 확정(저장/Export)을 **하나의 화면**에서 완결한다.

### 1.2 기존 방식의 문제점
| 문제 | 설명 |
|------|------|
| 빈 양식의 한계 | 공정번호/4M 행이 없어 사용자가 직접 구조를 잡아야 함 |
| 샘플의 한계 | 타이어 제조 12공정 고정 — 실제 업무와 무관 |
| 시트 수 혼동 | 14시트의 컬럼 구조를 사용자가 기억해야 함 |

### 1.3 해결 방향 (v2.0 — 통합 화면)
> **"설정만 선택하면 데이터가 미리보기에 바로 나타난다"**

**핵심 원칙**: 별도 미리보기를 만들지 않는다. 기존 ImportPreviewPanel이 이미 15개 아이템코드 탐색 + 편집 + 저장을 지원하므로, 템플릿 생성기는 **데이터만 생성**하고 기존 미리보기에 주입한다.

```
[설정 UI] → [생성] → flatData 주입 → 기존 ImportPreviewPanel
                                        ├─ 미리보기 (15개 아이템코드)
                                        ├─ 편집 (EditableCell)
                                        ├─ Import (Excel → flatData)
                                        ├─ Export (flatData → Excel)
                                        └─ 저장 (flatData → DB)
```

### 1.4 v1.0 → v2.0 변경 요약

| 항목 | v1.0 (분리) | v2.0 (통합) |
|------|------------|------------|
| 미리보기 | 별도 패널 (중복) | **기존 패널 재활용** |
| 시트탭 | 별도 탭바 (중복) | **기존 드랍다운 재활용** |
| 편집 | 새로 구현 | **기존 EditableCell 재활용** |
| 신규 컴포넌트 | 5개 (~1200행) | **2개 (~350행)** |
| 사용자 학습 | 새 UI 학습 필요 | **기존 화면 그대로** |

|--------|--------|--------|------|

| **A4** | 제품특성 | L2-4(A4) 제품특성 | |

| **A6** | 검출관리 | L2-6(A6) 검출관리 | |

| **B2** | 요소기능 | L3-2(B2) 요소기능 | |
| **B3** | 공정특성 | L3-3(B3) 공정특성 | |
| **B4** | 고장원인 | L3-4(B4) 고장원인 | |

| **C1** | 구분 | L1-1(C1) 구분 | YP/SP/USER |

## 4. 시트별 행 생성 규칙

### 4.1 A시리즈 (공정 레벨)

```
A1,2 행 수 = 공정 수
A3   행 수 = 공정 수 (공정당 1개 기능)
A4   행 수 = 공정 수 × N (제품특성은 공정당 복수 가능 → 기본 1)
A5   행 수 = A4와 동일 (제품특성:고장형태 = 1:1)
A6   행 수 = A5와 동일 (고장형태:검출관리 = 1:1)
```

### 4.2 B시리즈 (작업요소 레벨)

```
B1 행 수 = 공통(MN×n + EN×n) + Σ(공정별 MN×n + MC×n + IM×n + EN×n)
B2 행 수 = B1 행 수 × B2배수
B3 행 수 = B1 행 수 × B3배수
B4 행 수 = B1 행 수 × B4배수  (4M 컬럼 없음)
B5 행 수 = B1 행 수 × B5배수
```

> **③ 자동 모드**: B1 = 입력된 작업요소 수, B2~B5 = 작업요소당 N행

### 4.3 C시리즈 (완제품 레벨)

```
C1 = 3행 고정 (YP, SP, USER)
C2 = 3행 (구분당 1 기능)
C3 = 3행
C4 = 3행
```

## 5. 데이터 생성 로직

### 5.1 핵심 함수: `generateTemplateData()`

> **Excel 파일이 아닌 `ImportedFlatData[]` 배열을 생성한다.**

```typescript
// 수동 모드
function generateManualTemplate(config: ManualTemplateConfig): ImportedFlatData[]

// 자동 모드
function generateAutoTemplate(config: AutoTemplateConfig): ImportedFlatData[]

interface ManualTemplateConfig {
  processCount: number;          // 공정 수
  processNumbering: 'numeric' | 'alpha';  // 10,20... or A,B...
  commonMN: number;              // 공통 MN 수
  commonEN: number;              // 공통 EN 수
  perProcessMN: number;          // 공정별 MN 수
  perProcessMC: number;          // 공정별 MC 수
  perProcessIM: number;          // 공정별 IM 수
  perProcessEN: number;          // 공정별 EN 수
}

interface AutoTemplateConfig extends ManualTemplateConfig {
  workElements: WorkElementInput[];  // 작업요소 목록
  b2Multiplier: number;          // B2 배수
  b3Multiplier: number;          // B3 배수
  b4Multiplier: number;          // B4 배수
  b5Multiplier: number;          // B5 배수
}

interface WorkElementInput {
  processNo: string;
  processName: string;
  m4: 'MN' | 'MC' | 'IM' | 'EN';
  name: string;
}
```

### 5.2 생성 결과 → flatData 주입

```typescript
// [생성] 클릭 시
const templateData = generateManualTemplate(config);
setFlatData(templateData);  // 기존 미리보기 패널에 즉시 반영
setPreviewColumn('A2');     // A2(공정명)부터 표시
setDirty(true);             // 변경 표시
```

### 5.3 생성 규칙

#### 수동 템플릿

**A시트 데이터 (공정 레벨)**:
- A1: 공정번호 (10, 20, 30... 또는 A, B, C...)
- A2: 공정명 (빈 값 — 사용자가 미리보기에서 편집)
- A3~A6: 공정번호만 채우고 값은 빈 칸

**B시트 데이터 (작업요소 레벨)**:
- B1: 공통(00) MN/EN + 공정별 MN/MC/IM/EN 빈 행
- B2~B5: B1 행 구조 × 배수만큼 빈 행

**C시트 데이터 (완제품 레벨)**:
- C1: YP, SP, USER 고정 3행
- C2~C4: 구분별 빈 행

#### 자동 템플릿

- B1: 입력된 작업요소 이름 그대로 채움 (**데이터 완성**)
- B2~B5: 작업요소 복사 + 배수만큼 빈 행
- A1,2: 입력한 공정번호+공정명 자동 채움

## 7. 4M 행 생성 규칙 (절대 규칙)

### 7.1 4M 분류
| 코드 | 의미 | 행 구조 |
|------|------|---------|
| MN | 사람 | 공통(00공정) + 공정별 |
| MC | 설비/금형/지그 | 공정별만 |
| IM | 부자재 | 공정별만 |
| EN | 환경 | 공통(00공정) + 공정별 |

### 7.2 정렬 순서 (고정)
```
MN → MC → IM → EN (공정 내 정렬)
```

### 7.3 공통(00) 공정 규칙
- 공통 MN/EN은 **00 공통** 공정으로 생성
- B시트 최상단에 위치
- 공정별 MN은 각 공정 내에서 MC 앞에 위치

## 8. 파서 하위 호환

### 8.1 시트명 매핑 (SHEET_NAME_MAP 확장)

```typescript
// 새 시트명 → 아이템코드 (Export에서 생성하는 시트명)
'L2-1,2(A1,2) 공정번호': 'A1',  // 신규
'L2-3(A3) 공정기능': 'A3',      // 신규
'L2-4(A4) 제품특성': 'A4',      // 신규
'L2-5(A5) 고장형태': 'A5',      // 신규
'L2-6(A6) 검출관리': 'A6',      // 신규

// 기존 시트명 (하위 호환 유지)
'L2-1 공정번호': 'A1',
'L2-3 공정기능': 'A3',
'L2-4 제품특성': 'A4',
// ... 기존 매핑 그대로 유지
```

### 8.2 파싱 규칙
- A1,2 시트 → A1(공정번호) + A2(공정명) 각각 파싱
- 1열=공정번호, 2열=공정명 (기존과 동일)
- A3~A6: 1열=공정번호 (키), 2열~=값
