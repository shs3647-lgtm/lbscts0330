```
atomicToLegacyAdapter.ts (DB → Legacy 형식)
  L3Structure 1건당 → WorkElement 1건 생성
  같은 m4|name → 복수 WE 생성
    ↓
useL3Deduplication (useEffect)
  Step 1: deduplicateWorkElementsL3(m4|name)   → WE 병합, functions[] 단순 합침
  Step 2: deduplicateFunctionsL3(name)          → 같은 이름 function 병합, processChars[] 합침
  Step 2 ID dedup: 같은 id processChar 제거    ✅
  ❌ 누락: 같은 이름이지만 다른 ID의 processChar  → 중복 잔존!
    ↓
FunctionL3Tab 렌더링 → 중복 표시
```

**핵심**: `buildL3Functions()`에서 `processChars[].id = L3Function.id`를 사용.
다른 L3Structure의 같은 function은 다른 L3Function → 다른 ID.
WE + Function 병합 후 processChars가 같은 이름이지만 다른 ID로 남아 ID dedup 통과.

## 3. 해결 방안

### dedup 파이프라인 (수정 후)

```
atomicToLegacy (DB → state.l2)
    ↓
useL3Deduplication (useEffect)
    ├── Step 1 (기존): deduplicateWorkElementsL3(proc.l3)
    │     복합키 m4|name → 같은 WE 병합 (functions[] 합침)
    ├── Step 2 (기존): deduplicateFunctionsL3(we.functions)
    │     키: name → 같은 function 병합 (processChars[] 합침)
    ├── Step 2.5 (신규): deduplicateProcessCharsAfterWeMerge(uniqueFuncs)
    │     키: processChar name → WE 병합으로 생긴 공정특성 이름 중복 제거
    ├── Step 3 (기존): remapFailureCauseCharIds()
    ↓
FunctionL3Tab 렌더링 (중복 제거 완료)
```

### 복합키 설계

```
processChar dedup key = name.trim()
```

- 같은 function 내에서 같은 이름의 공정특성 → 첫 번째만 보존
- 빈 이름(placeholder) → 개별 보존 (dedup 안 함)
- specialChar: 첫 번째 항목 우선, 비어있으면 후속 값으로 보충

## 5. 구현 코드

### 5.1 `functionL3Utils.ts` — 신규 함수 (마지막 행 추가)

```typescript
/**
 * WE 병합 후 공정특성 복합키(name) 중복 제거
 *
 * ★ 적용 시점: deduplicateWorkElementsL3 → deduplicateFunctionsL3 이후.
 * ★ 기존 deduplicateFunctionsL3의 "이름 기준 dedup 금지" 정책과 다른 이유:
 *   기존 정책: 원래 같은 L3Structure 안에서 의도적으로 같은 이름의 공정특성을 둔 경우 보호.
 *   이 함수: WE 병합으로 인해 다른 L3Structure에서 온 동일 데이터의 복제본만 제거.
 *
 * ⚠️ AI주의: 이 함수를 deduplicateFunctionsL3 안에 넣지 말 것 — 별도 step으로만 호출.
 */
export const deduplicateProcessCharsAfterWeMerge = (
  functions: Array<{ id: string; name?: string; processChars?: any[]; [k: string]: any }>
): { functions: any[]; cleaned: boolean } => {
  let cleaned = false;
  const result = functions.map((f) => {
    const chars = f.processChars || [];
    if (chars.length <= 1) return f;
    const seenNames = new Map<string, any>();
    const uniqueChars: any[] = [];
    for (const c of chars) {
      const name = (c?.name || '').trim();
      if (!name) { uniqueChars.push(c); continue; }
      if (seenNames.has(name)) {
        const first = seenNames.get(name);
        if (!first.specialChar && c.specialChar) first.specialChar = c.specialChar;
        cleaned = true;
      } else {
        const copy = { ...c };
        seenNames.set(name, copy);
        uniqueChars.push(copy);
      }
    }
    return { ...f, processChars: uniqueChars };
  });
  return { functions: result, cleaned };
};
```

## 7. 병합하지 않는 경우

| 조건 | 이유 |
|------|------|
| 같은 function 내 다른 이름 processChar | 별개 공정특성 |
| 빈 이름 processChar | placeholder — 개별 보존 |
| 다른 function 내 같은 이름 processChar | 같은 WE 내라도 다른 기능의 공정특성 |
| 다른 공정의 같은 processChar | `proc.l3[]`가 이미 공정별 분리됨 |
