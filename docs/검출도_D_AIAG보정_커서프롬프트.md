# 커서 프롬프트: 검출도(D) AIAG-VDA P-FMEA 기준 보정

## 목적
P-FMEA 리스크분석(5단계) 데이터에서 검출도(D) 점수가 AIAG-VDA FMEA Handbook 1st Edition 기준과 불일치하는 건을 자동 보정한다.

## 입력 파일
`{입력파일경로}.xlsx`의 리스크분석 데이터
- 검출관리 열: 검출방법 텍스트 (D: 접두어로 시작, 줄바꿈으로 복수 방법 구분)
- D 열: 현재 검출도 점수 (1~10)

## 핵심 문제 (2가지)

### 문제 1: 자동검출 장비에 수동검사 수준 D값 부여
현재 데이터에서 자동측정/자동검출 장비(CMM, 체결검사기, 중량측정기, 카메라 검사기 등)에 D=7~8을 부여하고 있음. AIAG-VDA 기준으로 자동검출 장비는 D=3~5 범위가 적정.

### 문제 2: 관리활동을 검출방법으로 혼동
"검교정", "MSA 검증", "Gage R&R"은 검출방법(Detection Control)이 아니라 측정시스템 관리활동임. 이들이 D값 산정에 영향을 주면 안 됨. 실제 검출 장비가 함께 기재된 경우 그 장비 기준으로 D값을 산정해야 함.

## AIAG-VDA Detection Rating 기준표 (P-FMEA)

```
D=1  : 검출이 불필요 (에러프루프로 고장원인 자체를 차단)
D=2  : 거의 확실히 검출 (에러프루프로 고장원인 검출 + 즉시 차단)
D=3  : 높은 검출능력 (자동검출 + 자동배출/격리 — NG 자동제거)
D=4  : 높은 검출능력 (자동검출 + 자동분류 + 이상 시 자동정지)
D=5  : 보통 검출능력 (해당 스테이션 자동검출 — 자동정지 없음)
D=6  : 보통-낮은 검출능력 (후공정에서 수동 또는 자동검출)
D=7  : 낮은 검출능력 (해당 스테이션 수동검출 — 에러프루프/게이징 미검증)
D=8  : 매우 낮은 검출능력 (후공정에서 수동검출 — 주관적 평가)
D=9  : 극히 낮은 검출능력 (임의 또는 간헐적 검사)
D=10 : 검출 불가능 (검출 수단 없음)
```

## 검출장비 → D값 보정 규칙 (키워드 매핑)

### 자동검출+자동배출/격리 → D=3
| 키워드 | 장비 유형 | D값 | 근거 |
|--------|----------|-----|------|
| 바코드 스캐너, PDA 스캔 | 자동식별+자동차단 | 3 | 미스캔 시 다음공정 진행 불가 (에러프루프) |

### 자동검출+자동분류/정지 → D=4
| 키워드 | 장비 유형 | D값 | 근거 |
|--------|----------|-----|------|
| EOL 검사기 | End-of-Line 자동판정 | 4 | 자동검출+합/불 자동판정+라인정지 |
| 비전검사, 카메라 검사기 | 비전 자동검출 | 4 | 영상 자동판정+NG 자동분류 |
| 체결검사기, 체결검사 | 체결 자동판정 | 4 | 토크/각도 자동판정+이상시 정지 |
| 3차원 측정기, CMM | 좌표 자동측정 | 4 | 자동측정+합/불 자동판정 |
| 치수측정기 | 자동치수측정 | 4 | 자동측정+합/불 자동판정 |
| 핀휨 검사기 | 전용 자동검사 | 4 | 자동검출+NG 판정 |
| 중량측정, 발란스, 중량검사 | 자동중량판정 | 4 | 자동측정+허용범위 자동판정 |
| 도포검사기 | 도포량 자동측정 | 4 | 자동측정+합/불 자동판정 |
| 절연저항계, 통전검사 | 전기 자동검사 | 4 | 자동측정+합/불 자동판정 |
| 기능검사기 | 기능 자동검사 | 4 | 자동검출+합/불 자동판정 |
| 휘도측정기 | 광학 자동측정 | 4 | 자동측정+합/불 자동판정 |

### 교정공구+수동확인 → D=5
| 키워드 | 장비 유형 | D값 | 근거 |
|--------|----------|-----|------|
| 토크렌치, 토크검사 | 교정된 공구+수동체결 | 5 | 설정값 자동판정이나 작업자 개입 필요 |
| 환경측정기, 온습도계 | 센서 모니터링 | 5 | 연속 모니터링이나 자동정지 없음 |

### 관리활동 (검출방법 아님) → D값 산정 시 무시
| 키워드 | 성격 | 처리 |
|--------|------|------|
| 검교정 | 측정기 교정 관리 | 검출방법이 아님 → 함께 기재된 실제 장비의 D값 적용 |
| MSA 검증 | 측정시스템 분석 | 검출방법이 아님 → 함께 기재된 실제 장비의 D값 적용 |
| Gage R&R | 반복성재현성 분석 | 검출방법이 아님 → 함께 기재된 실제 장비의 D값 적용 |

### 수동검사 → D=7~8 (현행 유지)
| 키워드 | 장비 유형 | D값 | 근거 |
|--------|----------|-----|------|
| 육안검사 (단독) | 수동+주관적 | 8 | 작업자 의존, 검출 누락 가능성 높음 |

## 보정 로직 (처리 순서)

```
1. 검출관리 텍스트에서 "D:" 접두어 제거 후 각 검출방법 추출
2. 각 검출방법을 아래 순서로 키워드 매칭:
   a. "관리활동" 키워드(검교정, MSA 검증, Gage R&R) → 검출방법에서 제외
   b. 남은 실제 검출장비에 대해 키워드 매칭 → D값 후보 산출
   c. 복수 장비인 경우 → 가장 낮은(효과 높은) D값 채택
3. 관리활동만 있고 실제 장비가 없는 경우 → D=6 (검증된 관리 체계)
4. 보정된 D값이 현재 D값보다 높으면 → 현재값 유지 (악화 방지)
5. D열에 보정값 덮어쓰기
```

## 특수 처리

### "검교정 + 실제장비" 조합 처리
```
예: "D:검교정\nD:3차원 측정기(CMM)"
  → 검교정 제외 → CMM 기준 D=4 적용

예: "D:검교정\nD:MSA 검증" (관리활동만)
  → 실제 장비 없음 → D=6 (검증된 관리체계는 인정하되 자동검출 아님)

예: "D:Gage R&R\nD:검교정" (관리활동만)
  → 실제 장비 없음 → D=6
```

### "육안검사 + 자동장비" 조합 처리
```
예: "D:육안검사\nD:카메라 검사기"
  → 육안=D8, 카메라=D4 → 최소값 D=4 채택

예: "D:환경측정기(온습도계)\nD:육안검사"
  → 환경측정기=D5, 육안=D8 → 최소값 D=5 채택
```

## 보정 금지 규칙
- D=1은 불가 (에러프루프 수준은 현실적으로 모든 건에 적용 불가)
- 현재 D보다 높은 값으로는 절대 보정하지 않음 (악화 방지)
- 보정 최소값: D=3 (바코드/PDA 스캔 에러프루프 수준)

## 출력
- D열에 보정값 덮어쓰기
- 보정 전후 통계: 총 건수, 변경 건수, D값별 분포 변화
- AP 재산정: 보정된 D값으로 AP(H/M/L) 자동 재계산
- 보정 로그: 각 건별 [현재D → 보정D, 근거장비, 매칭키워드]

## 주의사항
- UI 변경 금지 — 데이터 보정만 수행
- 기존 서식(셀 색상, 글꼴, 테두리) 보존
- S(심각도), O(발생도)는 변경하지 않음
- 검출관리 텍스트 자체는 변경하지 않음 (D값만 보정)

## 버전 검사기 처리 규칙
"버전 검사기"는 SW 버전 자동확인 장비로 분류:
- 단독 또는 비자동장비와 조합 시 → D=4 (자동검출+자동판정)
- 자동장비(EOL, 바코드 등)와 조합 시 → 해당 자동장비의 D값 우선
