## 3. 네트워크 및 보안 설정

### 3.1 필요한 포트
| 포트 | 서비스 | 프로토콜 | 접근 범위 | 비고 |
|-----|--------|---------|----------|------|
| 80 | HTTP | TCP | Public | Nginx |
| 443 | HTTPS | TCP | Public | Nginx SSL |
| 3000 | Next.js | TCP | Internal | 앱 서버 |
| 5432 | PostgreSQL | TCP | Internal | 데이터베이스 |
| 6379 | Redis | TCP | Internal | 캐시/세션 |
| 22 | SSH | TCP | Admin | 관리자 접속 |

### 3.2 방화벽 설정 (Ubuntu/Debian)
```bash
# UFW 설치 및 활성화
sudo apt-get install ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 필요한 포트 열기
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS

# Docker 네트워크 허용
sudo ufw allow from 172.16.0.0/12 to any

# 방화벽 활성화
sudo ufw enable
sudo ufw status
```

### 3.3 방화벽 설정 (CentOS/RHEL)
```bash
# firewalld 설정
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-service=ssh

# Docker 네트워크 설정
sudo firewall-cmd --permanent --zone=trusted --add-source=172.16.0.0/12

# 설정 적용
sudo firewall-cmd --reload
sudo firewall-cmd --list-all
```

### 3.4 SELinux 설정 (CentOS/RHEL)
```bash
# SELinux 상태 확인
getenforce

# Docker를 위한 SELinux 설정
sudo setsebool -P container_manage_cgroup on
sudo setsebool -P httpd_can_network_connect on

# 문제 발생시 임시 비활성화 (권장하지 않음)
# sudo setenforce 0
```

## 4. Docker 설치 및 설정

### 4.1 Docker 설치 (Ubuntu/Debian)
```bash
# 기존 Docker 제거
sudo apt-get remove docker docker-engine docker.io containerd runc

# 필수 패키지 설치
sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# Docker GPG 키 추가
sudo mkdir -m 0755 -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Docker 저장소 추가
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Docker 설치
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 사용자를 docker 그룹에 추가
sudo usermod -aG docker $USER
newgrp docker

# Docker 서비스 시작 및 자동 시작 설정
sudo systemctl start docker
sudo systemctl enable docker

# 설치 확인
docker --version
docker compose version
```

### 4.2 Docker 설치 (CentOS/RHEL)
```bash
# 기존 Docker 제거
sudo yum remove docker \
                docker-client \
                docker-client-latest \
                docker-common \
                docker-latest \
                docker-latest-logrotate \
                docker-logrotate \
                docker-engine

# 필수 패키지 설치
sudo yum install -y yum-utils

# Docker 저장소 추가
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# Docker 설치
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Docker 서비스 시작
sudo systemctl start docker
sudo systemctl enable docker

# 사용자를 docker 그룹에 추가
sudo usermod -aG docker $USER
newgrp docker
```

### 5.3 비밀번호 생성 스크립트
```bash
#!/bin/bash
# generate-secrets.sh

# 랜덤 비밀번호 생성 함수
generate_password() {
    openssl rand -base64 32 | tr -d "=+/" | cut -c1-32
}

# 환경 변수 생성
cat > .env <<EOF
DB_PASSWORD=$(generate_password)
REDIS_PASSWORD=$(generate_password)
JWT_SECRET=$(generate_password)
SESSION_SECRET=$(generate_password)
ENCRYPTION_KEY=$(generate_password)
EOF

echo "비밀번호가 생성되었습니다. .env 파일을 확인하세요."
```

## 6. Docker 명령어 가이드

### 6.1 기본 Docker 명령어
```bash
# Docker 버전 확인
docker --version
docker compose version

# Docker 정보 확인
docker info
docker system df

# Docker 이미지 관리
docker images                      # 이미지 목록
docker pull <image>                # 이미지 다운로드
docker rmi <image>                 # 이미지 삭제
docker image prune -a              # 사용하지 않는 이미지 삭제

# Docker 컨테이너 관리
docker ps                          # 실행 중인 컨테이너
docker ps -a                       # 모든 컨테이너
docker start <container>           # 컨테이너 시작
docker stop <container>            # 컨테이너 중지
docker restart <container>         # 컨테이너 재시작
docker rm <container>              # 컨테이너 삭제
docker logs <container>            # 컨테이너 로그
docker exec -it <container> bash   # 컨테이너 접속

# Docker 볼륨 관리
docker volume ls                   # 볼륨 목록
docker volume create <volume>      # 볼륨 생성
docker volume rm <volume>          # 볼륨 삭제
docker volume prune                # 사용하지 않는 볼륨 삭제

# Docker 네트워크 관리
docker network ls                  # 네트워크 목록
docker network inspect <network>   # 네트워크 정보
docker network create <network>    # 네트워크 생성
docker network rm <network>        # 네트워크 삭제
```

### 6.2 Docker Compose 명령어
```bash
# 기본 명령어
docker compose up                  # 서비스 시작 (포그라운드)
docker compose up -d               # 서비스 시작 (백그라운드)
docker compose down                # 서비스 중지 및 삭제
docker compose stop                # 서비스 중지만
docker compose start               # 중지된 서비스 시작
docker compose restart             # 서비스 재시작

# 빌드 명령어
docker compose build               # 이미지 빌드
docker compose build --no-cache    # 캐시 없이 빌드
docker compose build <service>     # 특정 서비스만 빌드

# 로그 및 상태
docker compose ps                  # 서비스 상태
docker compose logs                # 전체 로그
docker compose logs -f             # 실시간 로그
docker compose logs <service>      # 특정 서비스 로그
docker compose logs --tail=100     # 최근 100줄

# 서비스 관리
docker compose exec <service> bash # 서비스 접속
docker compose run <service> cmd   # 명령 실행
docker compose scale <service>=3   # 서비스 스케일링

# 정리 명령어
docker compose down -v             # 볼륨 포함 삭제
docker compose rm -f               # 컨테이너 강제 삭제
```

## 7. 배포 프로세스

### 7.1 초기 배포 절차
```bash
#!/bin/bash
# deploy.sh - 초기 배포 스크립트

# 1. 작업 디렉토리 생성
sudo mkdir -p /opt/fmea
cd /opt/fmea

# 2. 소스 코드 다운로드
git clone <repository-url> .
# 또는 파일 직접 복사
# scp -r local-path/* user@server:/opt/fmea/

# 3. 환경 변수 설정
cp .env.example .env
nano .env  # 필요한 설정 수정

# 4. 권한 설정
sudo chown -R 1001:1001 /opt/fmea
chmod 600 .env

# 5. Docker 이미지 빌드
docker compose build

# 6. 서비스 시작
docker compose up -d

# 7. 데이터베이스 초기화
sleep 10  # DB 시작 대기
docker compose exec app npx prisma migrate deploy

# 8. 상태 확인
docker compose ps
docker compose logs --tail=50

# 9. 헬스체크
curl http://localhost:3000/api/health
```

### 7.2 업데이트 배포 절차
```bash
#!/bin/bash
# update.sh - 업데이트 배포 스크립트

# 1. 백업 생성
docker compose exec backup /scripts/backup.sh

# 2. 소스 코드 업데이트
git pull origin main
# 또는 새 파일 복사

# 3. 이미지 재빌드
docker compose build app

# 4. 무중단 배포 (Rolling Update)
docker compose up -d --no-deps --build app

# 5. 마이그레이션 실행
docker compose exec app npx prisma migrate deploy

# 6. 헬스체크
sleep 10
curl http://localhost:3000/api/health

# 7. 이전 이미지 정리
docker image prune -f
```

### 7.3 롤백 절차
```bash
#!/bin/bash
# rollback.sh - 롤백 스크립트

# 1. 현재 버전 태그 확인
CURRENT_VERSION=$(docker images fmea-app --format "{{.Tag}}" | head -1)
echo "현재 버전: $CURRENT_VERSION"

# 2. 이전 버전으로 롤백
PREVIOUS_VERSION="v1.0.0"  # 롤백할 버전 지정
docker compose down app
docker run -d --name app --network fmea-network fmea-app:$PREVIOUS_VERSION

# 3. 데이터베이스 롤백 (필요시)
docker compose exec backup /scripts/restore.sh

# 4. 서비스 재시작
docker compose restart nginx

# 5. 상태 확인
docker compose ps
```

```bash
    docker compose restart app
fi
```

## 9. 백업 및 복구

### 9.1 자동 백업 설정

```bash
0 3 * * 0 /opt/fmea/scripts/full-backup.sh
```

### 9.2 수동 백업
```bash
#!/bin/bash
# manual-backup.sh

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/fmea"

# 데이터베이스 백업
docker compose exec postgres pg_dump -U fmea_user fmea_db | gzip > $BACKUP_DIR/db_$TIMESTAMP.sql.gz

# 볼륨 백업
docker run --rm -v fmea_uploads:/data -v $BACKUP_DIR:/backup alpine tar czf /backup/uploads_$TIMESTAMP.tar.gz -C /data .

# Docker 이미지 백업
docker save fmea-app:latest | gzip > $BACKUP_DIR/image_$TIMESTAMP.tar.gz

# 설정 파일 백업
tar czf $BACKUP_DIR/config_$TIMESTAMP.tar.gz /opt/fmea/.env /opt/fmea/docker-compose.yml

echo "백업 완료: $BACKUP_DIR"
ls -lh $BACKUP_DIR/*$TIMESTAMP*
```

### 9.3 복구 절차
```bash
#!/bin/bash
# restore.sh

# 1. 서비스 중지
docker compose down

# 2. 데이터베이스 복구
gunzip < /backup/db_20240123.sql.gz | docker compose exec -T postgres psql -U fmea_user fmea_db

# 3. 볼륨 복구
docker run --rm -v fmea_uploads:/data -v /backup:/backup alpine tar xzf /backup/uploads_20240123.tar.gz -C /data

# 4. 서비스 재시작
docker compose up -d

# 5. 상태 확인
docker compose ps
```
