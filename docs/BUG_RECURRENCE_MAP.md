## 상세 재발 기록

### 패턴 A: processNo 포맷 불일치 (4회 재발)

```
조건: Excel INT() 수식이 선행 0 제거 ("010" → "10")
증상: B1/B3 parentItemId 매칭 실패 → 16개 작업요소 누락
근본원인: Set/Map 조회 시 normalizePno() 미적용
```

| 커밋 | 날짜 | 수정 내용 | 재발 여부 |
|------|------|----------|----------|
| `dd1ce8f` | 03-14 | 최초 발견 — 포맷 직접 비교 | 재발 |
| `ca1287c` | 03-15 | 부분 정규화 적용 | 재발 |
| `c47569a` | 03-15 | 공정별 갭 감지 추가 | 재발 |
| `b50761e` | 03-15 | normalizePno() 전면 적용 | ✅ 해결 |

**교훈**: 한 곳만 고치면 다른 비교 지점에서 재발. **모든 Map/Set 키에 일괄 적용 필수**.

## 재발 방지 체크리스트

새 버그 수정 전 반드시 확인:

- [ ] **이 버그가 위 5개 패턴 중 하나에 해당하는가?**
  - 해당하면 → 패턴별 근본 해법 적용 (증상 패치 금지)
- [ ] **processNo 비교가 포함되어 있는가?**
  - 있으면 → `normalizePno()` 사용 확인
- [ ] **엔티티 생성이 루프 안에 있는가?**
  - A4/FE처럼 공유 엔티티면 → 루프 밖에서 1회 생성 + FK 참조
- [ ] **파이프라인 순서에 의존하는 수정인가?**
  - 있으면 → Chain-Driven 전환 후 순서 의존성 자체가 사라지는지 확인
- [ ] **수정 범위가 1파일인가 vs 연쇄 영향이 있는가?**
  - 연쇄 영향 → 파급 파일 목록 먼저 나열 후 수정

## 근본 해결: Chain-Driven 전환

위 5개 패턴 중 **4개**(A, B, C, E)가 Chain-Driven 전환으로 **자동 해결**됩니다.

| 패턴 | Chain-Driven 후 | 이유 |
|------|----------------|------|
| A: processNo 불일치 | **자동 해결** | chain 행에 processNo 직접 포함 → 텍스트 매칭 불필요 |
| B: 카테시안 복제 | **자동 해결** | chain에서 직접 FK 발급 → distribute() 제거 |
| C: Phase 4 타이밍 | **자동 해결** | chain 1행 = FM+FC+FE+Link 동시 생성 → 순서 무관 |
| D: B1 공정별 갭 | 별도 수정 필요 | supplement 로직은 Chain-Driven과 독립 |
| E: C4(FE) 0건 | **자동 해결** | chain에서 FE 직접 생성 → processNo 루프와 무관 |
