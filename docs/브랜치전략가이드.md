## 📋 AI 리팩토링 체크리스트

### 리팩토링 전 (BEFORE)

```bash
# 1. 현재 브랜치 확인
git branch

# 2. 안정 버전 태그 생성 (아직 없다면)
git tag -a pre-refactor-$(date +%Y%m%d) -m "리팩토링 전 안정 버전"

# 3. 리팩토링 브랜치 생성
git checkout -b refactor/[작업명]

# 4. 변경할 파일 목록 확인
# AI에게 요청하기 전에 어떤 파일이 변경될지 물어보기!
```

### 리팩토링 중 (DURING)

```bash
# 1. 변경된 파일 수시 확인
git status

# 2. 핵심 파일 변경 감지 시 즉시 중단!
# - DataSelectModal.tsx
# - useWorksheet*.ts
# - *Tab.tsx

# 3. 커밋 자주 하기 (롤백 용이)
git add -A && git commit -m "WIP: [작업 내용]"
```

### 리팩토링 후 (AFTER)

```bash
# 1. 빌드 확인
npm run build

# 2. 핵심 기능 테스트 (필수!)
# - [ ] 구조분석 입력/저장
# - [ ] 기능분석 입력/저장
# - [ ] 고장분석 입력/저장
# - [ ] 탭 전환 후 데이터 유지
# - [ ] 새로고침 후 데이터 유지
# - [ ] 모달 열기/닫기/저장

# 3. 문제 없으면 main에 병합
git checkout main
git merge refactor/[작업명]

# 4. 문제 있으면 브랜치 삭제
git checkout main
git branch -D refactor/[작업명]
```

## ⚠️ AI 리팩토링 주의사항

### 절대 하지 말 것 (DON'T)

1. **한 번에 많은 파일 수정 요청 금지**
   - ❌ "전체 레이아웃 리팩토링해줘"
   - ✅ "StructureTab.tsx만 수정해줘"

2. **핵심 파일 직접 수정 요청 금지**
   - ❌ "DataSelectModal을 최적화해줘"
   - ✅ "별도 브랜치에서 DataSelectModal 수정해줘"

3. **"수평전개" 작업 주의**
   - 여러 파일에 동일 패턴 적용 시 연쇄 손상 위험
   - 하나씩 적용하고 테스트

### 반드시 할 것 (DO)

1. **작업 전 브랜치 생성**
   ```bash
   git checkout -b refactor/[작업명]
   ```

2. **변경 파일 수 제한**
   - 한 번에 최대 3개 파일만 수정

3. **자주 커밋**
   - 작은 단위로 커밋하여 롤백 용이하게

4. **테스트 후 병합**
   - 반드시 핵심 기능 테스트 후 main 병합

## 🔧 복구 명령어 모음

### 특정 파일만 이전 버전으로 복구
```bash
git checkout stable-v1.0 -- src/path/to/file.tsx
```

### 전체 롤백 (안정 버전으로)
```bash
git checkout stable-v1.0
git checkout -b main-backup
git checkout main
git reset --hard stable-v1.0
```

### 특정 커밋으로 복구
```bash
git log --oneline -20  # 커밋 해시 확인
git checkout [커밋해시] -- src/path/to/file.tsx
```
