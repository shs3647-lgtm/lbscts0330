# 코드프리즈 워크플로우

## 트리거
- 사용자가 "커밋", "코드프리즈" 키워드를 사용할 때

## 필수 질문
커밋 완료 후 반드시 다음을 물어본다:

```
✅ 커밋이 완료되었습니다.

🔒 코드프리즈를 적용하시겠습니까?

변경된 파일에 대해 다음 중 어떤 안정화 전략을 적용할까요?

| 전략 | 설명 | 적용 대상 |
|------|------|----------|
| 1️⃣ **하드코딩** | 동적 값을 정적 상수로 고정 | CONFIG, 컬럼 정의 등 |
| 2️⃣ **코드프리즈** | 파일 수정 금지 상태 표시 | 검증된 핵심 파일 |
| 3️⃣ **랩핑** | try-catch 안전 wrapper 적용 | API 호출, 이벤트 핸들러 |

어떤 전략을 적용하시겠습니까? (1/2/3 또는 조합)
```

## 적용 방법

### 1️⃣ 하드코딩 적용
- 동적 값에 `// HARDCODED` 주석 추가
- 상수로 분리하여 파일 상단에 배치

### 2️⃣ 코드프리즈 적용
- 파일 헤더에 다음 추가:
```typescript
/**
 * @status CODE_FREEZE 🔒
 * @frozen_date [날짜]
 * @freeze_level L1 | L2 | L3
 * 
 * ⚠️ 이 파일은 코드프리즈 상태입니다.
 */
```
- docs/CODE_STATUS.md에 기록

### 3️⃣ 랩핑 적용
- 함수에 try-catch wrapper 추가
- console.log 디버그 로그 추가
- 폴백 값 정의
