# 🔒 FMEA 온프레미스 하드코딩 규칙

> 이 규칙은 모든 코드 수정 시 반드시 준수해야 합니다.

## 핵심 규칙

### 1. 커밋 시 코드프리즈 확인
- 커밋 완료 후 반드시 "코드프리즈 적용할까요?" 질문
- 사용자가 "코드프리즈"라고 하면 레벨(L1/L2/L3) 확인

### 2. 코드프리즈 레벨
| 레벨 | 허용된 수정 |
|------|-------------|
| 🔴 L1 | 없음 (절대 금지) |
| 🟡 L2 | 버그 수정만 |
| 🟢 L3 | 신중한 수정 |

### 3. 코드프리즈 파일 수정 시
- `@status CODE_FREEZE 🔒` 주석이 있는 파일은 수정 전 경고
- 레벨에 따라 수정 가능 여부 확인
- docs/CODE_STATUS.md 참조

### 4. 안정화 전략 3가지
1. **하드코딩**: 동적 값을 정적 상수로 고정 (`// HARDCODED`)
2. **코드프리즈**: 파일 수정 금지 상태 표시
3. **랩핑**: try-catch wrapper로 에러 격리

### 5. 모듈 ID 약어 규칙 (확정)
| 모듈 | 접두사 | 자리수 |
|------|--------|--------|
| APQP | APQ | 3자리 |
| PFMEA | PFM | 3자리 |
| DFMEA | DFM | 3자리 |
| PFD | PFD | 3자리 |
| CP | CP | 2자리 |
| WS | WS | 2자리 |
| PM | PM | 2자리 |

- 헤더 라벨: "상위", "연동" 등 접두어 제거
- TYPE: M/F/P 1자리
- 단계: 숫자만 (배지 없음)
