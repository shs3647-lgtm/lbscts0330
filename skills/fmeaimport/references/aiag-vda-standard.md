# AIAG-VDA PFMEA 표준 가이드

## 개요

**AIAG-VDA PFMEA** (Process Failure Mode & Effects Analysis) 1st Edition은 자동차 및 관련 산업의 공정 분석 국제 표준입니다.

- **AIAG**: Automotive Industry Action Group (자동차 산업 협회)
- **VDA**: Verband der Automobilindustrie (독일 자동차 공업회)
- **버전**: 1st Edition (2008 이후, v4.2 현재)

---

## 1. PFMEA의 목적

```
제품/서비스 개발 또는 공정 개선 단계에서 잠재적 고장 및 영향을 체계적으로 식별,
평가, 개선하여 고객 만족과 품질 향상을 도모하는 도구.
```

### PFMEA 실행 시점

- **신규 공정 설계**: 공정 설계 초기 단계
- **공정 변경**: 공정 조건, 설비, 재료 변경 시
- **신규 제품**: 제품-공정 조합 신규 시
- **문제 해결**: 반복 불량 발생 시
- **공정 개선**: 능력 향상 목표 시

---

## 2. AIAG-VDA 7단계 분석 구조

AIAG-VDA PFMEA는 **7계층 분석** 구조를 갖습니다:

```
L1: 고객 관점 (Customer)
    └─ 제품 기능 & 요구사항 정의

L2: 공정 관점 (Process)
    └─ 공정 기능 & 제품 특성 정의

L3: 요소 관점 (Process Element)
    └─ 작업요소 & 공정 특성 정의

FC: 고장사슬 (Failure Chain)
    └─ 고장 경로 (FE → FM → FC) N:1:N 매핑

FA: 통합분석 (Failure Analysis)
    └─ 모든 분석 정보 통합 (26열)
```

### 2.1 L1: 고객 관점 (Customer Perspective)

**정의**: 제품의 최종 사용자 관점에서 기능과 요구사항 정의

| 항목 | 정의 | 예시 |
|------|------|------|
| **C1** | 구분 | YP (자사), SP (고객), USER (규제/안전) |
| **C2** | 제품 기능 | "[대상]이/가 [결과]를 제공한다" |
| **C3** | 요구사항 | 측정가능 특성 (영문 단위 포함) |
| **C4** | 고장 영향 | "[현상]으로 인한 [고객영향]" |

**C1 구분**:
```
YP (자사 공정 수율): "Wafer Spec Out으로 인한 수율 저하"
SP (고객 납품 품질): "불량 유출, 고객 반품"
USER (규제·안전): "RoHS 미준수", "ESD 손상", "환자 상해"
```

**C2 제품 기능 패턴**:
```
"[대상] + [행위] + [결과]"

올바름: "웨이퍼가 규격 내 두께를 제공한다"
금지:   "웨이퍼 두께", "두께 측정", "두께가 맞다"
```

**C3 요구사항 패턴**:
```
"[특성명(영문)] : [최소값] ~ [최대값] [단위]"

올바름: "Thickness: 10.0 ± 0.5 mm", "Resistance: 50~150 Ω"
금지:   "두께 적절", "규격 만족", "품질 좋음"
```

**C4 고장 영향 패턴**:
```
"[문제 현상] + [영향 대상]"

올바름: "두께 부족으로 인한 조립 간섭", "표면 거칠기 증가로 고객 반품"
금지:   "불량", "제품 불합격", "고장"
```

### 2.2 L2: 공정 관점 (Process Perspective)

**정의**: 공정 관점에서 공정 기능과 제품 특성 정의

| 항목 | 정의 | 예시 |
|------|------|------|
| **A1** | 공정번호 | 정수 (zero-pad: '01', '02', ..., '100') |
| **A2** | 공정명 | 공정 실명 |
| **A3** | 공정기능 | "[대상] + [방법] + [처리] + [결과]" |
| **A4** | 제품특성 (결과값) | 공정 출력, 명사형 |
| **A5** | 고장형태 | "[A4] + [이탈유형]" |
| **A6** | 검출관리 | 장비 + 방법 + 빈도 (선택) |

**A1 공정번호**:
```
Zero-padding 필수:
  올바름: '01', '10', '100'
  금지:   '1', '2'

규칙: str(공정번호).zfill(2)
```

**A3 공정기능 패턴**:
```
"[대상]을/를 [방법]으로 [처리]하여 [결과]를 확보한다"

예: "웨이퍼를 플라즈마로 에칭하여 패턴을 형성한다"
    "부품을 초음파로 세척하여 불순물을 제거한다"
```

**A4 제품특성 (공정 출력)**:
```
명사형만 허용, 수치/규격 제외:

올바름: "두께", "표면 거칠기", "색상", "경도"
금지:   "두께 > 10mm", "두께가 얇다", "두께 측정"
```

**A5 고장형태 (결과값 이탈)**:
```
패턴: "[A4] + [이탈유형]"

이탈유형 분류:
  · 초과: "부족", "과다", "증가", "감소"
  · 특성 이탈: "편차", "비대칭", "불균일"
  · 기타: "편차", "불량", "손상"

올바름: "두께 부족", "표면거칠기 편차", "색상 불균일"
금지:   "두께 측정 불가", "고객 불만", "불량 유출" (← C4)
        "압력 편차" (← B4)
```

**A6 검출관리 (선택)**:
```
패턴: "[장비] + [방법] + [빈도]"

예: "높이 측정기 전수검사, 매 1시간 검사"
    "XRF 분석기 매 lot 분석, 3개 샘플"
    "육안 검사 100% 매 일일"

규칙:
  · 동일 공정 A6는 1행만 (복수 방법은 쉼표 구분, 1셀)
  · A6 값이 여럿이면 컬럼 "|"로 구분 가능
```

### 2.3 L3: 요소 관점 (Process Element Perspective)

**정의**: 공정 실행 단계에서 작업요소 & 공정 특성 정의

| 항목 | 정의 | 예시 |
|------|------|------|
| **B1** | 작업요소 | 4M + 설비/재료/인원 고유명 |
| **B2** | 요소기능 | "[B1]이/가 [행위]하여 [결과]를 제공한다" |
| **B3** | 공정특성 (입력값) | 설비 입력 파라미터, 명사형 |
| **B4** | 고장원인 | "[B3] 편차" 또는 "[원인]에 의한 [결과]" |
| **B5** | 예방관리 | 발생 전 방지 (선택이지만 권장) |

**B1 작업요소 (4M)**:
```
4M 분류:
  · Man (인원): "작업자 A", "조작자 B"
  · Machine (설비): "프레스 A", "용접기 #1", "노즐 #3"
  · Material (재료): "Type A 수지", "알루미늄 6061", "접착제 XYZ"
  · Method (방법): "온도 조절", "압력 설정", "타이밍 조정"

원칙: 관리 항목/상태명 제외, 설비·재료·인원 고유명만

올바름: "프레스 유압 장치", "Type A 재료", "작업자 A"
금지:   "압력 관리", "온도 조절", "QC 확인"
```

**B3 공정특성 (공정 입력)**:
```
명사형, A4와 다른 관점:

예: B3 (입력값):       B4 (입력 편차):
    온도               온도 편차
    압력               압력 상한 이탈
    습도               습도 편차
    회전수(RPM)        RPM 편차

금지: "온도를 높인다" (동사형), "온도가 낮다" (형용사형)
```

**B4 고장원인 (명사형 3대 패턴)**:
```
패턴 1 - 이탈형: "[B3 파라미터] 편차/규격 이탈"
  예: "온도 편차", "압력 상한 이탈", "습도 편차"

패턴 2 - 인과형: "[원인]에 의한 [결과]"
  예: "노즐 막힘", "Target 소진", "접촉 불량"

패턴 3 - 누락형: "[대상] 미실시/미완료/누락"
  예: "유지보수 미실시", "QC 검사 누락", "설정 미적용"

금지: "온도가 낮다" (형용사), "온도 설정 실패" (동사)

★ 절대 동사형 종결 금지: ~한다, ~않음, ~진행함
```

**B5 예방관리 (발생 전 방지)**:
```
패턴: "[설비/절차/시스템] + [개선 조치]"

예: "온도 자동 조절 장치 설치"
    "설비 점검 절차 강화 (일일 점검)"
    "자동화 장치 도입"

vs B5 아닌 것 (A6 검출관리):
    "온도 측정기 설치" (← 검출, A6)
    "매 시간 온도 검사" (← 검출, A6)

원칙: B5/A6 교차 혼입 금지
```

### 2.4 FC: 고장사슬 (Failure Chain) ★핵심

**정의**: 고장 경로를 N:1:N 구조로 매핑

```
FE(고장영향) N개  :  FM(고장형태) 1개  :  FC(고장원인) N개

관계식:
  FE (C4에서 추출) ← FM (A5에서 추출) → FC (B4에서 추출)
```

**구조**:

| 열 | 항목 | 정의 | 셀병합 |
|----|------|------|--------|
| A | FE 구분 | 고장영향 분류 (고객불만, 안전, 규제) | ○ |
| B | FE | 고장영향 (C4) | ○ |
| C | 공정번호 | A1~A3 값 | ○ |
| D | FM | 고장형태 (A5) | ○ |
| E | 4M | 작업요소 분류 | × |
| F | WE | 작업요소 (B1) | × |
| G | FC | 고장원인 (B4) | × |
| H | PC | 예방관리 (B5) | × |
| I | DC | 검출관리 (A6) | × |
| J | SOD | 심각도 (1~10) | × |
| K | AP | 발생가능성 (H/M/L) | × |
| L | FP | 검출가능성 (선택) | × |

**N:1:N 예시**:

```
FM = "높이 부족" (1개)

FE (2개):          FM (1개):         FC (3개):
├─ 고객불만       ├─ 높이 부족      ├─ 압력 편차
│                  │                 ├─ 노즐 막힘
│                  │                 └─ 온도 상한 이탈
└─ 조립간섭        └─ (동일)
```

**셀병합 규칙**:

```
병합 대상 (A, B, C, D): 연속 동일값 병합
병합 비대상 (E~L): 병합 금지, 각 행별 고유값

예: FE "고객불만" (2행 병합) + FM "높이 부족" (2행 병합)
    FC는 3행: "압력 편차", "노즐 막힘", "온도 이탈"
```

### 2.5 FA: 통합분석 (Failure Analysis)

**정의**: FC + B5/A6 + 심각도/가능성/검출 모두 통합 (26열)

```
FC 12열 + RPN 계산 + 평가 결과 = 26열

RPN = SOD × AP × FP (또는 SOD × AP 만 사용)

평가 기준 (산업별 상이):
  · RPN > 200: 개선 필수 (우선순위 높음)
  · RPN > 100: 검토 필요
  · RPN ≤ 100: 현재 상태 유지 가능
```

## 3. 항목별 혼입 금지 규칙

### 3.1 A4 vs B3 (결과값 vs 입력값)

```
A4 = 공정 OUTPUT, 제품 특성
  예: "두께", "표면 거칠기", "색상"
  측정처: 제품/웨이퍼 상

B3 = 공정 INPUT, 설비 파라미터
  예: "온도", "압력", "습도", "회전수"
  설정처: 설비/장비 제어
```

### 3.2 A5 vs B4 (결과 이탈 vs 원인 이탈)

```
A5 = 공정 OUTPUT의 이탈 (증상/현상)
  예: "두께 부족", "표면 거칠기 증가"
  범위: 공정 내

B4 = 공정 INPUT의 이탈 (근본 원인)
  예: "온도 편차", "노즐 막힘"
  범위: 공정 내
```

### 3.3 A5 vs C4 (공정 고장 vs 고객 영향)

```
A5 = 공정 관점 (Wafer 단위)
  예: "높이 부족" (공정에서 발생)

C4 = 고객 관점 (제품 사용 시)
  예: "높이 부족으로 인한 조립 간섭" (고객 영향)
```

### 3.4 B5 vs A6 (예방 vs 검출)

```
B5 = 발생 전 방지 (예방)
  예: "온도 자동 조절 장치", "설비 일일 점검 절차"

A6 = 발생 후 검출 (검출)
  예: "온도 측정기", "매 1시간 검사"
```

## 4. PFMEA 작성 순서

```
Step 1. L1 정의 (고객 관점)
        └─ C1/C2/C3/C4 정의

Step 2. L2 정의 (공정 관점)
        └─ A1/A2/A3/A4/A5 정의, (A6 선택)

Step 3. L3 정의 (요소 관점)
        └─ B1/B2/B3/B4/(B5 선택) 정의

Step 4. FC 구성 (고장사슬 매핑)
        └─ FE ← FM → FC 관계 정의
        └─ N:1:N 구조 확인

Step 5. FA 통합 (26열)
        └─ RPN 계산
        └─ 개선 우선순위 결정

Step 6. 검증 & 교정
        └─ SSOT 3축 검증
        └─ 오류 자동 교정

Step 7. 최종 리뷰 & 승인
```

---

## 5. AIAG-VDA 검증 원칙

### 5.1 SSOT (Single Source Of Truth) 3축 검증

```
축 1: set(A5.FM) == set(FC.D열 FM)
      → A5에 있는 모든 FM이 FC에도 있어야 함

축 2: set(FC.G열) ⊆ set(B4.고장원인)
      → FC의 모든 고장원인이 B4에 존재해야 함

축 3: set(FC.F열) ⊆ set(B1.작업요소)
      → FC의 모든 작업요소가 B1에 존재해야 함
```

### 5.2 데이터 품질 규칙

```
구조:
  □ zero-padding (A1)
  □ A6 중복 공정번호 없음
  □ B5 빈 셀 없음
  □ C3/C4 필수 조건 만족

내용:
  □ A4 = 명사형
  □ A5 = [A4] + [이탈유입]
  □ B1 = 고유명 (상태명 제외)
  □ B4 = 명사형 3대 패턴
  □ C2 = 명사형 기능
  □ C3 = 영문 단위
  □ C4 = 현상 + 영향
```

## 6. 산업별 확장 항목

### 6.1 자동차 (IATF 16949)

```
추가 항목:
  · 특수특성 표기 (CC/CR)
  · 고객 요구사항 연계
  · 설계 리뷰 결과 반영
```

### 6.2 의료기기 (ISO 14971)

```
추가 항목:
  · 위해도 평가 (심각도 × 가능성)
  · 위해 통제 조치
  · 규제 요구사항 (FDA)
```

### 6.3 반도체

```
추가 항목:
  · Process Capability (Cpk)
  · Spec 수치 (B3 입력)
  · 엔지니어링 기준
```
