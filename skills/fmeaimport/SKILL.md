# 다중산업 FMEA Import 작성 스킬

## 개요

이 스킬은 **AIAG-VDA PFMEA 1st Edition** 표준에 따라 다양한 산업의 Import 엑셀 파일을 일관성 있게 생성·검증·교정합니다.

**지원 산업** (7개):
- 자동차 (Automotive) — 프레스, 용접, 도장, 조립
- 반도체 (Semiconductor) — 웨이퍼 가공, 포토, 에칭, 증착, CMP
- 의료기기 (Medical Device) — ISO 14971 위해도 연계
- 전자제품 (Electronics) — SMT, PCB Assembly
- 사출 (Molding) — 금형 설계, 사출, 후처리
- 주조 (Casting) — 금형, 주조, 열처리, 가공
- 전기전자조립 (Assembly) — 부품 조립, 검사, 패킹

## 3. 항목별 핵심 작성 규칙

### 3.1 항목 구분 원칙 (혼입 금지)

```
A4 = 제품 결과값 (Wafer/제품에서 측정)     ↔  B3 = 공정 입력값 (설비에서 설정)
A5 = 결과값 이탈 현상 (공정 내)             ↔  B4 = 입력값 이탈 원인 (공정 내)
C4 = 고객/다음공정 영향 (공정 외)
B5 = 사전 예방 (발생 전 방지)              ↔  A6 = 사후 검출 (발생 후 검출)

A5 ≠ C4  /  A5 ≠ B4  /  B5 ≠ A6
```

### 3.2 B4 고장원인 명사형 3대 패턴

```
이탈형: [파라미터] 편차/규격 이탈
  예: "전류밀도 편차", "온도 상한 이탈", "RPM 편차"

인과형: [원인]에 의한 [결과]
  예: "노즐 막힘", "Target 소진", "접촉 불량"

누락형: [대상] 미실시/미완료/누락
  예: "설비 유지보수 미실시", "QC 검사 누락"

★ 동사형 종결 절대 금지: ~한다, ~않음, ~진행함
```

### 3.3 공정번호 Zero-Padding 필수

```
올바름: '01', '10', '100'
금지:   '1', '2'  →  반드시 zero-pad
규칙:   str(공정번호).zfill(2)
```

### 3.4 A4 제품특성 (제품 결과값)

**명사형만 허용**, 수치/규격 제외:
```
올바름: "높이", "두께", "표면 거칠기", "색상"
금지:   "높이 > 10mm", "높이가 크다", "높이 측정"
```

### 3.5 A5 고장형태 (결과값 이탈)

**[A4] + [이탈유형]** 구조:
```
올바름: "높이 부족", "두께 초과", "색상 편차"
금지:   "높이 측정 불가", "고객 불만", "불량 유출"
```

### 3.6 B1 작업요소 (4M + 고유명)

**관리항목/상태명 제외**, 설비/재료/인원 고유명만:
```
올바름: "압력 설정기", "Type A 재료", "작업자 A"
금지:   "압력 관리", "온도 조절", "QC 확인"
```

### 3.7 C3 요구사항 (측정가능 특성)

**영문 단위 + 수치 범위**:
```
올바름: "Height: 10.0 ± 0.5 mm", "Density: 2.7 g/cm³"
금지:   "높이 적절", "규격 만족", "품질 좋음"
```

### 3.8 B5/A6 교차 혼입 금지

```
B5 = 예방 (설비 교정, 절차 개선, 자동화 장치)
A6 = 검출 (측정기, 검사 방법, 검사 빈도)

올바름:
  B5: "설비 자동 교정 장치 설치"
  A6: "높이 측정기 전수검사, 매 1시간 검사"

금지:
  B5: "검사 강화" (← A6 관점)
  A6: "설비 교정" (← B5 관점)
```

## 4. FC 고장사슬 작성 규칙 ★핵심

### 4.1 N:1:N 구조

```
FE(고장영향) N개  :  FM(고장형태) 1개  :  FC(고장원인) N개

FM = 행 그룹의 중심 축 (셀병합 기준)
```

예시:
```
FE           FM                      FC
├─ 고객불만  ├─ 높이 부족 (FM1)     ├─ 압력 편차
│             │                      ├─ 노즐 막힘
│             │                      └─ 온도 상한 이탈
└─ 반품      └─ 표면 거칠기 (FM2)   ├─ 연마재 마모
                                     └─ 냉각수 오염
```

### 4.2 FE vs FC 수 불균형 처리

```
FE 수 < FC 수  →  FE 마지막 행에 동일 FE 반복 (셀병합 대상)
FE 수 > FC 수  →  FC 마지막 행에 동일 FC 반복 (셀병합 대상)
```

### 4.3 셀병합 대상 / 비대상

```
병합 대상:    C열(공정번호), D열(FM), A열(FE구분), B열(FE) — 연속 동일값
병합 비대상:  E열(4M), F열(WE), G열(FC), H열(PC), I열(DC), J~L열(SOD/AP/FP)
```

### 4.4 SSOT 3축 교차검증 (Import 전 필수)

```
① set(A5.FM)  ==  set(FC.D열 FM)       → 불일치 시 Import 오류
② set(FC.G열) ⊆  set(B4.고장원인)      → FC 고장원인 B4 미존재 = UUID 연결 실패
③ set(FC.F열) ⊆  set(B1.작업요소)      → WE 불일치 = 연결 실패
④ set(FC.B열) ⊆  set(C4.고장영향)      → FE 불일치 = 매핑 실패
```

## 5. 검증 체크리스트 (생성/교정 후 반드시 실행)

### 5.1 구조 검증

```
□ 공정번호 zero-padding ('1' 형식 없음)
□ A6 동일 공정번호 2행 이상 없음
□ B5 ★작업요소 열 빈 셀 없음 (필수)
□ L3통합 B5 열 빈 셀 없음 (필수)
□ L1통합 C4 있는 행에 C2, C3 모두 기재 (필수)
□ C3 시트 YP→SP→USER 그룹 순서
□ FC FE:FC 수 균형 (불균형 시 반복 행 추가)
```

### 5.2 SSOT 3축 교차검증

```
□ A5 FM 집합 = FC FM 집합 (완전 일치)
□ FC 고장원인(G열) 전체가 B4 집합에 존재
□ FC 작업요소(F열) 전체가 B1 집합에 존재
□ FC 고장영향(B열) 전체가 C4 집합에 존재
```

### 5.3 FC 구조 검증

```
□ FM 그룹 내 FE:FC 수 균형
□ FM(D열) 셀병합 적용
□ 공정번호(C열) FM 그룹 단위 셀병합
□ SOD 값 1~10, AP 값 H/M/L
□ B4·FC(G열) 동사형 종결 없음
```

### 5.4 내용 품질

```
□ A4 명사형 (결과값), 수치/규격 없음
□ A5 동사형 없음, A5에 'FM'·'불량 유출' 없음
□ B1 관리항목/상태명 없음 (설비·재료·인원 고유명만)
□ B3 명사형 (입력값), 수치 없음
□ B4 명사형 3대 패턴 (이탈형/인과형/누락형)
□ B5/A6 교차 혼입 없음
□ C2 명사형 (기능)
□ C3 영문 단위 포함
□ C4 현상 + 고객영향 (또는 다음공정 영향)
```

## 6. 검증 규칙 및 자동 교정

`references/validation-rules.md` 참조.

**주요 규칙** (25개):

| 규칙 ID | 오류 | 자동 수정 방법 |
|---------|------|--------------|
| `PROC_ZERO_PAD` | 공정번호 '1' 형식 | `str(v).zfill(2)` |
| `A6_PROC_DUP` | A6 중복 공정번호 | 두 번째 이후 행 삭제 |
| `B5_WE_EMPTY` | B5 작업요소 빈 셀 | B1 매핑으로 자동 채우기 |
| `L3_B5_EMPTY` | L3통합 B5 빈 셀 | B5 시트 매핑 적용 |
| `L1_C3_EMPTY` | L1통합 C3 빈 셀 | C4 역매핑 |
| `C3_GROUP_ORDER` | C3 그룹 순서 위반 | SP 행을 USER 앞으로 이동 |
| `FC_PROC_MISMATCH` | FC 고장원인 B4 불일치 | B4 SSOT 값으로 교체 + 적색 폰트 |
| `B4_VERB_ENDING` | 동사형 B4 | 명사형 3대 패턴 제안 |
| `FC_BALANCE` | FE:FC 불균형 | 마지막 행 반복 추가 |
| `FC_FM_MISMATCH` | FC FM ≠ A5 FM | Import 오류, 교정 필요 |

## 9. 산업별 추가 규칙

### 9.1 자동차 (IATF 16949)

```
특수특성 표기:
  CC (Customer Characteristic) - 고객 요구 특성
  CR (Characterized Requirement) - 특성화된 요구사항

IATF 진수계 분석 (C4 패턴):
  "~에 의한 성능 저하" (제품 기능)
  "~로 인한 고객 불만" (납품 품질)
  "~Spec Out 리스크" (불량 유출)
```

### 9.2 의료기기 (ISO 14971)

```
위해도 평가:
  · 심각도 + 가능성 조합
  · 5×5 매트릭스 (1~25 점수)
  · 위해도 ≥ 8 → 위해 통제 필수 (B5)
  · 위해도 ≥ 15 → 위해 분석 필수

C4 고장영향 패턴:
  "환자 상해", "규제 위반", "장비 오작동"
```

### 9.3 반도체 (Process Capability)

```
B3 공정특성에 규격 추가 가능:
  "온도 (Setpoint: 400°C, Spec: ±10°C)"

B4 고장원인 + Cpk 연계:
  "온도 편차 (Cpk < 1.33)" → 프로세스 개선 필요
```

## 10. 오류 처리 및 FAQ

### 10.1 일반 오류

**Q: FC에 B4가 없다고 오류가 나요.**
```
A: FC(G열 고장원인)이 B4에 없다는 뜻입니다.
   1. B4 시트에서 해당 원인을 찾아 추가하거나
   2. FC의 G열 값을 기존 B4 값으로 수정하세요.
   SSOT 3축 검증 참조.
```

**Q: B5가 빈 셀이라고 오류가 나요.**
```
A: B5 (예방관리)는 필수입니다 (빈 셀 불가).
   자동 수정: B1 (작업요소) 매핑으로 채워집니다.
   또는 수동으로 구체적 예방 조치를 기입하세요.
```

**Q: FC FE:FC 불균형이란?**
```
A: FM(고장형태) 한 개에 대해
   FE(고장영향) 개수 ≠ FC(고장원인) 개수

   해결: 부족한 쪽 마지막 값을 반복 행으로 추가.
   예: FE 2개, FC 3개 → FE 마지막 행 1개 반복 추가.
```

### 10.2 산업별 오류

**자동차**: IATF 특수특성 미표기 → C4에 CC/CR 추가 확인

**의료기기**: ISO 14971 위해도 <8 → 위해 통제 불필요 (검토)

**반도체**: Cpk 불충분 → B5 프로세스 개선 항목 강화

### 예시 1: 자동차 프레스 공정 FMEA

```
사용자: "자동차 프레스 공정 FMEA를 만들어줘"

1. 산업 식별: 자동차 (Automotive)
2. 템플릿 로드: automotive-template
3. 공정 참조: automotive-processes.md → 프레스 공정
4. 데이터 수집:
   - A4: 부품 높이, 두께, 표면 거칠기
   - A5: 높이 부족, 두께 초과, 표면 흠집
   - B1: 프레스 유압, 금형, 재료, 작업자
   - B4: 유압 편차, 금형 마모, 재료 결함
   - FC: 고장사슬 구성 (3×3 N:1:N)
   - C4: "높이 부족으로 인한 조립 불량", "표면 흠집 고객 반품"
5. 자동 검증: validation-rules.md
6. 엑셀 생성: excel-generation.md
7. 출력: FMEA_Automotive_Stamping_20260315_v1.0.xlsx
```
