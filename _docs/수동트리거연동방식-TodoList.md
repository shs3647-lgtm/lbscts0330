## 🚨 핵심 개발 원칙 (필수 준수)

```
┌─────────────────────────────────────────────────────────────────┐
│  📌 FMEA 시스템 개발 5대 원칙                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1️⃣ 모든 동작은 [트리거]에 의해 시작한다                       │
│     - 자동 추론/자동 생성 금지                                  │
│     - 사용자 클릭 또는 명시적 이벤트로만 동작                   │
│                                                                 │
│  2️⃣ 모든 데이터는 [DB]에서 연동한다                            │
│     - Master DB가 Single Source of Truth                       │
│     - 조회/저장 모두 DB API를 통해 처리                         │
│     - 연동 정보(CP/PFD/DFMEA)는 DB에서 직접 로드                │
│                                                                 │
│  3️⃣ localStorage는 [임시 저장소]이다                           │
│     - 화면 이탈 시 임시 백업 용도만 사용                        │
│     - 최종 데이터는 반드시 DB에 저장                            │
│     - localStorage 데이터를 정식 데이터로 취급하지 않음         │
│                                                                 │
│  4️⃣ 모든 앱은 [표준화/모듈화/공용화]하여 일관성 유지           │
│     - 공통 컴포넌트, 유틸리티, API 패턴 사용                    │
│     - APQP/PFMEA/DFMEA/PFD/CP 동일 구조 유지                    │
│                                                                 │
│  5️⃣ 기능 수정 시 [연동 앱 동시 업데이트] 필수                  │
│     - 하나의 기능 수정 시 연동되는 모든 앱 검토                 │
│     - 수정 범위: 해당 기능을 사용하는 모든 모듈                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> [!IMPORTANT]
> **앞으로 모든 개발은 위 5대 원칙을 준수해야 합니다.**
> 자동 추론 금지, DB 우선, localStorage 임시용, 표준화/공용화, 연동 앱 동시 업데이트

## 📊 연동 아키텍처 구조

```
┌─────────────────────────────────────────────────────────────────┐
│  📌 앱 계층 구조 (개발 순서대로)                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [1단계] APQP (최상위 - 프로젝트)                               │
│           │                                                     │
│           ├── [2단계] PFD (공정흐름도)                         │
│           ├── [2단계] DFMEA (Design FMEA)                      │
│           └── [2단계] PFMEA (Process FMEA)                     │
│                         │                                       │
│                         └── [3단계] CP (관리계획서)            │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  ★ 연동 규칙                                                   │
│  - PFD, DFMEA, PFMEA: APQP 등록 화면에서 생성/연동             │
│  - CP: DFMEA 또는 PFMEA 등록 화면에서 생성/연동                │
│  - 상위 앱에서 하위 앱을 생성/연동 (트리거 방식)               │
└─────────────────────────────────────────────────────────────────┘
```

> [!NOTE]
> **개발 순서**: APQP → PFD/DFMEA/PFMEA → CP
> **DFMEA↔PFMEA 연동**: APQP 등록 화면에서 설정
