# CP 기초정보 입력 개선 기획안

> **작성일**: 2026-01-24  
> **벤치마킹 대상**: PFMEA 기초정보 Import 시스템  
> **목표**: PFMEA의 공정번호 기반 모달 입력 방식을 CP에 적용하여 사용성 개선

---

## 📊 현재 시스템 비교 분석

### PFMEA 기초정보 입력 방식 (현재)

```
┌─────────────────────────────────────────────────────────────────┐
│                      PFMEA Import 화면                           │
├─────────────────────────┬───────────────────────────────────────┤
│   📋 미리보기 패널        │     📊 분석 DATA 패널                  │
│   (공정번호별 단일항목)    │     (공정번호별 관계형 테이블)           │
│                         │                                       │
│  공정번호 | 공정명        │  공정번호 | 공정명 | 기능 | 특성 | 고장   │
│  ─────────────────       │  ──────────────────────────────────── │
│   10     | 프레스         │   10   |프레스|압출|외관|균열 | ← 클릭! │
│   20     | 가류          │   20   |가류 |가열|온도|화상연|         │
│                         │                                       │
│  [항목 선택 드롭다운]      │  셀 클릭 → 해당 공정 정보 모달 팝업     │
└─────────────────────────┴───────────────────────────────────────┘
```

**핵심 특징:**
1. **공정번호 기반 행 구조**: 공정번호가 기준 키
2. **관계형 테이블**: 한 행에 공정번호 + 관련 정보(기능/특성/고장) 표시
3. **셀 클릭 → 모달 입력**: 특정 셀 클릭 시 해당 공정의 정보를 모달로 표시
4. **고장 분석 팝업 (`FailureChainPopup`)**: 고장형태/원인/영향 연결 시각화

### CP 기초정보 입력 방식 (현재)

```
┌──────────────────────────────────────────────────────────────────────┐
│                      CP Import 화면                                   │
├──────────────────────────────────────────────────────────────────────┤
│  [전체 Import] [그룹 Import] [개별 Import] [워크시트 Import]            │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│       관리  │  공정현황(A)  │  관리항목(B)  │  관리방법(C)              │
│  ─────────────────────────────────────────────────────────────────── │
│  □ NO 작업 │ 번호│이름│설명│ 제품│공정│특별│ 방법│주기│책임│대응        │
│  ─────────────────────────────────────────────────────────────────── │
│  □  1  ✏️  │ 10 │프레│원료│ 외관│압력│ CC │ SPC │ 1h │생산│재작업       │
│  □  2  ✏️  │ 20 │가류│가열│ 물성│시간│ CC │ 자동│전수│생산│폐기         │
│                                                                      │
│        ↑ 모든 컬럼이 한 테이블에 표시 (플랫 구조)                        │
└──────────────────────────────────────────────────────────────────────┘
```

**현재 문제점:**
1. ❌ **단순 플랫 테이블**: 모든 컬럼이 한 줄에 나열되어 복잡
2. ❌ **셀 클릭 → 모달 없음**: 직접 입력만 가능, 마스터 데이터 선택 불가
3. ❌ **공정별 연계 뷰 없음**: 공정번호에 연결된 정보를 한눈에 파악 어려움
4. ❌ **Excel에서만 편집 가능**: 웹에서 효율적인 편집 기능 부족

### 2. 우측 패널: **관리계획 DATA 패널 (신규)**

PFMEA의 `ImportRelationPanel`을 벤치마킹한 **CP 관계형 뷰**.

#### 2-1. 탭 구성

| 탭 | 표시 컬럼 | 설명 |
|----|----------|------|
| **공정현황** (A) | 공정번호, 공정명, 레벨, 공정설명, 설비 | 기본 공정 정보 |
| **관리항목** (B) | 공정번호, 제품특성, 공정특성, 특별특성, 스펙 | 품질 특성 정보 |
| **관리방법** (C) | 공정번호, 평가방법, 샘플크기, 주기, 관리방법 | 관리/검사 정보 |
| **대응계획** (D) | 공정번호, 책임1, 책임2, 대응계획 | 이상 발생 시 대응 |

#### 2-2. 셀 클릭 → 모달 입력

```typescript
// 셀 클릭 핸들러
const handleCellClick = (processNo: string, columnKey: string) => {
  // 해당 공정의 마스터 데이터 로드
  const processData = cpMasterData.filter(d => d.processNo === processNo);
  
  // 컬럼별 모달 타입 결정
  const modalType = getModalTypeByColumn(columnKey);
  // 'productChar' → '제품특성 선택 모달'
  // 'evalMethod' → '평가방법 선택 모달'
  // etc.
  
  openInputModal({
    processNo,
    columnKey,
    modalType,
    currentValue: getCellValue(processNo, columnKey),
    availableOptions: processData,
  });
};
```

#### 2-3. 입력 모달 유형

| 모달 유형 | 적용 컬럼 | 데이터 소스 |
|-----------|----------|-------------|
| **텍스트 입력** | 공정명, 공정설명, 스펙 | 자유 입력 |
| **선택 모달** | 제품특성, 공정특성, 평가방법 | PFMEA → CP 연동 마스터 |
| **체크박스** | 특별특성 (CC/SC/IC) | 선택 |
| **드롭다운** | 레벨 (Main/Sub), 주기 | 고정 옵션 |
| **책임자 선택** | 책임1, 책임2 | 사용자 목록 |

---

### 3. 공정별 입력 모달 상세 설계

#### 3-1. 제품특성/공정특성 선택 모달

```
┌────────────────────────────────────────────────────────────┐
│  🔍 공정 10 - 프레스 │ 제품특성 선택                         │
├────────────────────────────────────────────────────────────┤
│  ▶ PFMEA 연동 데이터 (자동 로드)                             │
│  ────────────────────────────────────────────────────────  │
│  │ FM (고장형태)  │ A4 제품특성   │ 특별특성 │ 선택         │
│  ├───────────────┼──────────────┼────────┼───────────── ─ │
│  │ 균열 발생      │ 외관불량      │   CC   │ ☑ 선택       │
│  │ 치수 이탈      │ 치수불량      │   SC   │ □            │
│  │ 표면 거칠기    │ 표면조도      │   -    │ □            │
│  ├───────────────┴──────────────┴────────┴───────────────┤
│  │ ▶ 직접 입력                                            │
│  │  [               ] + [추가]                            │
│  ├───────────────────────────────────────────────────────┤
│  │                    [적용] [취소]                        │
│  └───────────────────────────────────────────────────────┘
```

**동작 방식:**
1. 해당 공정의 PFMEA 데이터 자동 로드 (제품특성 + 고장형태)
2. 체크박스로 다중 선택 가능
3. 직접 입력 옵션 제공
4. 적용 시 선택한 값들이 해당 셀에 저장

#### 3-2. 평가/측정방법 선택 모달

```
┌────────────────────────────────────────────────────────────┐
│  📊 공정 10 - 프레스 │ 평가/측정방법 선택                     │

│  ▶ 추천 방법 (공정특성 기반)                                 │

│  ├───────────────┴──────────────┴───────────────────────┤

```

## ✅ 결론

PFMEA의 **"공정번호 기반 모달 입력"** 방식을 CP에 적용하면:

1. **공정번호를 기준으로 관련 정보를 그룹화**하여 표시
2. **셀 클릭 시 해당 공정의 마스터 데이터**를 모달로 제안
3. **PFMEA 데이터 자동 연동**으로 중복 입력 최소화
4. **웹 기반 직관적 UI**로 Excel 의존도 감소

이를 통해 CP 기초정보 입력 시간을 **50% 이상 단축**하고, 데이터 품질을 크게 개선할 수 있습니다.
