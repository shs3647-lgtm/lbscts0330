# FMEA → CP 매핑 규칙 (확정)

> **버전**: 3.0  
> **작성일**: 2026-02-04  
> **목적**: FMEA 데이터를 CP로 동기화할 때의 매핑 규칙 정의

---

## 1. CP 병합 구조 (핵심)

```
공정번호/공정명 (최상위 병합)
├── 부품명 (병합) ← 제품특성 부모
│   ├── 제품특성1
│   ├── 제품특성2
│   └── 제품특성3
│
└── 설비 (병합) ← 공정특성 부모
    ├── 공정특성1
    ├── 공정특성2
    └── 공정특성3
```

---

## 2. CP 행 수 결정

```
CP 행 수 = 제품특성 개수 + 공정특성 개수
```

**배치 순서**:
1. 공정별 제품특성 (부품명 아래)
2. 공정별 공정특성 (설비 아래, MN 제외)

---

## 3. 부모-자식 관계

| 부모 컬럼 | 자식 컬럼 | 설명 |
|----------|----------|------|
| 공정번호/공정명 | 전체 행 | 최상위 병합 |
| **부품명 (partName)** | **제품특성 (productChar)** | 제품특성 부모 |
| **설비 (equipment, 4M=MC)** | **공정특성 (processChar)** | 공정특성 부모 |

---

## 4. FMEA → CP 컬럼 매핑

| CP 컬럼 | FMEA 소스 | 병합 | 설명 |
|--------|----------|:----:|------|
| 공정번호 | `L2.no` | ✅ | 10, 20, 30... |
| 공정명 | `L2.name` | ✅ | 메인공정명 |
| 레벨 | `L2.level` | ✅ | Main |
| 공정설명 | `L2.functions[].name` | ✅ | 공정기능 |
| **부품명** | `partName` | ✅ | 제품특성 부모 |
| **설비/금형/JIG** | `[4M] L3.name` (MC만) | ✅ | 공정특성 부모 |
| **제품특성** | `L2.functions[].productChars[].name` | | 부품명 자식 |
| **공정특성** | `L3.functions[].processChars[].name` | | 설비 자식 |
| 특별특성 | `specialChar` | | CC/SC/IC |

---

## 5. 4M 제외 규칙

| 4M 코드 | 설명 | CP 포함 |
|--------|------|:------:|
| MC | 기계/설비 | ✅ |
| MD | 금형/치구 | ✅ |
| JG | 지그/공구 | ✅ |
| IM | 투입자재 | ✅ |
