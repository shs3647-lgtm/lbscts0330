# 고장연결 Import 검증 체크리스트

> 작성일: 2026-02-24 | 버전: v1.0
> 
> Import 후 고장연결 데이터 정합성 검증을 위한 체크리스트

---

## 1. 수량 검증 (Count Validation)

### 1.1 FM (고장형태) 수량

| 검증 항목 | 기준 | 확인 방법 |
|----------|------|----------|
| Import FM 수 = DB FM 수 | 공정별 FM 수 일치 | 워크시트 구조분석 탭 확인 |
| FM 누락 = 0 | 모든 FM이 DB에 존재 | 고장연결 전체보기에서 확인 |
| 자동생성 FM 수 | 콘솔 로그에서 확인 | `자동생성: FM=XX` |

**체크리스트:**
- [ ] 공정별 FM 수가 Import 원본과 일치
- [ ] FM 누락 수 = 0
- [ ] 예상 FM 수와 DB FM 수 일치

### 1.2 FC (고장원인) 수량

| 검증 항목 | 기준 | 확인 방법 |
|----------|------|----------|
| Import FC 수 = DB FC 수 | FM별 FC 수 일치 | 고장분석 탭에서 확인 |
| FC 누락 = 0 | 모든 FC가 연결됨 | 고장연결 전체보기에서 확인 |
| 자동생성 FC 수 | 콘솔 로그에서 확인 | `자동생성: FC=XX` |

**체크리스트:**
- [ ] FM별 FC 수가 Import 원본과 일치
- [ ] FC 누락 수 = 0
- [ ] 연결 비율 ≥ 95%

### 1.3 FE (고장영향) 수량

| 검증 항목 | 기준 | 확인 방법 |
|----------|------|----------|
| Import FE 수 ≤ DB FE 수 | FE 재사용으로 중복 감소 가능 | L1 탭에서 확인 |
| FE 누락 = 0 | 모든 FE가 연결됨 | 고장연결 전체보기에서 확인 |
| 자동생성 FE 수 | 콘솔 로그에서 확인 | `자동생성: FE=XX` |

**체크리스트:**
- [ ] 고유 FE 수가 적절함
- [ ] FE 누락 수 = 0
- [ ] FE 중복 생성 없음

## 2. 일치성 검증 (Consistency Validation)

### 2.1 FM-FC 매핑 정합성

| 오류 유형 | 설명 | 확인 방법 |
|----------|------|----------|
| FC 잘못매핑 | 다른 FM의 FC가 현재 FM에 연결 | 샘플 FM의 FC 텍스트 확인 |
| 공정번호 불일치 | FC 공정번호 ≠ FM 공정번호 | FC 텍스트 접두사 확인 (예: "10번_MC_01") |

**체크리스트:**
- [ ] 샘플 FM 3개의 FC가 모두 동일 공정번호 접두사
- [ ] FC 텍스트에 FM 공정번호 포함
- [ ] Cross-contamination 없음 (다른 공정 FC 혼입)

**샘플 검증:**
```
FM: "10번F01_품명,규격,사양 불일치"
  → FC들: "10번_MC_01...", "10번_MC_02..." ✓
  → FC들: "20번_MC_01..." ✗ (Cross-contamination!)
```

### 2.2 FM-FE 매핑 정합성

| 오류 유형 | 설명 | 확인 방법 |
|----------|------|----------|
| FE 잘못매핑 | 동일 3-4개 FE만 반복 | 다양한 FE 연결 확인 |
| FE scope 불일치 | C/Y/U scope 매칭 오류 | FE 접두사 확인 |

**체크리스트:**
- [ ] FM별 FE가 다양함 (동일 FE 반복 아님)
- [ ] FE scope(C/Y/U)가 FM 특성에 맞음
- [ ] FE 텍스트가 FM 내용과 관련됨

**주의 패턴 (잘못된 예):**
```
모든 FM에 동일 FE 반복:
  - "C2-5:선별 부적합(외관,치수,조립성)으로 선별"
  - "U5-6:보조 기능 상실_전화 알림 미표시로 고객 불만 발생"
  - "Y1-6선별재작업_ 외관,이종자재 투입으로 라인내 선별 재작업"
→ FE 매칭 로직 오류!
```

### 2.3 공정번호 파싱 정합성

| 오류 유형 | 설명 | 확인 방법 |
|----------|------|----------|
| 공정번호 파싱 오류 | "140F04" → "14004" | 공정 목록에서 이상값 확인 |
| 공정번호 정규화 실패 | "010" ≠ "10" | 공정 매칭 실패 여부 |

**체크리스트:**
- [ ] 모든 공정번호가 2-3자리 숫자
- [ ] "14004" 같은 이상 공정번호 없음
- [ ] 공정 수가 Import 원본과 일치

## 3. 콘솔 로그 검증

### 3.1 필수 확인 로그

```
[FailureChainInjector] ═══ 최종 결과 ═══
  ┌── 입력 ──┐
  │ chains: XXX건          ← Import FC_고장사슬 행 수
  │ 고유 공정: XX개         ← 공정 수
  │ 고유 FM: XXX개          ← Import FM 수
  │ 고유 FE: XX개           ← Import FE 수
  └──────────┘
  ┌── 결과 ──┐
  │ 생성 links: XXX건       ← 생성된 고장연결 수 (≈ chains)
  │ 스킵: X건               ← 0에 가까워야 함
  │ 연결된 FM: XXX개        ← ≈ 고유 FM
  │ 연결된 FE: XX개         ← ≈ 고유 FE
  │ 연결된 FC: XXX개        ← ≈ chains
  │ 자동생성: FE=XX, FM=XX, FC=XX  ← 누락 시 자동 생성
  └──────────┘
```

**체크리스트:**
- [ ] `생성 links` ≈ `chains` (95% 이상)
- [ ] `스킵` ≤ 5%
- [ ] 경고 메시지 없음

### 3.2 경고 메시지

| 경고 | 의미 | 조치 |
|------|------|------|
| `공정 없음 (pNo=XX)` | 해당 공정번호 미매칭 | 공정번호 형식 확인 |
| `⚠️ 연결 비율 낮음: XX%` | 50% 미만 연결 | 데이터 재검토 |

## 4. 시각적 검증

### 4.1 워크시트 화면

**체크리스트:**
- [ ] 구조분석 탭: 모든 공정에 FM 존재
- [ ] 고장분석 탭: 각 FM에 FC 연결
- [ ] 고장연결 전체보기: FM-FC-FE 트리 완성

### 4.2 고장연결 SVG 다이어그램

**체크리스트:**
- [ ] 모든 FM에서 FE로 연결선 존재
- [ ] 모든 FM에서 FC로 연결선 존재
- [ ] 연결선 없는 고아(orphan) FM 없음

| 지표 | 정상 | 주의 | 오류 |

| 스킵 비율 | ≤5% | 5-20% | >20% |
| 자동생성 비율 | ≤30% | 30-50% | >50% |

## 6. 오류 발견 시 대응

### 6.1 FM 누락

```
원인: buildWorksheetState에서 A5 데이터 미파싱
조치: excel-parser 로그 확인 → A5 시트 헤더 점검
```

### 6.2 FC Cross-Contamination

```
원인: 공정번호 정규화 실패 또는 FM 텍스트 매칭 오류
조치: excel-parser-fc.ts 로그에서 processNo 확인
```

### 6.3 FE 반복 매핑

```
원인: FE scope 매칭 실패 → text-only fallback으로 첫 FE 반복
조치: FC 시트에 FE구분 컬럼 확인 또는 FE 텍스트 접두사 확인
```

### 6.4 공정번호 이상값

```
원인: "140F04" 같은 복합 공정번호 미처리
조치: sanitizeProcessNo 함수가 정상 작동 확인
예시: "140F04" → "140" (선행 숫자만 추출)
```
