# CP → PFD 연동 버그 수정 보고서

**작성일**: 2026-01-27  
**상태**: ✅ 수정 완료  
**심각도**: 🔴 Critical

---

## 1. 문제 현상

### 증상
- CP 워크시트에서 "PFD로 연동" 버튼을 클릭
- 연동 완료 메시지와 함께 PFD 워크시트로 이동
- **PFD 워크시트에 아무 데이터도 표시되지 않음**

### 스크린샷
- CP 화면: 21건의 데이터가 있음
- PFD 화면: 빈 테이블만 표시됨

---

## 2. 원인 분석

### 근본 원인: `/api/pfd/sync-from-cp` API가 DB에 저장하지 않음

기존 코드 문제점:

```typescript
// ❌ 기존 코드 (route.ts)
// 데이터 변환만 수행하고 DB 저장 없이 반환
const pfdItems = items.map((item, idx) => ({ ... }));
return NextResponse.json({
  success: true,
  data: { items: pfdItems }  // localStorage로 전달 의도
});
```

### 연동 실패 흐름

1. **CP 페이지**: API 호출 → 응답 받음 → localStorage에 저장 → 리다이렉트
2. **PFD 페이지**: DB에서 데이터 조회 시도 → PFD가 DB에 없음 → **404**
3. **결과**: 빈 화면

### 추가 문제
- localStorage 의존성이 불안정 (브라우저 새로고침 시 소실)
- PFD 로딩 로직이 `fromCpParam`와 localStorage를 먼저 확인하지만, 타이밍 이슈 발생

### 3.1 `/api/pfd/sync-from-cp/route.ts` - DB 저장 추가

```typescript
// ✅ 수정된 코드
import { getPrisma } from '@/lib/prisma';

export async function POST(request: NextRequest) {
  const prisma = getPrisma();
  
  // 1. PFD가 없으면 새로 생성
  let existingPfd = await prisma.pfdRegistration.findUnique({
    where: { pfdNo: targetPfdNo },
  });
  
  if (!existingPfd) {
    existingPfd = await prisma.pfdRegistration.create({
      data: { pfdNo: targetPfdNo, cpNo, status: 'draft' },
    });
  }
  
  // 2. 기존 PfdItem 삭제 (soft delete)
  await prisma.pfdItem.updateMany({
    where: { pfdId: existingPfd.id },
    data: { isDeleted: true },
  });
  
  // 3. 새 PfdItem 저장
  for (const item of items) {
    await prisma.pfdItem.create({
      data: { pfdId, processNo, processName, ... },
    });
  }
  
  // 4. 문서 연결 생성
  await prisma.documentLink.create({ ... });
}
```

## 8. 2차 수정 (2026-01-27 15:26)

### 8.1 CP→PFD 컬럼 매핑 수정

**문제**: 설비/금형/지그 데이터가 누락됨

**원인**: CP 스키마에서 컬럼 역할 혼동
- CP `partName`: 부품명
- CP `workElement`: 설비/금형/JIG (★ 이것이 실제 설비 정보!)
- CP `equipment`: 별도 설비 필드

**수정**:
```typescript
// route.ts - CP→PFD 매핑
workElement: item.partName || '',           // PFD 부품명 ← CP 부품명
equipment: item.workElement || item.equipment || '',  // PFD 설비 ← CP 설비
```

### 8.2 PFD 렌더러 수정

**문제**: NO 컬럼 번호 및 빈 값 표시가 CP와 불일치

**수정 내용**:
1. **NO 컬럼**: 공정별 특성 순번으로 표시 (공정 바뀌면 1로 리셋)
2. **빈 값 표시**: `미입력` → `-`로 변경 (CP와 동일)

```typescript
// renderers/index.tsx - NO 컬럼 로직 (CP charNo와 동일)
const currentProcess = `${item.processNo}-${item.processName}`;
let charIndex = 1;
for (let i = 0; i < rowIdx; i++) {
  const prevItem = items[i];
  const prevProcess = `${prevItem.processNo}-${prevItem.processName}`;
  if (prevProcess === currentProcess) {
    charIndex++;
  }
}
```
