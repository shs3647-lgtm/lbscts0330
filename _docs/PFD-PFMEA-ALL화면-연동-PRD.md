### 1.2 자동화 가능 구간 vs 수동 필수 구간

| 단계 | 자동화 | 설명 |
|------|--------|------|
| CP/PFD → 구조분석 | ✅ 가능 | 공정→L2, 작업요소→L3 |
| 구조분석 → 기능분석 | ✅ 가능 | 특성 연결 |
| 기능분석 → **고장분석** | ❌ 수동 | 고장형태/원인/영향 정의 |
| 고장분석 → **고장연결** | ❌ 수동 | 고장사슬 연결 (핵심) |
| 고장연결 → ALL 화면 | ✅ 가능 | 역전개 표시 |

## 2. CP/PFD → ALL 직접 연동이 불가능한 이유

### 2.1 핵심 불가 요인

```
CP/PFD에 있는 정보:
  ✅ 공정번호, 공정명
  ✅ 작업요소, 설비
  ✅ 제품특성, 공정특성
  ✅ 관리방법, 검사항목

CP/PFD에 없는 정보 (FMEA 핵심):
  ❌ 고장형태 (무엇이 고장나는가?)
  ❌ 고장원인 (왜 고장나는가?)
  ❌ 고장영향 (고장나면 어떻게 되는가?)
  ❌ 인과관계 (어떤 원인이 어떤 형태를 만드는가?)
  ❌ 고장연결 (L1-L2-L3 사슬)
```

### 2.2 ALL 화면 생성 조건

ALL 화면을 표시하려면:

1. **고장연결 완료** 필수
   - L1 고장영향 ← L2 고장형태 ← L3 고장원인 연결
   
2. **상하관계 정의** 필수
   - 어떤 고장원인이 어떤 고장형태의 원인인지
   - 어떤 고장형태가 어떤 고장영향을 만드는지

3. **역전개 가능** 조건
   - 고장연결이 없으면 역전개 불가
   - → ALL 화면 표시 불가

## 3. 현재 구현 상태 (확정)

### 3.1 CP/PFD → PFMEA 연동 범위

| 연동 대상 | 상태 | 도착 화면 |
|----------|------|----------|
| 구조분석 | ✅ 완료 | structure 탭 |
| 기능분석 | 🔶 부분 | 특성 연결 |
| 고장분석 | ❌ 수동 | 사용자 작업 |
| 고장연결 | ❌ 수동 | 사용자 작업 |
| ALL 화면 | ❌ 불가 | 고장연결 후 가능 |

### 3.2 연동 후 사용자 작업 흐름

```
1. PFD 워크시트 → "PFMEA 연동" 버튼 클릭
   ↓
2. PFMEA 구조분석 화면으로 이동 (L2/L3 자동 생성)
   ↓
3. [사용자] 구조분석 확인/수정
   ↓
4. [사용자] 기능분석 → 특성별 기능 입력
   ↓
5. [사용자] 고장분석 → 고장형태/원인/영향 입력
   ↓
6. [사용자] 고장연결 → 고장사슬 연결
   ↓
7. ALL 화면 → 역전개된 전체 데이터 확인
```

## 4. 결론

### 4.1 아키텍처 확정

- **CP/PFD → ALL 직접 연동**: ❌ 불가능 (원래 불가능한 구조)
- **CP/PFD → 구조분석 연동**: ✅ 가능 (현재 구현됨)
- **ALL 화면 표시**: 고장연결 완료 후에만 가능

### 4.2 이유

> "고장형태를 클릭해서 고장사슬을 연결하면, 이 기준으로 상하관계가 정의된 내용이 역전개해서 구조분석/기능분석이 완성되는 구조"

ALL 화면은 **고장연결의 결과물**이지, 입력 데이터가 아닙니다.  
따라서 고장연결 없이 ALL 화면을 만드는 것은 **원리적으로 불가능**합니다.

### 4.3 향후 검토 사항

| 항목 | 우선순위 | 설명 |
|------|----------|------|
| 고장형태 마스터 | 중간 | 입력 편의성 향상 |
| 고장연결 UI 개선 | 높음 | 사용자 경험 개선 |
| 역전개 로직 최적화 | 낮음 | 성능 개선 |
