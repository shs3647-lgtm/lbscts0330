## 버그 상세 분석

### 🔴 BUG-1: `equipmentM4` Prisma 런타임 오류 (CRITICAL)

**증상**: CP 생성 전체연동, CP 구조연동, CP→PFMEA 역연동 시 아래 오류 발생
```
Invalid 'tx.controlPlanItem.create()' invocation
Unknown argument 'equipmentM4'. Did you mean 'equipment'?
```

**영향 범위**: 
- `PFMEA → CP 생성` (create-cp/route.ts:271, 306)
- `PFMEA → CP 구조연동` (sync-to-cp/structure/route.ts:166)
- `PFMEA → CP 전체연동` (sync-to-cp/all/route.ts:346, 379)
- `CP → PFMEA 역연동` (pfmea/sync-from-cp route)

**원인 분석**:
- Prisma 스키마(`schema.prisma:972`)에는 `equipmentM4 String?` 필드가 **정의되어 있음**
- `prisma db push` 결과 "The database is already in sync" — **DB 컬럼 존재**
- Prisma 클라이언트 `index.d.ts`에 `equipmentM4` 타입 정의 **존재**
- 그러나 런타임에서 Prisma가 이 필드를 **인식하지 못함**

**추정 원인**:
1. `prisma generate` 실행 후 서버 재시작이 안되었을 가능성
2. 또는 `.next` 캐시에 오래된 Prisma 클라이언트가 캐시되어 있을 가능성

**수정 방안**:
```bash
npx prisma generate        # Prisma 클라이언트 재생성
rm -rf .next               # Next.js 캐시 삭제
npm run dev                # 서버 재시작
```

**관련 소스 파일**:
- `src/app/api/pfmea/create-cp/route.ts` (line 271, 306)
- `src/app/api/pfmea/sync-to-cp/structure/route.ts` (line 166)
- `src/app/api/pfmea/sync-to-cp/all/route.ts` (line 346, 379)
- `prisma/schema.prisma` (line 972 — ControlPlanItem.equipmentM4)

### 🟡 BUG-3: PFD 생성 API 405 Method Not Allowed (HIGH)

**증상**: PFMEA 워크시트에서 "PFD" 버튼 → 확인 팝업 → 서버 `405` 응답

**영향 범위**: PFMEA → PFD 연동 경로

**원인 분석**:
- API 엔드포인트: `/api/pfd/pfd-26-f001-l68-01/items`
- 해당 라우트 파일(`src/app/api/pfd/[id]/items/route.ts`)에는 `POST`와 `PUT` 메서드만 export
- 프론트엔드가 `GET` 또는 다른 메서드로 호출하고 있을 가능성

**수정 방안**:
1. 프론트엔드에서 PFD 생성 시 어떤 HTTP 메서드로 호출하는지 확인
2. 필요한 메서드를 라우트에 추가하거나, 프론트엔드 호출 메서드를 수정

**관련 소스 파일**:
- `src/app/api/pfd/[id]/items/route.ts` (POST, PUT만 정의)
- PFD 버튼 클릭 핸들러 (PFMEA 워크시트 TabMenu 또는 관련 컴포넌트)
