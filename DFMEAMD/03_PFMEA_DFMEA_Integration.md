# DFMEA-PFMEA 연동 (Data Linkage) 황금 파이프라인

## 1. 두 모듈 간의 "연결 고리" 구조
[AIAG-VDA 가이드라인]
설계에 기인한 결함과 제조/공정에 기인한 결함은 동전의 양면과 같습니다. 시스템 단절(Silo)을 막고 완벽한 협업 시스템을 구축하려면 **DFMEA의 특정 데이터가 PFMEA의 시작점**이 되도록 연결(Bridge)해야 합니다.

### 연결 모델 1: 부분품 특성 연계
* **DFMEA:** 제품의 "설계적 요구사항/공차" (예: "지름 10mm ±0.2 확보")
* **PFMEA:** 위 조건 만족을 위한 "공정 목적(L2 Function)"으로 이관. (예: "지름 검사 공정을 통해 10mm ±0.2 달성 보장")

### 연결 모델 2: 고장원인(FC) ➔ 고장형태(FM) 연계
**[DFMEA의 Failure Cause(FC)]** (설계 변경으로 완전히 막을 수 없어서 공정에 부담을 넘긴 이슈들)
⏬ (버튼 1-Click 자동 스크랩)
**[PFMEA의 Failure Mode(FM)]** 로 강제 치환되어 제조팀 공정 엔지니어의 PFMEA 워크시트에 "반드시 조치해야 할 항목" 뱃지가 찍힌 채 Import 됨.

## 2. 향후 시스템(UI) 연동 모델 제안
1. **PFMEA 워크시트 상단 헤더 영역:** `DFMEA 데이터 불러오기` 버튼 장착 (이미 개발된 LLD Import 포맷 재사용)
2. **연관 프로젝트 추적 (Project Linkage):** 한 차종(Vehicle)에 대해 개발된 DFMEA 프로젝트 No. 와 PFMEA 프로젝트 No. 를 트리거 연동하는 SSoT 로직 구축.

이 파이프라인 개발이 본 프로젝트의 품질 우수성을 폭발적으로 끌어올리는 킬러 솔루션이 됩니다.
