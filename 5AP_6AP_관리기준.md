# 5AP / 6AP 관리기준

> **용도**: Smart FMEA 리스크 분석 — 5단계(현재) AP 판정 및 6단계(최적화) AP 개선 관리
> **AP 산출 기준**: AIAG & VDA FMEA Handbook 1st Edition (2019)
> **PFMEA 원칙**: 심각도(S)는 설계 영역 → PFMEA에서 변경 불가. 발생도(O)와 검출도(D)만 개선 가능.

---

## 1. 5단계 / 6단계 정의

| 구분 | 단계 | 의미 | SOD 입력 시점 |
|------|------|------|-------------|
| **5단계** | 현재 상태 | 현재 예방관리(PC) + 검출관리(DC) 기준의 S/O/D → AP 판정 | 고장사슬 생성 시 |
| **6단계** | 최적화 | 개선활동 완료 후 재평가된 S/O/D → AP 재판정 | 개선활동 완료 후 |

### 2.3 AP=H/M → 최단거리 L 개선방향 제시

> **5단계 AP가 H 또는 M인 경우**, 발생도(O)와 검출도(D)를 최소한으로 낮추어 **AP=L에 도달하는 최단거리 개선방향**을 자동 제시한다.

**최단거리 원칙**:

| # | 규칙 | 설명 |
|---|------|------|
| 1 | **AP=L 도달 최소 이동** | 현재 O/D에서 AP=L이 되는 가장 가까운 O/D 조합을 탐색 |
| 2 | **O=1, D=1은 제외** | 발생도 1(발생 거의 불가) 및 검출도 1(확실히 검출)은 비현실적이므로 개선 목표에서 제외 |
| 3 | **S는 고정** | 심각도는 PFMEA에서 변경 불가 → O와 D만 변경하여 탐색 |
| 4 | **개선방향 표시** | ↓ = D만 개선(검출관리 강화), → = O만 개선(예방관리 강화), ↘ = O+D 동시 개선 |

**최단거리 탐색 알고리즘**:

```
입력: 현재 S, O, D (5단계)
조건: AP(S, O, D) = H 또는 M

1. 후보 목록 생성:
   - O 범위: 2 ~ 현재O (O=1 제외)
   - D 범위: 2 ~ 현재D (D=1 제외)
   - 각 (O후보, D후보) 조합에 대해 AP(S, O후보, D후보) 계산

2. AP=L인 후보만 필터

3. 최단거리 선택:
   - 거리 = |현재O - O후보| + |현재D - D후보| (맨해튼 거리)
   - 거리가 같으면 O 감소를 우선 (예방관리 강화 우선)

4. 개선방향 결정:
   - O후보 < 현재O 이고 D후보 < 현재D → ↘ (O+D 동시 개선)
   - O후보 < 현재O 이고 D후보 = 현재D → → (O만 개선, 예방관리)
   - O후보 = 현재O 이고 D후보 < 현재D → ↓ (D만 개선, 검출관리)

5. 출력: 목표O, 목표D, 개선방향(↓/→/↘), 개선 내용(예방관리/검출관리/예방관리+검출관리)
```

**최단거리 개선방향 예시**:

| 현재 S | 현재 O | 현재 D | 현재 AP | → 목표 O | 목표 D | 목표 AP | 방향 | 개선 내용 |
|--------|--------|--------|---------|----------|--------|---------|------|----------|
| 8 | 5 | 6 | M | 5 | 4 | L | ↓ | 검출관리 강화 |
| 8 | 5 | 7 | H | 3 | 7 | M→탐색계속 | → | 예방관리 강화 |
| 8 | 4 | 5 | M | 3 | 5 | M→탐색계속 | → | - |
| 8 | 4 | 5 | M | 2 | 5 | M→탐색계속 | → | - |
| 8 | 4 | 5 | M | 4 | 4 | L | ↓ | 검출관리 강화 ✅ 최단 |
| 6 | 7 | 6 | M | 7 | 4 | L | ↓ | 검출관리 강화 ✅ 최단(거리2) |
| 6 | 7 | 6 | M | 5 | 6 | L | → | 예방관리 강화 (거리2, 동일거리) |
| 6 | 8 | 6 | H | 6 | 6 | M→탐색계속 | → | - |
| 6 | 8 | 6 | H | 6 | 4 | L | ↘ | 예방관리+검출관리 ✅ |

## 3. 6단계 AP (최적화)

### 3.1 6단계 핵심 규칙

| # | 규칙 | 설명 |
|---|------|------|
| 1 | **심각도(S) 항상 일치** | 6단계 S = 5단계 S (변경 불가, 항상 동일값) |
| 2 | **기본값 = 5단계와 동일** | 개선활동이 완료되지 않은 경우, 6단계 O = 5단계 O, 6단계 D = 5단계 D |
| 3 | **수정 조건: 개선활동 완료** | 예방관리 또는 검출관리 개선활동이 **완료된 경우에만** 6단계 O/D 수정 가능 |
| 4 | **미완료 시 편집 잠금** | 개선활동 미완료 → 6단계 O/D 셀 편집 불가 (읽기전용, 5단계 값 표시) |
| 5 | **AP 자동 재산출** | 6단계 O/D 변경 시 → AP(S, 6단계O, 6단계D) 자동 재산출 |

### 3.2 6단계 O/D 수정 조건

```
6단계 O 수정 가능 조건:
  - 해당 고장사슬의 예방관리(PC) 개선활동이 완료됨 (LLD 상태 = G)
  - 또는 예방관리+검출관리 동시 개선활동이 완료됨

6단계 D 수정 가능 조건:
  - 해당 고장사슬의 검출관리(DC) 개선활동이 완료됨 (LLD 상태 = G)
  - 또는 예방관리+검출관리 동시 개선활동이 완료됨

수정 불가:
  - 개선활동 없음 → 5단계 O/D 그대로 표시 (편집 잠금)
  - 개선활동 진행 중(Y) → 5단계 O/D 그대로 표시 (편집 잠금)
  - 개선활동 미착수(R) → 5단계 O/D 그대로 표시 (편집 잠금)
```

### 3.3 6단계 AP 재산출 후 추가 개선 (행 추가 기능)

> **6단계 AP가 여전히 H 또는 M인 경우**, 추가 개선을 위해 **행을 추가**하여 2차/3차 개선활동을 등록할 수 있다.

**행 추가 규칙**:

| # | 규칙 | 설명 |
|---|------|------|
| 1 | **행 추가 트리거** | 6단계 AP = H 또는 M인 경우에만 행 추가 버튼 활성화 |
| 2 | **상위 셀 병합** | 추가된 행의 고장사슬 정보(S, FM, FC, FE 등)는 상위 행과 **셀 병합** 처리 |
| 3 | **새 행의 5단계** | 추가 행의 5단계 S/O/D = 직전 6단계의 S/O/D (이전 개선 결과가 새 시작점) |
| 4 | **새 행의 6단계** | 추가 개선활동 완료 후 재평가된 O/D 입력 |
| 5 | **반복 가능** | 6단계 AP가 L이 될 때까지 행 추가를 반복할 수 있음 |
| 6 | **AP=L 도달 시** | 행 추가 버튼 비활성화 (추가 개선 불필요) |

**행 추가 구조 (셀 병합 예시)**:

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ FM(A5)      │ FC(B4)        │ FE(C4)        │ 5S │ 5O │ 5D │ 5AP │ PC/DC 개선 │ 6S │ 6O │ 6D │ 6AP │
├──────────────┼───────────────┼───────────────┼────┼────┼────┼─────┼────────────┼────┼────┼────┼─────┤
│              │               │               │  8 │  5 │  7 │  H  │ PC: 센서교정│  8 │  4 │  5 │  M  │ ← 1차 개선
│  (셀 병합)   │  (셀 병합)     │  (셀 병합)     ├────┼────┼────┼─────┼────────────┼────┼────┼────┼─────┤
│              │               │               │  8 │  4 │  5 │  M  │ DC: 전수검사│  8 │  4 │  3 │  L  │ ← 2차 개선 (행 추가)
└──────────────┴───────────────┴───────────────┴────┴────┴────┴─────┴────────────┴────┴────┴────┴─────┘
                                                                     ↑ 상위 셀 병합    ↑ AP=L 도달 → 완료
```

**행 추가 상세 플로우**:

```
[1차 평가]
  5단계: S=8, O=5, D=7 → AP=H
  개선방향: ↘ (O+D 동시 개선, 최단거리)
  예방관리 개선: 센서 교정 주기 단축
  → 개선 완료 후
  6단계: S=8, O=4, D=5 → AP=M (여전히 M!)

[행 추가 → 2차 개선]
  새 행 5단계: S=8, O=4, D=5 → AP=M (1차 6단계 결과가 시작점)
  개선방향: ↓ (D만 개선, 최단거리)
  검출관리 개선: 전수검사 도입
  → 개선 완료 후
  새 행 6단계: S=8, O=4, D=3 → AP=L ✅ (목표 달성!)
  → 행 추가 버튼 비활성화
```

## 4. 5AP ↔ 6AP 관계 요약

```
5단계 (현재)                          6단계 (최적화)
┌─────────────────┐                  ┌─────────────────┐
│ S = 현재 심각도   │ ───── 항상 ────→ │ S = 동일 (불변)   │
│ O = 현재 발생도   │                  │ O = 개선후 발생도  │
│ D = 현재 검출도   │                  │ D = 개선후 검출도  │
│ AP = f(S,O,D)   │                  │ AP = f(S,O,D)   │
└─────────────────┘                  └─────────────────┘
        │                                    │
        │ AP = H or M                        │ AP = H or M
        ↓                                    ↓
  최단거리 L방향 제시                     행 추가 → 추가 개선
  (O=1, D=1 제외)                       (반복 가능)
        │                                    │
        │                                    │ AP = L
        │                                    ↓
        │                               개선 완료 ✅
        │
        │ AP = L
        ↓
  개선 불필요 (유지)
```

### 데이터 흐름 규칙

| 상태 | 6단계 S | 6단계 O | 6단계 D | 6단계 AP |
|------|---------|---------|---------|----------|
| 개선활동 없음 | = 5단계 S | = 5단계 O (잠금) | = 5단계 D (잠금) | = 5단계 AP |
| 개선활동 미착수(R) | = 5단계 S | = 5단계 O (잠금) | = 5단계 D (잠금) | = 5단계 AP |
| 개선활동 진행중(Y) | = 5단계 S | = 5단계 O (잠금) | = 5단계 D (잠금) | = 5단계 AP |
| **PC 개선 완료(G)** | = 5단계 S | **편집 가능** | = 5단계 D (잠금) | 자동 재산출 |
| **DC 개선 완료(G)** | = 5단계 S | = 5단계 O (잠금) | **편집 가능** | 자동 재산출 |
| **PC+DC 개선 완료(G)** | = 5단계 S | **편집 가능** | **편집 가능** | 자동 재산출 |

## 5. 행 추가 기능 UI 사양

### 5.1 행 추가 버튼

| 항목 | 사양 |
|------|------|
| **표시 위치** | 6단계 AP 셀 우측 또는 행 끝 |
| **활성화 조건** | 6단계 AP = H 또는 M |
| **비활성화 조건** | 6단계 AP = L, 또는 5단계 AP = L |
| **버튼 레이블** | "+ 추가 개선" 또는 "행 추가" |

### 5.2 행 추가 시 동작

```
1. 새 행 생성
2. 새 행의 고장사슬 정보 = 상위 행과 셀 병합 (FM, FC, FE, B1 등)
3. 새 행의 5단계 S/O/D = 직전 행의 6단계 S/O/D
4. 새 행의 5단계 AP = f(S, 새5O, 새5D) 자동 산출
5. 새 행의 6단계 = 기본적으로 5단계와 동일 (잠금 상태)
6. 새 행에 개선활동 등록 → 완료 시 6단계 O/D 편집 가능
7. 새 행의 6단계 AP 재산출
8. 여전히 H/M이면 → 행 추가 버튼 다시 활성화 (반복)
```

### 5.3 셀 병합 규칙

| 병합 대상 | 범위 | 이유 |
|-----------|------|------|
| FM(A5) | 원본 행 + 추가 행 전체 | 같은 고장형태에 대한 반복 개선 |
| FC(B4) | 원본 행 + 추가 행 전체 | 같은 고장원인에 대한 반복 개선 |
| FE(C4) | 원본 행 + 추가 행 전체 | 같은 고장영향 |
| 작업요소(B1) | 원본 행 + 추가 행 전체 | 같은 작업요소 |
| 4M | 원본 행 + 추가 행 전체 | 같은 4M 분류 |
| S(심각도) | 원본 행 + 추가 행 전체 | S는 항상 동일 |
| **비병합** | 5O, 5D, 5AP, PC/DC, 6O, 6D, 6AP | 각 행마다 고유값 |

## 6. calculateAP() 구현 (5단계/6단계 공통)

```typescript
function calculateAP(s: number, o: number, d: number): 'H' | 'M' | 'L' | '' {
  if (!s || !o || !d) return '';
  if (s < 1 || s > 10 || o < 1 || o > 10 || d < 1 || d > 10) return '';

  if (s === 1) return 'L';

  if (s >= 2 && s <= 3) {
    if (o >= 8 && d >= 5) return 'M';
    return 'L';
  }

  if (s >= 4 && s <= 6) {
    if (o >= 8) {
      if (d >= 5) return 'H';
      return 'M';
    }
    if (o >= 6) {
      if (d >= 2) return 'M';
      return 'L';
    }
    if (o >= 4 && d >= 7) return 'M';
    return 'L';
  }

  if (s >= 7 && s <= 8) {
    if (o >= 8) return 'H';
    if (o >= 6) {
      if (d >= 2) return 'H';
      return 'M';
    }
    if (o >= 4) {
      if (d >= 7) return 'H';
      return 'M';
    }
    if (o >= 2 && d >= 5) return 'M';
    return 'L';
  }

  if (s >= 9) {
    if (o >= 8) return 'H';
    if (o >= 6) return 'H';
    if (o >= 4) {
      if (d >= 2) return 'H';
      return 'M';
    }
    if (o >= 2) {
      if (d >= 7) return 'H';
      if (d >= 5) return 'M';
      return 'L';
    }
    return 'L';
  }

  return 'L';
}
```

## 7. findShortestPathToL() 구현 (최단거리 개선방향)

```typescript
interface ImprovementDirection {
  targetO: number;
  targetD: number;
  targetAP: 'L';
  direction: '↓' | '→' | '↘';   // ↓=D개선, →=O개선, ↘=O+D동시
  improveType: '검출관리' | '예방관리' | '예방관리+검출관리';
  distance: number;               // 맨해튼 거리
}

function findShortestPathToL(
  s: number, o: number, d: number
): ImprovementDirection | null {

  const currentAP = calculateAP(s, o, d);
  if (currentAP === 'L' || currentAP === '') return null;

  let best: ImprovementDirection | null = null;

  // O=1, D=1은 제외 → 최소값 2
  for (let newO = 2; newO <= o; newO++) {
    for (let newD = 2; newD <= d; newD++) {
      if (newO === o && newD === d) continue; // 변화 없음
      if (calculateAP(s, newO, newD) !== 'L') continue;

      const distance = (o - newO) + (d - newD);

      // 방향 결정
      let direction: '↓' | '→' | '↘';
      let improveType: '검출관리' | '예방관리' | '예방관리+검출관리';

      if (newO < o && newD < d) {
        direction = '↘';
        improveType = '예방관리+검출관리';
      } else if (newO < o) {
        direction = '→';
        improveType = '예방관리';
      } else {
        direction = '↓';
        improveType = '검출관리';
      }

      if (!best || distance < best.distance ||
          (distance === best.distance && newO < best.targetO)) {
        best = { targetO: newO, targetD: newD, targetAP: 'L',
                 direction, improveType, distance };
      }
    }
  }

  return best;
}
```
